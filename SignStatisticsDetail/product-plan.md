# 万能零售助手首页签收数量明细查询优化方案

## Part A: 价值层

### 一、文档头

● 文档名称：万能零售助手首页签收数量明细查询优化方案  
● 产品线/模块：万能零售助手 / 首页数据概览 / 签收记录  
● 迭代类型：优化  
● 作者(PM)：待补充  
● 审批人(Owner)：待补充  

### 二、项目目的

1. 补齐门店签收数据追溯能力：门店在首页查看签收数量时，可直接进入对应签收记录列表，查看签收流水与单据明细。
2. 对齐开箱记录体验：签收数量点击后的页面独立命名为【签收记录】，筛选区、数量汇总与列表结构参考【开箱记录】页面。
3. 保持操作边界清晰：【签收记录】仅用于查看已签收记录，不承载扫码签收、快速签收、一键签收等操作。

### 三、现状背景与问题陈述

● 现状背景：客户渠道仅开放门店扫码签收模式，系统已隐藏独立【商品签收】菜单入口。门店在万能零售助手首页查看货品签收统计时，仅能看到签收总数，无法继续查看签收单据流水。  
● 问题：当前【开箱数量】已支持点击数值跳转查看明细，【签收数量】缺少同等的明细查询能力，导致门店无法追溯签收单据与商品明细。  
● 需求：优化首页【签收数量】交互，点击签收数量后进入独立【签收记录】页面；页面样式参考【开箱记录】，记录卡片内容参考【商品签收】，点击记录后进入签收详情。

### 四、用户范围

| 角色 | 诉求 |
| --- | --- |
| 门店负责人/店员 | 查看本店商品签收单据与签收明细，核对签收数量来源 |
| 客户运营/客服 | 辅助门店定位签收数据问题，降低人工排查成本 |

### 五、实现思路

#### 5.1 结构归母

● 需求归属于【万能零售助手 - 首页数据概览】与【签收记录】模块。  
● 首页【签收数量】作为明细查询入口；新增【签收记录】页面作为承接页面，详情页可复用现有签收详情能力。

#### 5.2 设计思路

1. 首页数据概览支持切换【今日 / 本月 / 累计】。
2. 门店点击【签收数量】后，进入对应周期的【签收记录】页面。
3. 【签收数量】入口保持普通统计样式，不增加箭头、蓝框高亮或“点击查看”等提示。
4. 【签收记录】页面参考【开箱记录】布局，保留顶部筛选区与数量汇总，将“开箱数量：X”调整为“签收数量：X”。
5. 记录卡片展示内容参考【商品签收】卡片，包括订单号、出货单号、出货时间、经销商、出货数量、签收数量。
6. 列表仅展示已签收记录，不展示卡片分隔横线及“状态：XXX”行。
7. 点击记录进入签收详情页，详情页可复用现有签收详情能力。

### 六、功能清单

| F01 | 首页签收数量可点击 | UI_01 | 作为门店用户，我想点击首页签收数量，以便查看该统计值对应的签收记录 |
| F02 | 签收记录列表查询 | UI_02 | 作为门店用户，我想在签收记录页查看对应周期的已签收记录 |
| F03 | 签收详情查看 | UI_03 | 作为门店用户，我想点击签收记录查看签收明细，以便核对具体商品数量 |

### 七、页面与界面结构

#### 7.1 页面清单

| 页面ID | 页面名称 |
| --- | --- |
| UI_01 | 万能零售助手首页 - 数据概览 |
| UI_02 | 签收记录 |
| UI_03 | 签收详情 |

#### UI_01 万能零售助手首页 - 数据概览

1. 保留【今日 / 本月 / 累计】切换。
2. 【签收数量】展示对应周期统计值。
3. 点击【签收数量】进入【签收记录】页面，并带入当前周期条件。
4. 【签收数量】不展示箭头、蓝框选中态或“点击查看”提示。

#### UI_02 签收记录

1. 页面标题为【签收记录】。
2. 页面样式参考【开箱记录】，顶部筛选区沿用开箱记录的筛选布局。
3. 数量汇总将【开箱数量：X】调整为【签收数量：X】，数值与首页所选周期的签收统计口径一致。
4. 下方展示签收记录卡片，卡片内容参考【商品签收】页面：订单号、出货单号、出货时间、经销商、出货数量、签收数量。
5. 卡片不展示横线及【状态：XXX】行；列表默认表示已签收记录。
6. 页面不展示底部【快速签收】按钮，不展示【扫码签收 / 一键签收】等签收操作。
7. 点击记录进入签收详情。

#### UI_03 签收详情

1. 展示出货单号、订单号、经销商、状态等基础信息。
2. 展示出货明细、收货信息、商品签收信息。
3. 不展示顶部“只读明细模式”提示栏。
4. 不提供签收类操作按钮。

### 八、核心规则库

#### 8.1 业务规则

| 规则ID | 规则描述 |
| --- | --- |
| R01 | 首页【签收数量】点击后，按当前选择周期进入对应【签收记录】页面。 |
| R02 | 【签收记录】仅展示已签收记录，不承载待签收处理和签收操作。 |
| R03 | 记录卡片与详情数据来源沿用现有签收数据，不新增独立签收流水口径。 |
| R04 | 签收数量统计值与跳转后的列表查询口径需保持一致。 |

#### 8.2 权限规则

| 规则ID | 规则描述 |
| --- | --- |
| PRM01 | 不新增菜单权限；仅在首页【签收数量】开放明细查看入口。 |
| PRM02 | 若客户已隐藏独立【商品签收】菜单，仍允许通过首页签收数量进入【签收记录】查看已签收数据。 |
| PRM03 | 签收操作权限不因该入口开放而新增。 |

#### 8.3 交互规则

| 规则ID | 规则描述 |
| --- | --- |
| ITR01 | 【签收数量】保持普通统计卡片样式，点击数值区域进入明细。 |
| ITR02 | 列表页返回后回到首页，并保留用户当前选择的统计周期。 |
| ITR03 | 签收详情页返回后回到【签收记录】列表。 |

#### 8.4 旧数据处理规则

● 是否需要：否  
● 说明：本需求仅新增查询入口与签收记录展示页，签收记录和明细沿用现有数据，无需数据迁移或历史数据回填。
