# 整体业务流程

```mermaid
graph TD
    A[系统启动] --> B[系统初始化配置]
    B --> C{是否首次配置?}
    C -->|是| D[配置设备接口]
    D --> E[配置系统参数]
    E --> F[配置产品信息]
    F --> G[系统测试验证]
    G --> ConnCheck1{设备连接是否成功?}
    ConnCheck1 -->|是| H[生产前准备]
    ConnCheck1 -->|否| D
    C -->|否| ConnCheck2{设备连接是否成功?}
    ConnCheck2 -->|是| H
    ConnCheck2 -->|否| D
    H --> I[导入码包]
    I --> J[码包比对]
    J --> K[选择生产产品]
    K --> L[配置包装比例]
    L --> M[启动生产执行;开始读码]
    M --> N[1号机: 瓶箱垛关联采集]
    N --> P{是否成垛?}
    P -->|否| N
    P -->|是| Q[自动上传数据]
    Q --> R{上传是否成功?}
    R -->|成功| S[绿灯亮一分钟]
    R -->|失败| T[红灯常亮+声音提醒]
    T --> T1[根据失败提示进行数据处理]
    T1 --> T2[数据处理完成]
    T2 --> V[手动重新上传]
    S --> U{是否继续生产?}
    V --> U
    U -->|是| N
    U -->|否| W[生产结束]
    W --> X[统计分析]
    X --> Y[数据维护]
    Y --> Z[结束]
    
    style B fill:#e1f5ff
    style H fill:#fff4e1
    style M fill:#e8f5e9
    style Q fill:#f3e5f5
    style S fill:#c8e6c9
    style T fill:#ffcdd2
```

**流程说明：** 系统启动后完成初始化配置，首次使用需配置设备接口、系统参数及产品信息并做测试验证；非首次则检查设备连接。配置完成后进入生产前准备，导入码包并比对，选择产品、配置包装比例后启动生产。生产完成瓶箱垛关联采集，成垛后自动上传数据，上传成功则绿灯亮一分钟，失败则红灯常亮并声音提醒，需根据提示处理数据后手动重新上传。生产结束后可进行统计分析及数据维护。

# 码包对比流程

```mermaid
graph TD
    Start[生产前准备] --> ImportCode[导入码包、包材厂数据回传的码包文件]
    ImportCode --> MultiCode[支持多个码包、数据累计导入]
    MultiCode --> CheckSource{码包来源}
    
    CheckSource -->|在线导入| OnlineImport[软件启动时、在线导入码关系]
    CheckSource -->|本地导入| LocalImport[本地导入码包文件、存储到数据库]
    
    OnlineImport --> CheckNetwork{网络是否正常?}
    CheckNetwork -->|正常| DownloadCode[下载码包数据]
    CheckNetwork -->|异常| UseLocal[使用本地备份码包]
    
    LocalImport --> LoadLocal[加载本地码包文件]
    LoadLocal --> ParseCode[解析码包数据]
    
    DownloadCode --> ParseCode
    UseLocal --> ParseCode
    
    ParseCode --> ValidateCode[校验码包格式、一般为txt: 盖外码每行小标/箱码每行大标]
    ValidateCode -->|格式错误| ShowError[显示错误提示]
    ValidateCode -->|格式正确| SaveCode[保存码包到本地数据库、数据累计]
    
    SaveCode --> StartProduction[开始生产]
    
    ShowError --> RetryImport[重新导入]
    RetryImport --> ImportCode
    
    StartProduction --> DuringProd[生产过程中]
    DuringProd --> CollectCode[采集盖外码、箱码]
    CollectCode --> CompareCheck{采集的码、是否在导入的码包范围内?}
    CompareCheck -->|在范围内| NormalCollect[正确采集关联]
    CompareCheck -->|不在范围内| MarkReject[标记待剔除]
    MarkReject --> WaitPosition[等箱移动到剔除位置]
    WaitPosition --> RejectBox[剔除对应箱]
    
    NormalCollect --> DuringProd
    RejectBox --> DuringProd
    
    style CompareCheck fill:#fff9c4
    style MarkReject fill:#ffcdd2
```

**流程说明：** 生产前导入码包（包材厂回传的码包文件），支持多码包累计导入。码包可在线导入（启动时下载）或本地导入；在线导入时若网络异常则使用本地备份。解析后校验格式（盖外码每行小标、箱码每行大标），格式正确则保存到本地数据库，错误则提示并重新导入。生产过程中采集的盖外码、箱码需与码包比对，在范围内则正常关联，不在范围内则标记待剔除，等箱到剔除位置后执行剔除。

# 瓶箱垛关联流程

```mermaid
graph TD
    Start[生产任务启用] --> CheckCode[码包比对、生产前]
    CheckCode --> CheckResult{包材厂数据回传?}
    CheckResult -->|有数据回传| NormalCheck[正常比对]
    CheckResult -->|无数据回传| MidoCheck[软件校验并告知操作人员]
    NormalCheck --> Stage1
    MidoCheck --> Stage1
    
    Stage1[阶段1: 瓶箱关联 - 1号机]
    Stage1 --> CollectBottle[采集瓶码、12瓶]
    CollectBottle --> CollectBoxCode[箱码相机采集箱码]
    CollectBoxCode --> PackBox[装箱]
    PackBox --> Stage1End[瓶箱关联完成、进入下一步]
    
    Stage1End --> Stage2[阶段2: 剔除判断 - 1号机]
    Stage2 --> RejectCamera[剔除相机采集箱码]
    RejectCamera --> JudgeReject{判断是否要剔除?}
    JudgeReject -->|是| DoReject[执行剔除、下一箱]
    JudgeReject -->|否| Stage2End[通过、进入下一阶段]
    DoReject --> RejectCamera
    
    Stage2End --> Stage3[阶段3: 箱垛关联 - 1号机]
    Stage3 --> CollectBoxPallet[箱码相机采集箱码]
    CollectBoxPallet --> CheckPallet{检查箱属于垛}
    CheckPallet -->|否| CollectBoxPallet
    CheckPallet -->|是| AssociatePallet[建立箱垛关联]
    AssociatePallet --> FullPallet{是否成垛?}
    FullPallet -->|否| Stage1
    FullPallet -->|是| GenPallet[生成虚拟垛标]
    GenPallet --> RetryCheck{极端失败?}
    RetryCheck -->|是| RetryGen[程序内重试生成]
    RetryGen --> GenPallet
    RetryCheck -->|否| SuccessPallet[建立完整码关系]
    SuccessPallet --> AutoUpload[自动上传数据、至米多大数据引擎]
    
    AutoUpload --> UploadResult{上传是否成功?}
    UploadResult -->|成功| GreenLight[绿灯亮一分钟]
    UploadResult -->|失败| RedLight[红灯常亮、+ 声音提醒]
    GreenLight --> Complete[完成一垛的采集生产]
    RedLight --> RetryUpload[手动重新上传]
    RetryUpload --> AutoUpload
    
    style Stage1 fill:#e3f2fd
    style Stage2 fill:#fff3e0
    style Stage3 fill:#e8f5e9
    style Stage1End fill:#e3f2fd
    style Stage2End fill:#fff3e0
    style JudgeReject fill:#fff9c4
    style CheckPallet fill:#fff9c4
    style GenPallet fill:#e8f5e9
    style SuccessPallet fill:#c8e6c9
    style GreenLight fill:#c8e6c9
    style RedLight fill:#ffcdd2
```

**流程说明：** 生产任务启用后先进行码包比对（有包材厂回传则正常比对，无则软件校验并告知操作人员）。

- **阶段1 瓶箱关联：** 采集 12 瓶瓶码，箱码相机采集箱码，装箱后进入下一步。
- **阶段2 剔除判断：** 剔除相机采集箱码，判断是否要剔除；需剔除则执行剔除并处理下一箱，不剔除则进入箱垛关联。
- **阶段3 箱垛关联：** 箱码相机采集箱码，检查箱属于垛后建立箱垛关联；未成垛则继续下一箱（回到阶段1），成垛则生成虚拟垛标、建立完整码关系并自动上传至米多大数据引擎。上传成功则绿灯亮一分钟，失败则红灯常亮并声音提醒，需手动重新上传。
