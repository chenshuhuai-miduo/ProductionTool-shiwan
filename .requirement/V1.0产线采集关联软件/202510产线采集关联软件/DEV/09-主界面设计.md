# 主界面 - 任务执行控制界面

## 子功能名称
主界面 - 任务执行控制功能界面

## 功能概述
作为产线采集关联软件的主界面，为生产线操作员提供任务执行过程中的控制操作界面，支持任务的启用/停用、订单结单、强制满垛等操作，实时显示任务执行状态、数据采集记录、操作日志和报警信息。界面采用专业的工业软件设计风格，包含完整的菜单栏、工具栏和三栏布局，针对工控机鼠标键盘操作优化。

**重要说明**：本界面为系统主界面，其他所有功能模块（如产品管理、任务管理、数据上传等）都通过主界面顶部的工具栏按钮点击打开，并以模态对话框的形式展示，确保用户始终以主界面为中心进行任务执行控制，对话框操作完成后自动返回主界面。

## 界面布局设计

### 整体布局
- **布局方式**：桌面应用风格 + 顶部工具栏 + 左中右三栏布局
- **适配要求**：工控机鼠标键盘操作优化，支持快速操作
- **主要区域**：
  
  - 菜单栏：文件、控制、监控、工具、帮助等功能菜单
  - 工具栏：系统功能入口，包含产品管理、任务管理、数据上传、查询码、码替换、清屏等功能按钮
  - 左侧栏：当前生产信息、实时上传数据区
  - 中间栏：数据接收区（采集记录日志）、操作日志、报警信息
  - 右侧栏：单位实时统计、任务控制
  - 状态栏：系统状态、时间显示等信息



### 菜单栏设计
- **文件(F)**：
  - 退出软件
- **配置**：
  - 系统参数
- **数据**：
  - 产品管理
  - 任务管理
  - 操作日志
- **帮助(H)**：
  - 操作帮助
  - 关于系统

### 顶部工具栏设计

#### 功能分组
- **产品管理组**：
  - 产品管理：点击弹出产品管理对话框，支持产品信息的查看、编辑和管理
- **任务管理组**：
  - 任务管理：点击弹出任务管理对话框，支持生产任务的创建、编辑和状态管理
- **数据上传组**：
  - 数据上传：点击弹出数据上传对话框，支持生产数据的批量上传和同步
- **查询码组**：
  - 查询码：点击弹出码查询对话框，支持各级码的查询和追溯
- **码替换组**：
  - 码替换：点击弹出码替换对话框，支持问题码的替换和修正
- **清屏组**：
  - 清屏：点击调用clearScreen()函数，清空数据接收区、操作日志和报警信息

#### 对话框特性
- **弹出方式**：模态对话框形式，阻止背景操作直到对话框关闭
- **关闭方式**：通过对话框标题栏关闭按钮、ESC键或确认/取消按钮关闭
- **状态保持**：主界面状态在对话框打开期间保持，对话框关闭后恢复交互
- **数据同步**：对话框操作完成后，主界面相关数据自动刷新更新
- **界面适配**：对话框尺寸适配工控机屏幕，支持最小化和最大化
- **键盘支持**：支持Tab键切换焦点，Enter键确认，ESC键取消

- **区域样式**：白色背景，边框分割，单列垂直布局
- **信息字段**：

    - 筛选：支持单号模糊搜索、完工时间范围筛选
    - 过滤：仅显示"计划"和"生产中"状态订单
    - 联动：选择后自动更新所有相关字段
  - **产品编号**：只读显示，来源于选择的订单
  - **产品名称**：只读显示，来源于选择的订单
  - **产品规格**：只读显示，来源于选择的订单
  - **采集规格**：可编辑输入框，垛:箱二级比例配置
  - **计划数量**：只读显示，来源于选择的订单
  - **生产批次**：可编辑输入框
  - **类型**：下拉选择框（有箱码、无箱码）

#### 模块二：实时上传数据区
- **功能定位**：显示垛码数据的上传状态和进度
- **显示内容**：
  - **垛码信息**：P20241201001、P20241201002等格式
  - **数量信息**：1200箱等统计数据
  - **上传状态**：四种状态实时显示
- **状态标识**：
  - 上传中：蓝色背景，带动画效果
  - 上传成功：绿色背景
  - 待上传：灰色背景
  - 上传失败：红色背景
- **更新机制**：上传状态实时变化，最新记录在上方

### 中间栏设计（宽度：自适应，最小600px）

#### 模块一：数据接收区
- **功能定位**：实时显示码采集和垛码关联的操作记录
- **显示结构**：三列布局（时间 + 内容 + 状态）
  - **时间列**：精确到毫秒（2024-12-01 09:15:23.456）
  - **内容列**：采集操作详情（如"读取#箱码#成功&&&20241201001001"）
  - **状态列**：显示操作结果状态标识
- **状态类型**：
  - **读取码状态**：
    - 合格：码符合规范且有效，存入采集关系表
    - 无码：未读取到码或空码，存入采集关系表（标记无效）
    - 重码：系统中已存在的重复码，不存入数据库
    - 错码：格式错误或不符合规范，不存入数据库
    - 系统生成：通过"指定当前箱数"功能生成的虚拟箱码
  - **托盘关联状态**：
    - 成功：生成虚拟垛标，建立完整码关系，推送第三方系统
    - 失败：箱码数据正常生成虚拟垛标存入，托盘码字段为空
- **视觉标识**：
  - 合格/成功/系统生成：绿色背景
  - 无码：灰色背景
  - 重码/错码/失败：红色背景
- **数据管理**：实时更新，最新记录在上方，支持滚动查看

#### 模块二：操作日志
- **功能定位**：记录用户操作和系统响应的详细日志
- **显示格式**：HH:MM:SS 操作描述 - 结果状态
- **记录内容**：
  - 用户操作：按钮点击、参数设置、功能切换等
  - 系统响应：操作执行结果、状态变更、错误信息等
  - 设备通信：指令发送、状态反馈、通信异常等
- **显示特性**：滚动显示，最新记录在上方
- **初始状态**：显示"系统初始化完成 - 成功"

#### 模块三：报警信息
- **功能定位**：显示系统运行过程中的异常和报警信息
- **报警分类**：
  - **质量异常**：产品质量检测异常
  - **关联异常**：包装关联失败
  - **系统异常**：设备通信超时
- **级别标识**：
  - 警告级别：黄色背景显示
  - 严重级别：红色背景显示
- **显示格式**：时间戳 + 报警内容描述
- **状态管理**：初始空白，动态生成报警信息

### 右侧栏设计（宽度：336px）

#### 模块一：单位实时统计
- **功能定位**：实时显示当前生产任务的关键统计数据
- **布局结构**：
  - **第一行（双列）**：当前箱数 + 每垛箱数
  - **第二行（单列）**：已生产垛数
  - **第三行（单列）**：完成度百分比
- **数值展示**：
  - **当前箱数**：6（红色数值，80px大字体，突出显示）
  - **每垛箱数**：12（蓝色数值，80px大字体，突出显示）
  - **已生产垛数**：0（标准字体）
  - **完成度**：0%（百分比格式显示）
- **更新机制**：基于采集数据实时计算和更新

#### 模块二：任务控制
- **功能定位**：提供生产任务执行过程中的核心控制操作
- **布局设计**：2列网格排列，共9个按钮，按功能分类组织
- **视觉样式**：纯色背景、立体阴影效果、鼠标键盘操作优化
- **主题配色**：辅助控制按钮使用紫色主题区分

##### 主要控制功能
- **启用/停用切换**：
  - 功能：任务状态切换，支持启用和停用状态切换
  - 样式：纯色背景和阴影效果
  - 状态反馈：根据任务状态动态变化按钮文字和样式
  - 确认机制：启用时显示任务详细信息，停用时显示警告提示
- **订单结单**：
  - 功能：针对生产中的订单完成结单操作
  - 前置条件：订单状态为"生产中"且任务状态为"停用"
  - 操作流程：停用任务 → 点击结单 → 输入管理员密码 → 确认完成
  - 密码验证：使用系统预设固定密码，错误时显示提示
  - 状态变更：订单状态变为"已完成"，数据库更新production_task表
  - 按钮状态：根据条件动态启用/禁用，显示相应文字

##### 数据操作功能
- **添加码**：
  - 功能：针对当前垛添加系统生成的箱码
  - 生成规则：按系统配置的无码规则自动生成箱码
  - 操作逻辑：生成新箱码 → 添加到当前垛 → 更新统计数据
  - 界面影响：数据接收区显示"系统生成"状态记录（绿色）
  - 使用场景：生产线出现无码产品需要补充箱码时
- **清除码**：
  - 功能：清除上一个采集的箱码并减少计数
  - 操作逻辑：删除最后一个采集的箱码记录，当前箱码数自动减1
  - 界面影响：数据接收区移除对应记录，统计数据实时更新
  - 确认机制：操作前需要确认，防止误操作
  - 使用场景：误采集或需要撤销上一次采集操作时

##### 托盘操作功能
- **指定当前箱数**：
  - 功能：弹框输入目标箱数，系统自动调整箱码数量
  - 操作流程：点击按钮 → 弹出对话框 → 输入目标数 → 预览影响 → 确认执行
  - 业务逻辑：
    - 输入数 < 实际数：按时间顺序删除多余箱码，移除对应界面记录
    - 输入数 > 实际数：按无码规则生成虚拟箱码，显示"系统生成"记录
    - 输入数 = 实际数：无需操作，显示"无需调整"
  - 输入验证：1-999范围，仅允许正整数，必填验证
  - 界面影响：实时更新统计数据和数据接收区显示
- **强制满托盘**：
  - 功能：忽略采集规格要求，以当前箱数立即形成一托盘
  - 操作逻辑：改变采集规格要求，仅采集当前箱数就关联托盘码
  - 业务流程：点击按钮 → 确认对话框 → 忽略预设要求 → 生成虚拟垛标 → 重置计数
  - 界面影响：已生产托盘数+1，当前箱数重置为0，完成度重新计算
  - 操作记录：记录托盘码和箱数到系统日志
  - 使用场景：换班、设备维护、订单调整等需要提前结束托盘时
- **删除本托盘无码**：
  - 功能：删除当前托盘中的无码数据
  - 操作逻辑：删除灰色状态记录，当前箱数减去对应无码箱数
  - 数据库操作：将code_collection_record表中is_valid=FALSE的记录标记为删除
  - 界面影响：清除数据接收区无码记录，更新统计数据
  - 确认机制：显示"确定要删除当前托盘中的 X 个无码记录吗？此操作不可恢复"

##### 辅助控制功能
- **读码剔除**：
  - 功能：控制剔除设备启停，动态切换按钮文字
  - 开启操作：
    - 发送剔除设备工作指令，状态变为"运行中"
    - 状态栏显示"剔除功能：已开启"（蓝色，带发光效果）
    - 按钮文字变更为"关闭剔除"
  - 关闭操作：
    - 发送剔除设备停止指令，状态变为"已关闭"
    - 状态栏显示"剔除功能：已关闭"（红色背景）
    - 按钮文字变更为"开启剔除"
  - 指令发送：支持串口、网络、IO控制等多种通信方式
  - 状态同步：设备状态与界面显示实时同步，通信失败时显示错误
  - 操作记录：记录启动/停止指令发送结果到操作日志
  - 业务关联：与码校验、质量检测、生产统计联动
- **关闭报警**：
  - 功能：发送指令停止报警机制
  - 操作逻辑：向报警设备发送停止指令，清除当前报警状态和显示
  - 指令发送：支持多种设备通信协议（串口RS232/RS485、网络TCP/IP）
  - 状态更新：清空报警信息模块，更新状态栏报警指示
  - 操作记录：记录关闭报警指令发送结果
  - 特殊性：不受任务状态限制，随时可用，无需确认直接执行

## 数据采集功能

### 实时数据采集展示
- **采集频率**：每2秒采集1-2个产品
- **数据格式**：
  - 箱码格式：20241201001001等
  - 垛码格式：P20241201001等
- **时间精度**：精确到毫秒（2024-12-01 09:15:23.456）
- **质量检测**：98%合格率，2%不合格率
- **实时更新**：采集数据以日志条目形式实时显示

### 包装关联
- **关联类型**：二级关联采集模式（托盘码关联箱码）
- **关联成功率**：99%成功，1%失败
- **异常处理**：关联失败时自动报警
- **数据展示**：在数据接收区以日志形式显示关联结果

### 统计计算
- **箱数统计**：基于采集数据自动累计
- **托盘数统计**：按采集规格自动形成托盘
- **进度计算**：基于合格数量实时计算完成进度

- **对话框特性**：

  - 支持ESC键关闭对话框
- **筛选功能**：
  - 生产单号筛选：支持模糊匹配搜索

- **数据传递**：主界面与对话框之间通过参数传递和回调函数进行数据交互
- **状态保持**：对话框打开期间主界面状态冻结，关闭后恢复正常交互

### 主界面状态管理

- **数据刷新机制**：对话框关闭后自动刷新主界面相关数据
- **界面焦点管理**：对话框打开/关闭时的焦点管理和状态恢复

### 操作确认机制
- **启用确认**：显示任务详细信息的确认对话框

- **退出确认**：关闭应用时的确认对话框
- **辅助功能确认**：添加码、清除码、删除本托盘无码等操作需要确认
  - 删除本托盘无码确认：显示"确定要删除当前托盘中的 X 个无码记录吗？此操作不可恢复。"
  - 包含无码数量统计和操作不可恢复的警告信息

## 数据存储逻辑

### 码采集数据存储
- **合格状态**：存入code_collection_record表，is_valid=TRUE，validation_result记录"合格"
- **无码状态**：存入code_collection_record表，is_valid=FALSE，validation_result记录"无码"
- **重码状态**：不存入数据库，仅在界面显示，validation_result记录"重码"
- **错码状态**：不存入数据库，仅在界面显示，validation_result记录"错码"
- **系统生成状态**：存入code_collection_record表，is_valid=TRUE，validation_result记录"系统生成"

### 垛码关联数据存储
- **托盘关联成功**：
  - 生成虚拟垛标并存入采集关系表
  - 建立完整码关系，推送第三方系统
  - 托盘码、商品编号、本托盘数量等信息推送
- **托盘关联失败**：
  - 箱码数据正常生成虚拟垛标存入
  - 托盘码字段为空，记录失败原因到操作日志

### 数据关联关系
- **父子级关联**：通过parent_code_id字段建立码之间的层级关系
- **任务关联**：所有码记录通过task_id字段关联到具体的生产任务
- **时间追溯**：collection_time字段记录精确的采集时间，支持时序分析
- **状态标记**：is_valid和validation_result字段记录码的有效性和来源
