# 后端开发文档

## 1. 概述

本文档描述产线采集关联软件的后端实现，包括业务逻辑、数据库操作、接口设计等。后端采用Spring Boot框架，使用MyBatis-Plus进行数据库操作，遵循领域驱动设计（DDD）架构。

## 2. 技术架构

### 2.1 技术栈
- **框架**: Spring Boot
- **ORM**: MyBatis-Plus
- **数据库**: SQL Server
- **架构模式**: 领域驱动设计（DDD）
- **事务管理**: Spring @Transactional

### 2.2 项目结构
```
miduo-basebackend/
├── miduo-common/          # 公共模块（DTO、工具类）
├── miduo-entity/          # 实体模块（VO、Request、Response）
├── miduo-domain/          # 领域模块（Domain Model、Repository接口）
├── miduo-infrastructure/  # 基础设施模块（Repository实现、Mapper、PO）
├── miduo-application/     # 应用服务模块（Application Service）
├── miduo-controller/      # 控制器模块（REST API）
├── miduo-frontend/        # 前端模块（JavaFX UI）
└── miduo-bootstrap/       # 启动模块
```

## 3. 核心业务逻辑

### 3.1 任务管理

#### 3.1.1 添加任务
**业务逻辑**:
1. 先操作`ProductionOrder`表：
   - 设置`OrderStatus=1`（生产中）
   - 设置`ProductSourceType=2`（产线）
   - 设置`ProductionLine=001`（暂定）
   - 生成订单号
2. 再操作`ProductionOrderDetail`表：
   - 插入任务明细记录
   - 设置`IsDel=0`（未删除）
   - 设置`Type`（1-有箱码，2-无箱码）

**数据库操作**:
```java
// 1. 插入ProductionOrder
ProductionOrderPO orderPO = new ProductionOrderPO();
orderPO.setOrderStatus(1);
orderPO.setProductSourceType(2);
orderPO.setProductionLine(001);
// ... 其他字段
productionOrderMapper.insert(orderPO);

// 2. 插入ProductionOrderDetail
ProductionOrderDetailPO detailPO = new ProductionOrderDetailPO();
detailPO.setOrderNo(orderPO.getOrderNo());
detailPO.setIsDel(0);
// ... 其他字段
productionOrderDetailMapper.insert(detailPO);
```

#### 3.1.2 更新任务
**业务逻辑**:
- 更新`ProductionOrderDetail`表的字段
- 不修改`IsDel`字段（逻辑删除由删除接口处理）

**数据库操作**:
```java
ProductionOrderDetailPO detailPO = productionOrderDetailMapper.selectById(id);
// 更新字段
detailPO.setProductName(request.getProductName());
// ... 其他字段
productionOrderDetailMapper.updateById(detailPO);
```

#### 3.1.3 删除任务
**业务逻辑**:
- 逻辑删除：设置`ProductionOrderDetail.IsDel=1`
- 不删除`ProductionOrder`表记录

**数据库操作**:
```java
UpdateWrapper<ProductionOrderDetailPO> wrapper = new UpdateWrapper<>();
wrapper.eq("Id", id).set("IsDel", 1);
productionOrderDetailMapper.update(null, wrapper);
```

#### 3.1.4 查询任务
**业务逻辑**:
1. 从`ProductionOrderDetail`表查询任务明细（`IsDel=0`）
2. 关联查询`ProductionOrder`表获取订单状态
3. 支持模糊查询：订单号、批次号、产品名称
4. 支持状态筛选：`OrderStatus`（0-待生产，1-生产中，2-已完成，5-提前结单）
5. 完成数量：从`ProductionOrderDetail.OrderCount`字段获取

**数据库操作**:
```java
// 查询ProductionOrderDetail
QueryWrapper<ProductionOrderDetailPO> wrapper = new QueryWrapper<>();
wrapper.eq("IsDel", 0);
if (orderNo != null) wrapper.like("OrderNo", orderNo);
if (syBatchNo != null) wrapper.like("SyBatchNo", syBatchNo);
if (productName != null) wrapper.like("ProductName", productName);
Page<ProductionOrderDetailPO> page = detailMapper.selectPage(page, wrapper);

// 关联查询ProductionOrder获取状态
for (ProductionOrderDetailPO detail : page.getRecords()) {
    ProductionOrderPO order = orderMapper.selectOne(
        new QueryWrapper<ProductionOrderPO>()
            .eq("OrderNo", detail.getOrderNo())
    );
    // 设置订单状态等信息
}
```

#### 3.1.5 更新任务状态
**业务逻辑**:
1. 更新`ProductionOrder.OrderStatus`字段
2. 状态值：0-待生产，1-生产中，2-已完成，5-提前结单
3. 启用任务时（status=1）需要校验采集模式切换：
   - 查询`CodeRelationUpload`表最新记录
   - 检查`Type`是否一致
   - 如果不一致，检查`BigSerialNumber`是否有值
   - 如果`BigSerialNumber`为空，提示无法切换模式

**数据库操作**:
```java
// 更新ProductionOrder状态
UpdateWrapper<ProductionOrderPO> wrapper = new UpdateWrapper<>();
wrapper.eq("OrderNo", orderNo).set("OrderStatus", status);
productionOrderMapper.update(null, wrapper);

// 校验模式切换（启用任务时）
if (status == 1 && type != null) {
    // 查询CodeRelationUpload最新记录
    Page<CodeRelationUploadPO> page = new Page<>(1, 1);
    Page<CodeRelationUploadPO> resultPage = codeRelationUploadMapper.selectPage(page,
        new LambdaQueryWrapper<CodeRelationUploadPO>()
            .eq(CodeRelationUploadPO::getOrderNo, orderNo)
            .eq(CodeRelationUploadPO::getIsDel, 0)
            .orderByDesc(CodeRelationUploadPO::getAddTime)
    );
    
    if (!resultPage.getRecords().isEmpty()) {
        CodeRelationUploadPO lastRecord = resultPage.getRecords().get(0);
        if (!lastRecord.getType().equals(type) && 
            (lastRecord.getBigSerialNumber() == null || lastRecord.getBigSerialNumber().isEmpty())) {
            return ApiResult.error("上次采集模式仍有未进行关联的码，无法切换采集模式");
        }
    }
}
```

### 3.2 码采集与关联

#### 3.2.1 有箱码采集
**业务逻辑**:
1. 接收箱码（`SmallSerialNumber`）
2. 检查重复码：
   - 查询`CodeRelationUpload`表的`SmallSerialNumber`和`BigSerialNumber`字段
   - 如果存在重复，返回重复提示
3. 生成或获取`TagNo`（标签编号）：
   - 使用`ConcurrentHashMap`缓存当前订单的`TagNo`
   - 如果不存在，生成新的`TagNo`
4. 记录采集数量：
   - 使用`ConcurrentHashMap`缓存当前`TagNo`的采集数量
   - 每次采集后数量+1
5. 插入`CodeRelationUpload`表：
   - 设置`Type=1`（有箱码）
   - 设置`Status=0`（未完成）
   - 设置`IsUpload=1`（不同步）
   - 设置`IsDel=0`（未删除）
6. 满垛判断：
   - 当采集数量达到`Ratio`（每垛箱数）时，触发满垛逻辑
   - 将`TagNo`保存到待关联Map中（无箱码模式）

**数据库操作**:
```java
// 检查重复码
Long smallCodeCount = codeRelationMapper.selectCount(
    new LambdaQueryWrapper<CodeRelationPO>()
        .eq(CodeRelationPO::getSmallSerialNumber, boxCode)
        .eq(CodeRelationPO::getIsDel, 0)
);

// 插入码关系记录
CodeRelationPO entity = new CodeRelationPO();
entity.setSmallSerialNumber(boxCode);
entity.setOrderNo(orderNo);
entity.setTagNo(tagNo);
entity.setType(1); // 有箱码
entity.setStatus(0);
entity.setIsUpload(1);
entity.setIsDel(0);
codeRelationMapper.insert(entity);

// 更新采集计数
palletCountMap.put(tagNo, currentCount + 1);
```

#### 3.2.2 无箱码采集
**业务逻辑**:
1. 接收箱码采集设备数据（应为"null"）
2. 如果是无箱码模式且箱码不为"null"，提示错误并忽略
3. 系统生成唯一箱码：
   - 格式：订单号 + 时间戳 + 随机数
4. 其他逻辑与有箱码采集相同
5. 满垛后保存`TagNo`到`pendingNoBoxTagNoMap`，等待托盘码关联

**数据库操作**:
```java
// 生成唯一箱码
String uniqueBoxCode = generateUniqueBoxCode(orderNo);

// 插入码关系记录（Type=2）
CodeRelationPO entity = new CodeRelationPO();
entity.setSmallSerialNumber(uniqueBoxCode);
entity.setType(2); // 无箱码
// ... 其他字段
codeRelationMapper.insert(entity);
```

#### 3.2.3 有箱码托盘码关联
**业务逻辑**:
1. 接收托盘码（`BigSerialNumber`）和触发箱码（`SmallSerialNumber`）
2. 根据触发箱码查找`TagNo`：
   - 查询`CodeRelationUpload`表，找到对应`SmallSerialNumber`的记录
   - 获取该记录的`TagNo`
3. 验证箱数匹配：
   - 查询该`TagNo`下的实际采集箱数
   - 与前端传递的`Qty`（应采箱数）比较
   - 如果不匹配且未启用强制满垛，返回错误
   - 如果启用强制满垛（`forcePalletFlag`），忽略箱数差异
4. 查询该`TagNo`的最新`Qty`条记录（按`AddTime`降序）
5. 批量更新记录：
   - 设置`BigSerialNumber`（托盘码）
   - 生成虚拟垛标（`VirtualSerialNumber`）
   - 设置`IsVirtual=1`
   - 设置`IsUpload=0`（待上传）
   - **不更新`Status`字段**（保持原值）
6. 更新`Qty`字段：
   - 重新统计该`TagNo`下`BigSerialNumber`有值的数量
   - 更新所有记录的`Qty`字段
7. 更新统计字段：
   - 更新`ProductionOrder.OrderSumCount`（累加已生成虚拟垛标的数量）
   - 更新`ProductionOrderDetail.OrderCount`（累加已生成虚拟垛标的数量）
8. 清除缓存：
   - 清除`currentTagNoMap`中的`TagNo`（如果与当前采集的`TagNo`一致）
   - 清除`palletCountMap`中的计数
   - 清除`forcePalletFlagMap`中的强制满垛标记

**数据库操作**:
```java
// 1. 根据触发箱码查找TagNo
List<CodeRelationPO> triggerRecords = codeRelationMapper.selectList(
    new LambdaQueryWrapper<CodeRelationPO>()
        .eq(CodeRelationPO::getOrderNo, orderNo)
        .eq(CodeRelationPO::getSmallSerialNumber, triggerBoxCode)
        .eq(CodeRelationPO::getIsDel, 0)
        .orderByDesc(CodeRelationPO::getAddTime)
);
String tagNo = triggerRecords.get(0).getTagNo();

// 2. 验证箱数
Long actualBoxCount = codeRelationMapper.selectCount(
    new LambdaQueryWrapper<CodeRelationPO>()
        .eq(CodeRelationPO::getTagNo, tagNo)
        .eq(CodeRelationPO::getOrderNo, orderNo)
        .eq(CodeRelationPO::getIsDel, 0)
);

// 3. 查询最新Qty条记录
Page<CodeRelationPO> page = new Page<>(1, qty);
Page<CodeRelationPO> palletPage = codeRelationMapper.selectPage(page,
    new LambdaQueryWrapper<CodeRelationPO>()
        .eq(CodeRelationPO::getTagNo, tagNo)
        .eq(CodeRelationPO::getOrderNo, orderNo)
        .eq(CodeRelationPO::getIsDel, 0)
        .orderByDesc(CodeRelationPO::getAddTime)
);

// 4. 批量更新
LambdaUpdateWrapper<CodeRelationPO> updateWrapper = new LambdaUpdateWrapper<>();
updateWrapper.in(CodeRelationPO::getId, recordIds)
    .set(CodeRelationPO::getBigSerialNumber, palletCode)
    .set(CodeRelationPO::getVirtualSerialNumber, virtualPalletCode)
    .set(CodeRelationPO::getIsVirtual, 1)
    .set(CodeRelationPO::getIsUpload, 0);
    // 注意：不更新Status字段
codeRelationMapper.update(null, updateWrapper);

// 5. 更新ProductionOrder.OrderSumCount
ProductionOrderPO order = productionOrderMapper.selectOne(
    new LambdaQueryWrapper<ProductionOrderPO>()
        .eq(ProductionOrderPO::getOrderNo, orderNo)
);
order.setOrderSumCount(order.getOrderSumCount() + updateCount);
productionOrderMapper.updateById(order);

// 6. 更新ProductionOrderDetail.OrderCount
LambdaUpdateWrapper<ProductionOrderDetailPO> detailUpdateWrapper = new LambdaUpdateWrapper<>();
detailUpdateWrapper.eq(ProductionOrderDetailPO::getOrderNo, orderNo)
    .eq(ProductionOrderDetailPO::getProductNo, productNo)
    .eq(ProductionOrderDetailPO::getIsDel, 0)
    .set(ProductionOrderDetailPO::getOrderCount, 
         productionOrderDetailPO.getOrderCount() + updateCount);
productionOrderDetailMapper.update(null, detailUpdateWrapper);
```

#### 3.2.4 无箱码托盘码关联
**业务逻辑**:
1. 从`pendingNoBoxTagNoMap`获取待关联的`TagNo`
2. 验证箱数匹配（与有箱码逻辑相同）
3. 查询该`TagNo`下`BigSerialNumber`为空的最新`Qty`条记录
4. 批量更新记录（与有箱码逻辑相同）
5. 更新统计字段（与有箱码逻辑相同）
6. 清除缓存：
   - 清除`pendingNoBoxTagNoMap`中的`TagNo`
   - 清除`palletCountMap`中的计数
   - 清除`forcePalletFlagMap`中的强制满垛标记

**数据库操作**:
```java
// 查询BigSerialNumber为空的记录
Page<CodeRelationPO> palletPage = codeRelationMapper.selectPage(page,
    new LambdaQueryWrapper<CodeRelationPO>()
        .eq(CodeRelationPO::getOrderNo, orderNo)
        .eq(CodeRelationPO::getTagNo, tagNo)
        .eq(CodeRelationPO::getType, 2) // 无箱码
        .eq(CodeRelationPO::getIsDel, 0)
        .and(wrapper -> wrapper
            .isNull(CodeRelationPO::getBigSerialNumber)
            .or()
            .eq(CodeRelationPO::getBigSerialNumber, ""))
        .orderByDesc(CodeRelationPO::getAddTime)
);
```

### 3.3 数据上传

#### 3.3.1 查询上传数据
**业务逻辑**:
1. 查询`ProductionOrderDetail`表获取订单列表
2. 对每个订单：
   - 查询`CodeRelationUpload`表中`BigSerialNumber`有值的记录数（完成数量）
   - 查询`ProductionOrder`表获取`OrderStatus`（生产状态）
3. 返回订单列表和完成数量

**数据库操作**:
```java
// 查询订单列表
List<ProductionOrderDetailPO> orderList = productionOrderDetailMapper.selectList(
    new LambdaQueryWrapper<ProductionOrderDetailPO>()
        .eq(ProductionOrderDetailPO::getIsDel, 0)
);

// 查询完成数量
Long completedCount = codeRelationUploadMapper.selectCount(
    new LambdaQueryWrapper<CodeRelationUploadPO>()
        .eq(CodeRelationUploadPO::getOrderNo, orderNo)
        .eq(CodeRelationUploadPO::getIsDel, 0)
        .isNotNull(CodeRelationUploadPO::getBigSerialNumber)
        .ne(CodeRelationUploadPO::getBigSerialNumber, "")
);
```

#### 3.3.2 上传数据到WMS
**业务逻辑**:
1. 查询`CodeRelationUpload`表中`IsUpload=0`（待上传）的记录
2. 调用WMS接口上传数据
3. 上传成功后：
   - 设置`IsUpload=1`（已上传）
   - 设置`UploadTime`（上传时间）
4. 上传失败：
   - 增加`ErrCount`（失败次数）
   - 设置`Msg`（错误信息）

**数据库操作**:
```java
// 查询待上传记录
List<CodeRelationUploadPO> uploadList = codeRelationUploadMapper.selectList(
    new LambdaQueryWrapper<CodeRelationUploadPO>()
        .eq(CodeRelationUploadPO::getIsUpload, 0)
        .eq(CodeRelationUploadPO::getIsDel, 0)
);

// 上传成功后更新
LambdaUpdateWrapper<CodeRelationUploadPO> updateWrapper = new LambdaUpdateWrapper<>();
updateWrapper.eq(CodeRelationUploadPO::getId, id)
    .set(CodeRelationUploadPO::getIsUpload, 1)
    .set(CodeRelationUploadPO::getUploadTime, LocalDateTime.now());
codeRelationUploadMapper.update(null, updateWrapper);
```

### 3.4 码查询

#### 3.4.1 查询码关联关系
**业务逻辑**:
1. 根据输入的码（箱码、托盘码、虚拟垛标）查询`CodeRelationUpload`表
2. 如果输入的是箱码，查询`SmallSerialNumber`字段
3. 如果输入的是托盘码，查询`BigSerialNumber`字段
4. 如果输入的是虚拟垛标，查询`VirtualSerialNumber`字段
5. 返回关联的所有记录（同一`VirtualSerialNumber`下的所有记录）
6. 关联查询`ProductionOrderDetail`表获取产品信息

**数据库操作**:
```java
// 根据箱码查询
List<CodeRelationPO> records = codeRelationMapper.selectList(
    new LambdaQueryWrapper<CodeRelationPO>()
        .eq(CodeRelationPO::getSmallSerialNumber, code)
        .eq(CodeRelationPO::getIsDel, 0)
);

// 根据虚拟垛标查询所有关联记录
List<CodeRelationPO> allRecords = codeRelationMapper.selectList(
    new LambdaQueryWrapper<CodeRelationPO>()
        .eq(CodeRelationPO::getVirtualSerialNumber, virtualSerialNumber)
        .eq(CodeRelationPO::getIsDel, 0)
        .orderByAsc(CodeRelationPO::getAddTime)
);
```

## 4. 核心数据结构

### 4.1 内存缓存
```java
// 当前订单的TagNo缓存
private final ConcurrentHashMap<String, String> currentTagNoMap = new ConcurrentHashMap<>();

// 当前TagNo的采集数量缓存
private final ConcurrentHashMap<String, Integer> palletCountMap = new ConcurrentHashMap<>();

// 无箱码模式待关联的TagNo缓存
private final ConcurrentHashMap<String, String> pendingNoBoxTagNoMap = new ConcurrentHashMap<>();

// 强制满垛标记缓存
private final ConcurrentHashMap<String, Boolean> forcePalletFlagMap = new ConcurrentHashMap<>();
```

### 4.2 虚拟垛标生成规则
```java
private String generateVirtualPalletCode(String palletCode) {
    String dateStr = LocalDateTime.now().format(DateTimeFormatter.ofPattern("yyyyMMdd"));
    String randomNum = String.format("%03d", new Random().nextInt(1000));
    return "#" + palletCode + "#_" + dateStr + randomNum + "C";
}
```

## 5. 接口设计

- `POST /api/task/add` - 添加任务
- `PUT /api/task/update/{id}` - 更新任务

- `GET /api/data-upload/orders` - 查询上传订单列表

### 5.4 码查询接口
- `POST /api/code/query` - 查询码关联关系

## 6. 事务管理

### 6.1 事务注解
所有涉及数据库写操作的方法都使用`@Transactional(rollbackFor = Exception.class)`注解，确保数据一致性。

### 6.2 事务范围
- 任务添加、更新、删除
- 码采集和关联
- 数据上传

## 7. 异常处理

### 7.1 业务异常
- 使用`ApiResult`统一返回格式
- 成功：`ApiResult.success(data)`
- 失败：`ApiResult.error(message)`

### 7.2 异常捕获
- 所有业务方法使用try-catch捕获异常
- 记录异常日志
- 返回友好的错误信息

## 8. 数据库操作规范

### 8.1 查询规范
- 所有查询必须过滤`IsDel=0`的记录
- 使用MyBatis-Plus的`LambdaQueryWrapper`构建查询条件
- 分页查询使用`Page`对象

### 8.2 更新规范
- 逻辑删除使用`UpdateWrapper`设置`IsDel=1`
- 批量更新使用`LambdaUpdateWrapper`
- 更新操作需要记录操作日志

### 8.3 插入规范
- 设置默认值：`IsDel=0`，`Status=0`，`IsUpload=1`
- 设置时间字段：`AddTime=LocalDateTime.now()`
- 字符串字段如果为空，设置为空字符串（不是null）

## 9. 注意事项

### 9.1 并发控制
- 使用`ConcurrentHashMap`管理内存缓存
- 码采集和关联操作需要考虑并发安全

### 9.2 数据一致性
- 码关联后需要同步更新多个表的统计字段
- 使用事务确保数据一致性

### 9.3 性能优化
- 使用分页查询避免一次性加载大量数据
- 使用索引优化查询性能
- 合理使用缓存减少数据库查询

### 9.4 状态管理
- 任务状态：0-待生产，1-生产中，2-已完成，5-提前结单
- 码状态：0-未完成，1-已完成，2-已解除（码关联时不更新此字段）
- 上传状态：0-同步，1-不同步

## 10. 开发建议

### 10.1 代码规范
- 遵循Java编码规范
- 使用有意义的变量名和方法名
- 添加必要的注释

### 10.2 测试建议
- 单元测试覆盖核心业务逻辑
- 集成测试覆盖数据库操作
- 性能测试关注并发场景

### 10.3 日志记录
- 关键业务操作记录日志
- 使用不同日志级别（INFO、WARN、ERROR）
- 记录操作前后数据变化
