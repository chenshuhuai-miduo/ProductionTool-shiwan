# 前端开发文档（无代码版）

## 1. 概述

本文档描述产线采集关联软件的前端实现，包括UI组件、交互逻辑、数据流、设备通信等。前端采用JavaFX框架，遵循MVC架构模式，使用FXML进行界面布局。

## 2. 技术架构

### 2.1 技术栈
- **框架**: JavaFX
- **布局**: FXML
- **架构模式**: MVC（Model-View-Controller）
- **通信方式**: HTTP API（HttpUtil）
- **线程模型**: JavaFX Application Thread + 后台线程

### 2.2 项目结构
- **controller**: 控制器（Controller）
  - MainController - 主界面控制器
  - TaskManagementController - 任务管理控制器
  - SelectOrderDialogController - 选择订单对话框控制器
  - DataUploadController - 数据上传控制器
  - CodeQueryController - 码查询控制器
  - CodeReplaceController - 码替换控制器
  - OperateLogController - 操作日志控制器
  - SystemConfigController - 系统配置控制器
- **service**: 服务层
  - DeviceConnectionManager - 设备连接管理器
- **util**: 工具类
  - HttpUtil - HTTP请求工具
  - OperateLogBuilder - 操作日志构建器
- **fxml**: FXML布局文件

## 3. 主界面实现

### 3.1 界面布局

#### 3.1.1 整体布局
- **布局方式**: BorderPane + 顶部工具栏 + 左中右三栏布局
- **左侧栏**: 当前生产信息 + 实时上传数据区（宽度：340px）
- **中间栏**: 数据接收区 + 操作日志 + 报警信息（自适应宽度，最小600px）
- **右侧栏**: 单位实时统计 + 任务控制（宽度：280px）

#### 3.1.2 主要组件
- **左侧面板**:
  - 生产单号（TextField + 选择按钮）
  - 产品编号（Label）
  - 产品名称（Label）

  - 采集规格（TextField，只能输入正整数）
  - 计划数量（Label）
  - 生产批次（TextField）
  - 类型（ComboBox：有箱码/无箱码）
  - 实时上传数据区（VBox）

- **中间面板**:
  - 数据接收区（ListView，显示采集记录）
  - 操作日志（TextArea）
  - 报警信息（TextArea）

- **右侧面板**:
  - 当前箱数（Label）
  - 每垛箱数（Label）
  - 已生产垛数（Label）
  - 完成度（Label）
  - 启用/停用按钮（Button）
  - 结单按钮（Button）
  - 强制满垛按钮（Button）
  - 其他功能按钮

### 3.2 核心功能实现

#### 3.2.1 选择生产订单
**功能描述**:
- 点击"选择生产订单"按钮，弹出订单选择对话框
- 对话框显示任务列表，支持搜索和分页
- 选择订单后，自动填充主界面的生产信息
- 自动过滤已完成状态的任务（OrderStatus=2）

**实现逻辑**:
1. 检查当前任务是否处于启用状态，如果是则提示用户先停用
2. 加载选择订单对话框FXML文件
3. 创建模态对话框Stage并显示
4. 等待用户选择订单
5. 如果用户确认选择，获取选中的任务信息
6. 调用`fillTaskInfo`方法填充主界面信息

**API调用**:
- 使用`POST /api/task/page/selectable`接口查询可选择的任务（自动排除已完成状态）

#### 3.2.2 启用/停用任务
**功能描述**:
- 点击"启用任务"按钮，启用当前任务
- 启用时传递当前选择的采集类型（有箱码/无箱码）
- 后端会校验采集模式切换
- 启用成功后，更新按钮文本为"停用任务"

**实现逻辑**:
1. 检查是否已选择生产订单
2. 根据当前任务状态确定新状态（启用/停用）
3. 获取当前选择的采集类型（有箱码=1，无箱码=2）
4. 构建API URL，启用任务时传递type参数
5. 调用更新任务状态API
6. 根据返回结果更新界面状态

**API调用**:
- 使用`PUT /api/task/updateStatus/{id}/{status}?type={type}`接口更新任务状态

#### 3.2.3 箱码采集
**功能描述**:
- 接收箱码采集设备的数据
- 根据采集类型（有箱码/无箱码）处理数据
- 无箱码模式时，箱码必须为"null"，否则提示错误
- 有箱码模式时，检查重复码
- 采集成功后，更新当前箱数统计

**实现逻辑**:
1. 检查是否已选择生产订单
2. 获取当前采集类型
3. 无箱码模式校验：如果箱码不为"null"，记录警告并忽略
4. 构建采集请求对象
5. 调用箱码采集API
6. 根据返回结果更新界面显示

**API调用**:
- 使用`POST /api/code/collect`接口进行箱码采集

#### 3.2.4 托盘码关联
**功能描述**:
- 接收托盘码关联设备的数据（托盘码）
- 使用之前保存的触发箱码进行关联
- 有箱码模式：调用托盘码关联接口
- 无箱码模式：调用无箱码托盘码关联接口
- 关联成功后，更新统计信息和实时上传数据区

**实现逻辑**:
1. 检查是否已选择生产订单
2. 获取当前采集类型
3. 根据采集类型选择不同的API：
   - 有箱码模式：调用`/api/code/associate-pallet`接口
   - 无箱码模式：调用`/api/code/collect-no-box`接口
4. 传递托盘码、订单号、触发箱码（有箱码模式）、每垛箱数等参数
5. 根据返回结果更新界面显示

**API调用**:
- 有箱码模式：`POST /api/code/associate-pallet`
- 无箱码模式：`POST /api/code/collect-no-box`

#### 3.2.5 强制满垛
**功能描述**:
- 点击"强制满垛"按钮，弹出确认对话框
- 确认后，设置强制满垛标记
- 托盘码关联时，忽略箱数差异检查
- 关联成功后，自动清除强制满垛标记

**实现逻辑**:
1. 检查是否已选择生产订单
2. 弹出确认对话框
3. 用户确认后，调用强制满垛API
4. 设置成功后，记录操作日志

**API调用**:
- 使用`POST /api/code/force-pallet?orderNo={orderNo}`接口设置强制满垛

#### 3.2.6 实时统计更新
**功能描述**:
- 定时查询当前箱数、已生产垛数、完成度
- 当前箱数：从内存缓存和数据库查询（无箱码模式需要查询CodeRelationUpload表）
- 已生产垛数：查询CodeRelationUpload表中BigSerialNumber有值的不同TagNo数量
- 完成度：已采集箱数 / 计划数量 * 100%

**实现逻辑**:
1. 在后台线程中调用统计API
2. 获取已生产垛数并更新界面显示
3. 获取已采集箱数并计算完成度
4. 使用`Platform.runLater`切换到JavaFX线程更新UI

**API调用**:
- 使用`GET /api/code/produced-pallet-count/{orderNo}`获取已生产垛数
- 使用`GET /api/code/collected-count/{orderNo}`获取已采集箱数

#### 3.2.7 实时上传数据区
**功能描述**:
- 显示垛码数据的上传状态和进度
- 四种状态：上传中（蓝色）、上传成功（绿色）、待上传（灰色）、上传失败（红色）
- 实时更新上传状态
- 最新记录在上方显示

**实现逻辑**:
1. 在后台线程中调用实时上传数据API
2. 获取当前订单的上传数据列表
3. 根据上传状态设置不同的背景颜色
4. 使用`Platform.runLater`更新界面显示
5. 最新记录添加到列表顶部

**API调用**:
- 使用`GET /api/upload/realtime/{orderNo}`获取实时上传数据

### 3.3 数据接收区

#### 3.3.1 显示采集记录
**功能描述**:
- 实时显示码采集和垛码关联的操作记录
- 使用不同颜色显示不同状态：合格（绿色）、重码（红色）、错码（橙色）、无码（灰色）
- 显示时间、设备名称、码值、状态

**实现逻辑**:
1. 接收采集结果数据
2. 根据状态类型设置不同的文字颜色
3. 构建TextFlow对象显示带颜色的文本
4. 添加到ListView顶部（最新记录在上方）
5. 限制列表最大记录数（如1000条）

### 3.4 操作日志

#### 3.4.1 记录操作日志
**功能描述**:
- 记录所有关键操作：选择订单、启用任务、箱码采集、托盘码关联等
- 显示操作时间、操作内容、操作结果
- 自动滚动到最新日志

**实现逻辑**:
1. 在操作执行时记录日志信息
2. 使用`Platform.runLater`更新操作日志区域
3. 自动滚动到底部显示最新日志
4. 同时调用后端API记录操作日志到数据库

### 3.5 报警信息

#### 3.5.1 显示报警信息
**功能描述**:
- 显示系统报警信息：重码、错码、设备异常等
- 报警信息以红色显示
- 支持声光报警（如果配置了报警器）

**实现逻辑**:
1. 在异常情况发生时记录报警信息
2. 使用`Platform.runLater`更新报警信息区域
3. 设置文字颜色为红色
4. 自动滚动到底部
5. 如果启用报警器，触发声光报警

## 6. 数据流设计

### 6.1 数据流向
1. 设备数据 → DeviceConnectionManager → MainController → HTTP API → 后端服务 → 数据库
2. 返回结果 → MainController → UI更新

### 6.2 数据更新机制
- **实时更新**: 设备数据实时接收和处理
- **定时更新**: 统计信息定时从后端查询
- **事件驱动**: 操作完成后触发数据刷新

## 7. 线程模型

### 7.1 JavaFX Application Thread
- 所有UI更新必须在JavaFX Application Thread上执行
- 使用`Platform.runLater()`切换到JavaFX线程
- 避免在UI线程上执行耗时操作

### 7.2 后台线程
- HTTP请求在后台线程执行
- 设备数据接收在后台线程处理
- 数据处理完成后，使用`Platform.runLater()`更新UI

### 7.3 线程安全
- 所有UI更新操作都使用`Platform.runLater()`包装
- 避免在后台线程直接访问UI组件
- 使用线程安全的数据结构
