# P02-08 - 数据上传

## 子功能名称
数据上传 - 数据上传管理和状态查询

## 功能概述
为生产管理员提供数据上传管理功能界面，支持手动上传、上传状态查询和状态管理。界面采用桌面应用风格设计，与系统整体风格保持一致。界面提供数据上传管理功能，支持手动触发上传、查询指定垛码的上传状态、重置垛码状态为"未上传"或标记为"已上传"。

**重要说明**：当系统设置→业务→上传配置中「自动上传开关」开启时，成垛后系统自动上传数据至米多大数据引擎（默认开启）；关闭时需手动上传。本页面主要用于手动触发上传、查询上传状态、处理异常数据（重置状态、标记状态）。

## 界面布局设计

### 整体布局
- **布局方式**：作为Tab内容区直接显示，左右两栏布局
- **适配要求**：全宽布局，充分利用屏幕空间
- **主要区域**：
  - 左侧栏：手动上传区（上传按钮 + 上传信息展示区）
  - 右侧栏：操作区（查询表单 + 查询结果展示区）
  - 底部状态栏：遵循 通用规则文档「02-用户界面设计」之状态栏设计
- **布局比例**：
  - 左侧手动上传区：占40%宽度（最小宽度400px）
  - 右侧操作区：占60%宽度（自适应剩余空间）

**布局说明**：
- 页面采用左右两栏布局
- 左侧手动上传区占40%宽度，右侧操作区占60%宽度
- 左侧区域：上部手动上传按钮，下部上传信息展示区（接收**数据采集页面满垛自动上传**与**本页手动上传**的日志，展示垛码、箱数及上传状态，**最新日志在最上方**）
- 右侧区域：上部查询表单（垛码输入+三个按钮），下部查询结果展示区
- 按钮高度统一为40px，字体16px，确保工控机环境下的可操作性

### 左侧区域设计：手动上传区

#### 1. 手动上传按钮
- **位置**：左侧区域顶部
- **功能**：手动触发上传所有未上传的垛数据
- **样式**：主要操作按钮，蓝色背景，白色加粗文字，高度40px，宽度120px
- **帮助提示**：按钮后面显示问号图标 `(?)`，点击(?)图标显示帮助信息
  - 提示内容："批量上传所有未上传的垛数据"
- **操作流程**：
  1. 点击"手动上传"按钮
  2. 系统查找所有未上传的垛数据
  3. 执行上传操作
  4. 在上传信息展示区实时显示上传进度和结果

#### 2. 上传信息展示区
- **位置**：左侧区域，手动上传按钮下方
- **样式**：白色背景卡片，内部竖向布局，支持垂直滚动；**单条日志文字过长时自动换行**，不截断
- **功能说明**：接收并显示数据采集页面满垛自动上传、本页手动上传产生的日志信息（同一展示区、同一格式）
- **显示内容**：
  - 以日志形式显示，**最新在上**；单行过长时换行展示。
  - **行格式**：`HH:mm:ss` + 空格 + `垛码` + 垛码值 + `，箱数` + 箱数 + `，` + 状态文案；上传失败时原因紧跟其后，用中文括号包裹，如 `上传失败（网络超时）`。
  - **示例**：
    - 开始上传：`10:30:15 垛码 P20241201001，箱数 70，开始上传`（灰色）
    - 上传中：`10:30:16 垛码 P20241201001，箱数 70，上传中…`（蓝色）
    - 上传成功：`10:30:18 垛码 P20241201001，箱数 70，上传成功`（绿色）
    - 上传失败：`10:30:20 垛码 P20241201001，箱数 70，上传失败（网络超时）`（红色）
  - **上传失败原因**：**具体到箱/盒/瓶层级**，由服务端接口返回，示例：
    - 网络超时、连接失败等：显示「网络超时」「连接服务器失败」
    - 业务校验失败：直接告知码值，如「箱码 xxx 盒码 xxx 瓶码 xxx 不在码包范围内」「箱码 xxx 盒码 xxx 重复」「箱码 xxx 盒数不符」等
  - **颜色标识**：
    - 开始上传：灰色文字
    - 上传中：蓝色文字
    - 上传成功：绿色文字
    - 上传失败：红色文字
  - **显示数量**：显示所有上传日志信息，不限制数量
- **数据更新**：
  - 数据采集页面满垛自动上传、或本页点击「手动上传」触发上传时，均将新日志追加到**顶部**（最新的在最上方）
  - 每个垛的上传过程依次产生 3 条日志：开始上传 → 上传中 → 上传成功/失败
  - 支持垂直滚动查看所有历史日志（向下滚动查看更早的日志）
  - 日志按时间倒序显示（时间越晚越靠上）

### 右侧区域设计：垛码查询与状态管理

#### 1. 查询表单区
- **位置**：右侧区域顶部
- **布局方式**：竖向布局
- **组成元素**：
  - **垛码输入框**：
    - 功能：输入垛码进行状态查询和管理
    - 样式：输入框高度40px，文字18px，全宽
    - 占位符："请输入垛码"
    - 支持扫码枪输入
  - **操作按钮组**（横向排列，三个按钮+一个帮助图标）：
    - **查询状态按钮**：
      - 功能：查询指定垛码的上传状态
      - 样式：标准按钮，高度40px，宽度100px
      - 操作：点击后在查询结果展示区显示该垛码的详细信息
    - **设为未上传按钮**：
      - 功能：将指定垛码状态重置为"未上传"（用于异常数据修复后重新补传）
      - 样式：标准按钮，高度40px，宽度110px
      - 操作：点击后弹出确认对话框 → 确认后重置状态 → 显示操作结果
    - **设为已上传按钮**：
      - 功能：将指定垛码标记为"已上传"（用于已人工确认无误，避免系统重复上传）
      - 样式：标准按钮，高度40px，宽度110px
      - 操作：点击后弹出确认对话框 → 确认后标记状态 → 显示操作结果
    - **帮助图标**：
      - 位置：在最后一个按钮（设为已上传）后面，间距4-6px
      - 样式：问号图标 `(?)`
      - 功能：点击(?)图标显示三个按钮的帮助信息
      - 帮助内容：
        - 查询状态：查询指定垛码的上传状态详情
        - 设为未上传：将垛码状态重置为未上传，用于数据修复后重新补传
        - 设为已上传：手动标记为已上传，避免系统重复上传此垛

#### 2. 查询结果展示区
- **位置**：右侧区域，查询表单下方
- **样式**：白色背景卡片，带边框
- **显示内容**：
  - **空状态**：未查询时显示提示"请输入垛码并点击查询状态"
  - **查询结果**：查询后分行展示垛码、状态、上传时间、箱数、失败原因（成功时不显示失败原因）；标签与值可对齐留白。
    - **垛码**：20px 字体，垛码值可用蓝色
    - **状态**：上传状态标签（成功/失败/未上传）
      - 成功：绿色标签
      - 失败：红色标签
      - 未上传：灰色标签
    - **上传时间**：`YYYY-MM-DD HH:mm:ss`（已上传时显示）
    - **箱数**：该垛箱数
    - **失败原因**：上传失败时显示**具体原因**（具体到箱/盒/瓶），成功时不显示
  - **未找到提示**：查询的垛码不存在时显示"未找到该垛码的上传记录"

## 交互功能设计

### 自动上传机制
- **配置依赖**：根据系统设置→业务→上传配置中「自动上传开关」执行；默认开启。
- **触发时机**：开关开启时，成垛后系统自动上传数据至米多大数据引擎；关闭时成垛后不自动上传，需手动上传。
- **上传内容**：
  - 垛码信息
  - 箱码关联关系
  - 盒码关联关系
  - 瓶码关联关系（通过1号机数据）
  - 任务信息
- **上传结果**：
  - 自动记录到数据库上传记录表中
  - 同时将上传信息推送到数据上传页面的上传信息展示区显示

### 手动上传流程
1. **点击手动上传**：
   - 点击"手动上传"按钮
   - 系统自动查找所有未上传的垛数据
2. **执行上传**：
   - 批量执行上传操作
   - 每上传一个垛，将上传信息推送到上传信息展示区
3. **上传结果**：
   - 在上传信息展示区按**时间倒序**显示（**最新在上**），与自动上传日志一致
   - 上传完成后显示提示（例如："手动上传完成，成功3个，失败1个"）

### 垛码状态查询流程
1. **输入垛码**：
   - 在垛码输入框中输入垛码（支持手动输入或扫码枪输入）
2. **点击查询状态**：
   - 点击"查询状态"按钮
3. **显示查询结果**：
   - 在查询结果展示区显示该垛码的详细信息：
     - 垛码
     - 上传状态（成功/失败/未上传）
     - 上传时间（已上传时显示）
     - 箱数
     - 失败原因（上传失败时显示）
   - 如果该垛码不存在，显示提示："未找到该垛码的上传记录"

### 重置状态流程（设为未上传）
1. **输入垛码**：
   - 在垛码输入框中输入垛码
2. **点击设为未上传**：
   - 点击"设为未上传"按钮
3. **确认操作**：
   - 弹出确认对话框："确认将垛码 [XXX] 的状态重置为未上传？"
   - 说明：用于异常数据修复后重新补传
4. **执行重置**：
   - 确认后重置垛码状态为"未上传"
   - 显示操作结果提示（例如："垛码 [XXX] 已重置为未上传状态"）
   - 如果当前查询结果展示区显示的是该垛码，自动刷新显示

### 标记状态流程（设为已上传）
1. **输入垛码**：
   - 在垛码输入框中输入垛码
2. **点击设为已上传**：
   - 点击"设为已上传"按钮
3. **确认操作**：
   - 弹出确认对话框："确认将垛码 [XXX] 标记为已上传？"
   - 说明：用于已人工确认无误，避免系统重复上传
4. **执行标记**：
   - 确认后标记垛码状态为"已上传"
   - 显示操作结果提示（例如："垛码 [XXX] 已标记为已上传状态"）
   - 如果当前查询结果展示区显示的是该垛码，自动刷新显示

## 页面数据展示逻辑

### 上传信息展示区数据来源
- **数据来源**：
  - 接收数据采集页面（主界面-盒箱垛关联）满垛后自动上传的日志信息
  - 接收本页面手动上传按钮触发的上传日志信息
- **显示逻辑**：
  - 以日志形式显示，格式同「上传信息展示区」（含过长换行）；最新的信息在最上方
  - 每个垛的上传过程会产生多条日志：
    - 第1条：开始上传（灰色文字）
    - 第2条：上传中（蓝色文字）
    - 第3条：上传成功/失败（绿色/红色文字）
  - 显示所有上传日志信息，不限制数量
  - 支持垂直滚动查看所有历史日志
  - 日志按时间倒序显示（最新的在最上方）

### 查询结果展示区数据来源
- **数据来源**：根据输入的垛码查询上传记录表
- **查询内容**：
  - 垛码信息
  - 上传状态（成功/失败/未上传）
  - 上传时间（已上传时显示）
  - 箱数（该垛包含的箱数）
  - 失败原因（上传失败时记录）
- **显示逻辑**：
  - 未查询时显示空状态提示
  - 查询后显示该垛码的详细信息
  - 执行状态管理操作后，自动刷新当前显示的垛码信息

## 服务端接口要求（上传失败）

为支持上传失败时显示**具体到箱/盒/瓶**的报警原因，服务端上传接口需调整返回结构：

- **成功时**：返回成功标识即可
- **失败时**：返回失败原因列表，每条包含：
  - 层级（垛/箱/盒/瓶）
  - 问题码（如有）
  - 具体原因描述（如「不在码包范围内」「重复」「格式错误」等）

示例失败返回结构（供接口设计参考）：
```json
{
  "success": false,
  "message": "数据校验失败",
  "details": [
    {"level": "箱", "code": "C20241201003", "reason": "箱码不在大标码包范围内"},
    {"level": "盒", "code": "B202412010302", "reason": "盒码重复"},
    {"level": "瓶", "code": "V202412010503", "reason": "瓶码不在小标码包范围内"}
  ]
}
```

客户端将 `details` 拼接为可读文案展示在报警信息区、上传信息展示区及查询结果中。

- **上传信息刷新响应时间**：≤ 1秒

- **输入设备**：完全支持鼠标键盘操作和扫码枪输入

- **分辨率支持**：1024x768 ~ 1920x1080

## 功能边界

### 包含的功能
1. 手动上传功能（批量上传所有未上传的垛）
2. 上传信息展示功能（接收并显示自动上传和手动上传的日志信息）
3. 垛码状态查询功能（输入垛码查询上传状态）
4. 状态重置功能（设为未上传）
5. 状态标记功能（设为已上传）

### 适用场景
- **手动上传管理**：手动触发批量上传未上传的垛
- **状态查询**：快速查询指定垛码的上传状态
- **异常处理**：处理上传异常的垛（重置状态重新上传）
- **状态纠正**：手动标记垛状态（避免重复上传）
