# P02-06 - 取消关联

## 子功能名称

取消关联 - 单码取消和多码取消

## 功能概述

取消码之间的关联关系，使追溯数据与实物状态一致。支持单码取消、多码取消两种模式，均需密码确认。

**适用对象**：本功能仅针对**已写入关联表**的码关联数据。生产过程中门禁不通过被剔除的品（不计数、不建立关联），其数据从未进入关联表，无需也无法通过本功能操作。

**与码替换的区别**：


| 功能       | 适用场景              | 关联关系       | 码去向           |
| -------- | ----------------- | ---------- | ------------- |
| **码替换**  | 实物未变，仅码损坏/错误      | 保留，用新码替换旧码 | 原码不回收         |
| **取消关联** | 实物已剔除/返工/重包，或关联错误 | 取消，从追溯链中移除 | 码回迁至热表，可参与码替换 |


**业务场景**：已关联但实物已剔除/返工/重包、装错箱盒修正、召回退货、数据纠错等。

**重要说明**：

- 单码取消/多码取消通过选项卡切换；取消均需弹窗二次确认+密码123456。
- 取消顺序：垛→箱→盒→瓶，须从上级起逐级取消；**取消某级时连带取消其下全部**，故不能跳过上级直接取消下级（有上级时系统禁用确认并提示先取消上级）。
- 已上传云端：先取消云端再取消本地，保证一致。
- **冷热分离**：取消关联后，被取消的码需**回迁**（冷表→热表），方可参与后续码替换。详见 AI 文档 10.1.1。

**取消逻辑（取消粒度）**：

- **单码模式**：扫下级码（瓶/盒）→ **子查父**找上级 → 取消**上级单元**及其下全部。扫瓶码→取消该盒（含盒内所有瓶）；扫盒码→取消该箱（含箱内所有盒、瓶）。
- **多码模式**：输入某层级码（盒/箱/垛）→ 取消**该包装单元**及其下全部。扫盒码→取消该盒；扫箱码→取消整箱（含箱内盒、瓶）；扫垛码→取消整垛（含垛内箱、盒、瓶）。

**设计说明**：

**两种模式的区别**：

| 维度 | 单码取消 | 多码取消 |
| -- | -- | -- |
| **输入** | 1个码，**瓶/盒**（不支持箱码、垛码） | 多个码，**盒/箱/垛** |
| **逻辑** | **子查父**：扫下级码找上级，取消上级单元及其下全部 | **直接指定**：输入要取消的单元码，逐个取消 |
| **取消对象** | 扫瓶码→取消该盒（含盒内所有瓶）；扫盒码→取消该箱（含箱内所有盒、瓶） | 盒码→取消盒；箱码→取消箱；垛码→取消垛 |
| **适用场景** | 手上有实物，扫下级码（如扫盒确认箱）快速取消 | 已知要取消的盒/箱/垛，批量添加后一次性取消；**整箱、整垛取消**仅多码模式支持 |

**说明**：单码模式**不支持箱码、垛码**。扫箱码会取消整垛（约70箱），属危险操作，不予支持；整箱、整垛取消需用多码模式。垛码为虚拟垛标，可从数据查询、生产统计等界面复制。多码模式执行时按垛→箱→盒顺序依次取消，若盒在箱内，取消箱时已连带取消盒，自动跳过。

### 模式切换与帮助设计

- **选项卡**：切换；当前选中高亮（蓝色背景）
- **帮助 (?)**：紧贴选项卡右侧，点击弹出整合说明；**不随选项卡切换变化**，两种模式共用同一帮助内容

**帮助内容（整合说明）**：

- **单码模式**：扫1个下级码（瓶/盒），**子查父**找上级，取消上级单元及其下全部；不支持箱码/垛码（扫箱会取消整垛，危险）。有上级时需先逐级取消。
- **批量模式**：输入盒/箱/垛码加入列表（垛码为虚拟垛标，可从数据查询等界面复制），可添加多个、支持混合层级；**直接指定**要取消的单元，按垛→箱→盒顺序执行；若盒在箱内，取消箱时已连带取消盒。
- **通用**：取消顺序垛→箱→盒→瓶；已上传须先取消云端再取消本地；操作不可恢复，需密码123456确认。

## 单个取消关联模式（单码取消）

### 操作表单设计（左侧区域）

- **表单**：左 600px，白底卡片；标题「单码取消」24px
- **码输入**：*码；必填；支持手动/扫码；占位：「请输入或扫描瓶码/盒码」
- **识别按钮**：蓝底，40×100px；触发识别，结果在右侧
- **警告**：操作不可恢复，请确认后执行（红字，按钮上方）
- **清空**：清空输入，不清空右侧识别结果
- **成功后清空**：自动清空码输入框
- **确认取消关联**：红底，40×150px；有上级关联时禁用；点击弹出密码确认

### 识别结果展示设计（右侧区域）

- **区域**：右 700px，白底；标题「识别结果」24px
- **格式**：可取消 `盒码 xxx，可取消关联，取消N个`；不可取消 `箱码 xxx，不可取消（原因）链路: xxx→xxx`。N 为底层瓶数，由系统配置计算。
- **原则**：首要显示能否取消，原因放括号内；长文字换行
- **显示**：浅灰底，min 500px，可滚动；初始提示「请在左侧输入码值并点击识别」

### 取消结果展示设计（右侧区域下方）

- **区域**：识别结果下方，高约200px，可滚动
- **成功**：绿字「取消关联成功！」+ 包装信息、影响范围、云端状态、时间
- **失败**：红字「取消关联失败！」+ 失败原因、时间
- **格式**：`10:25:33 盒码xxx 取消N个`；初始「暂无取消关联记录」

### 单码取消确认对话框

- 标题：取消关联确认；密码：输入框；确认信息含包装信息+警告「此操作不可恢复」
- 按钮：取消（灰）、确认（红）

### 操作流程

1. 扫/输入1个码（瓶/盒，不支持箱码/垛码）→ 验证格式、存在性
2. 点击识别 → 子查父找上级 → 显示识别结果（上级单元、取消范围）
3. 若有上级关联：先逐级取消（垛→箱→盒→瓶）再回本单元
4. 若无上级：点击确认取消关联 → 弹窗输密码 → 云端检查 → 取消云端（如有）→ 取消本地 → 记录日志
5. 成功后自动清空码输入框

## 多码取消模式

### 操作表单设计（左侧区域）

- **盒/箱/垛码输入**：*盒/箱/垛码；支持手动输入/扫码；占位「盒码、箱码或垛码（垛码为虚拟，可从数据查询等界面复制）」；添加按钮将码加入列表
- **添加按钮**：蓝底，将码加入列表
- **已添加的码列表**：max 200px 可滚动；格式 `序号. 码类型 码值 [删除]`；空时「暂无添加」
- **混合层级**：列表可含盒码、箱码、垛码；执行时按垛→箱→盒顺序依次取消。若盒码在箱码内，取消箱时已连带取消该盒，执行盒级时自动跳过已无关联的码。
- **识别按钮**：触发批量识别
- **清空**：清空输入和列表，不清空右侧；**成功后清空**：自动清空输入和列表
- **确认取消关联**：红底，40×180px，点击弹出密码确认

### 识别结果展示设计（批量模式）

- 格式同单个模式，每单元一行；底部汇总**总影响范围**
- **影响范围统计**：仅统计可取消当前级的单元；**码关系=底层瓶数**（取消某级时连带取消其下全部，按瓶数统计）。按石湾配置（1垛=70箱、1箱=4盒、1盒=6瓶）：取消1盒→6个，取消1箱→24个（4盒×6瓶），取消1垛→1680个（70×4×6）；示例「2个可取消，涉及1686个码关系」（如1盒+1垛=6+1680=1686）
- 初始提示「请在左侧添加盒/箱/垛码并点击识别」

### 取消结果展示设计（批量模式）

- 与单个共用取消结果区域；逐条展示每个可取消码的结果
- 部分失败：显示「部分取消关联失败」+ 按单元列示失败原因

### 密码确认对话框

- 标题：取消关联确认；密码：输入框；显示单元数、码关系数；警告「此操作不可恢复」

### 操作流程

1. 输入盒码/箱码/垛码 → 添加至列表（可多个，可删除）；垛码为虚拟垛标，需从数据查询、生产统计等界面复制
2. 点击识别 → 显示每个单元的上级状态、链路、总影响范围
3. 若有上级：先逐级取消再回该单元
4. 确认取消关联 → 弹窗输密码 → 云端检查 → 取消云端（如有）→ **按垛→箱→盒顺序依次取消**（含包含关系时自动去重）→ 取消本地 → 记录日志
5. 成功后自动清空输入和列表

## 交互功能设计

### 验证规则

- **码值**：必填、格式正确、**必须在关联表中存在**（本功能不处理从未入关联表的数据）
- **单码模式**：仅支持瓶/盒，**不支持箱码、垛码**（扫箱会取消整垛，危险）
- **关联关系**：码间关联、包装单元、影响范围

### 取消关联规则

- **二次确认与密码**：均需弹窗+密码123456
- **取消顺序**：垛→箱→盒→瓶，须从上级起逐级取消；取消某级时连带取消其下全部，故不能跳过上级直接取消下级；每级前判断上级已取消
- **识别展示**：显示上级状态、链路；有上级时禁用确认按钮；影响范围仅统计可取消单元
- **批量混合层级**：列表可含盒/箱/垛；**执行时按垛→箱→盒顺序依次取消**；若列表中存在包含关系（如盒码在箱码内），**先取消上级会连带取消下级**，执行下级时自动跳过已无关联的码，避免重复操作
- **云端**：已上传则先取消云端再取消本地
- **成功后清空**：单码模式清空码输入；多码模式清空输入和列表；右侧识别/结果不清空

## 页面数据展示逻辑

- 云端检查 → 取消云端（如有）→ 取消本地 → **码回迁**（冷表→热表）→ 记录日志 → 显示结果

## 功能边界

### 包含

单码取消、多码取消、模式切换、密码保护、操作日志、统计信息

### 适用场景（均针对已写入关联表的数据）

- **已关联但实物已剔除/返工/重包**  
数据已进入关联表（采集时门禁通过），但实物后来被移除或需重做，追溯链需同步取消。典型情形：
  - 成垛后发现缺陷：箱已入垛，后续质检发现该箱有问题，实物从成品中剔除
  - 出货前/入库后发现异常：生产时通过校验，出货或入库环节发现实物需剔除
  - 返工/重包：外包装损坏需换箱、换码重贴，实物已从原包装取出，原关联失效
- **装错/贴错**：装错箱盒、贴错码的修正，需取消错误关联后重新建立正确关联
- **召回/退货**：已销售产品召回或退货，实物退回，需取消销售/出货关联
- **数据纠错**：误操作、系统错误修正，需取消错误建立的关联

### 不适用

- **生产过程中剔除**：门禁不通过时，系统不计数、不建立关联，数据从未进入关联表，写入剔除记录表供追溯；此类数据无需本功能操作。

### 与生产闭环的关系

- 采集（建立关联）→ 剔除（不建关联，写剔除记录表）→ 成垛 → 上传 → **数据维护**（码替换、取消关联）
- 取消关联后回迁的码可参与后续码替换，形成「取消→回迁→再使用」闭环。
