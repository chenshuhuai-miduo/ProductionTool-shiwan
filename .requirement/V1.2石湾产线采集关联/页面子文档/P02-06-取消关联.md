# P02-06 - 取消关联

## 子功能名称
取消关联 - 单个取消关联和批量取消关联

## 功能概述

解除码之间的关联关系，使追溯数据与实物状态一致。支持单个、批量两种模式，均需密码123456确认。

**与码替换的区别**：

| 功能 | 适用场景 | 关联关系 |
| -- | -- | -- |
| **码替换** | 实物未变，仅码损坏/错误 | 保留，用新码替换旧码 |
| **取消关联** | 实物已剔除/返工/重包，或关联错误 | 解除，从追溯链中移除 |

**业务场景**：剔除品/缺陷品移除、装错箱盒修正、返工重包、召回退货、数据纠错等。

**重要说明**：
- 单个/批量通过选项卡切换；均需弹窗二次确认+密码123456。
- 解除顺序：垛→箱→盒→瓶，逐级解除，不可一次全解。
- 已上传云端：先解云端再解本地，保证一致。
- **冷热分离**：解除关联后，被解除的码需**回迁**（冷表→热表），方可参与后续码替换。详见 AI 文档 10.1.1。

## 单个取消关联模式

### 操作表单设计（左侧区域）

- **表单**：左 600px，白底卡片；标题「单个取消关联」24px
- **第一个码输入**：*第一个码 (?)；必填；支持手动/扫码；占位：「请输入或扫描第一个码」；帮助：「同一包装内任意一码」
- **第二个码输入**：*第二个码 (?)；必填；支持手动/扫码；占位：「请输入或扫描第二个码（同一包装内）」；帮助：「与第一个码同包装的另一码」
- **识别按钮**：蓝底，48×100px；触发识别，结果在右侧
- **警告**：操作不可恢复，请确认后执行（红字，按钮上方）
- **清空**：清空输入，不清空右侧识别结果
- **成功后清空**：自动清空两个码输入框
- **确认取消关联**：红底，48×150px；有上级关联时禁用；点击弹出密码确认

### 识别结果展示设计（右侧区域）

- **区域**：右 700px，白底；标题「识别结果」24px
- **格式**：可解除 `盒码 xxx，可解除关联，解除12个`；不可解除 `箱码 xxx，不可解除（原因）链路: xxx→xxx`
- **原则**：首要显示能否解除，原因放括号内；长文字换行
- **显示**：浅灰底，min 500px，可滚动；初始提示「请在左侧输入码值并点击识别」

### 取消结果展示设计（右侧区域下方）

- **区域**：识别结果下方，高约200px，可滚动
- **成功**：绿字「取消关联成功！」+ 包装信息、影响范围、云端状态、时间
- **失败**：红字「取消关联失败！」+ 失败原因、时间
- **格式**：`10:25:33 盒码xxx 解除12个`；初始「暂无取消关联记录」

### 单个取消关联确认对话框
- 标题：取消关联确认；密码：123456；确认信息含包装信息+警告「此操作不可恢复」
- 按钮：取消（灰）、确认（红）

### 操作流程
1. 扫第一个码、第二个码（同包装内）→ 验证格式、存在性
2. 点击识别 → 显示识别结果、影响范围
3. 若有上级关联：先逐级解除（垛→箱→盒→瓶）再回本包装
4. 若无上级：点击确认取消关联 → 弹窗输密码 → 云端检查 → 解云端（如有）→ 解本地 → 记录日志
5. 成功后自动清空两个码输入框

## 批量取消关联模式

### 操作表单设计（左侧区域）

- **大码输入**：*大码 (?)；支持手动/扫码；占位「盒码/箱码/垛码」；帮助「需取消关联的盒/箱/垛码」
- **添加按钮**：蓝底，将大码加入列表
- **已添加的大码列表**：max 200px 可滚动；格式 `序号. 码类型 码值 [删除]`；空时「暂无添加的大码」
- **混合层级**：列表可含盒码、箱码、垛码；**执行时按每个码的层级分别解除**（盒级解盒、箱级解箱、垛级解垛）
- **识别按钮**：触发批量识别
- **清空**：清空输入和列表，不清空右侧；**成功后清空**：自动清空输入和列表
- **确认批量取消关联**：红底，点击弹出密码确认

### 识别结果展示设计（批量模式）

- 格式同单个模式，每单元一行；底部汇总**总影响范围**
- **影响范围统计**：仅统计可解除当前级的单元；码关系=子级码数（盒级=瓶数，箱级=盒数，垛级=箱数）；示例「2个可解除，涉及32个码关系」
- 初始提示「请在左侧添加大码并点击识别」

### 取消结果展示设计（批量模式）

- 与单个共用取消结果区域；逐条展示每个可解除码的结果
- 部分失败：显示「部分取消关联失败」+ 按单元列示失败原因

### 密码确认对话框
- 标题：批量取消关联确认；密码123456；显示单元数、码关系数；警告「此操作不可恢复」

### 操作流程
1. 输入大码 → 添加至列表（可多个，可删除）
2. 点击识别 → 显示每个单元的上级状态、链路、总影响范围
3. 若有上级：先逐级解除再回该单元
4. 确认批量取消 → 弹窗输密码 → 云端检查 → 解云端（如有）→ **按每个码的层级分别解除**（盒级解盒、箱级解箱、垛级解垛）→ 解本地 → 记录日志
5. 成功后自动清空输入和列表

## 状态栏设计
- 时间：YYYY-MM-DD HH:mm:ss，右对齐，每秒更新

## 交互功能设计

### 验证规则
- 码值：必填、格式、存在性
- 关联关系：码间关联、包装单元、影响范围

### 取消关联规则
- **二次确认与密码**：均需弹窗+密码123456
- **解除顺序**：垛→箱→盒→瓶逐级，不可一次全解；每级前判断上级已解
- **识别展示**：显示上级状态、链路；有上级时禁用确认按钮；影响范围仅统计可解除单元
- **批量混合层级**：列表可含盒/箱/垛；**执行时按每个码的层级分别解除**
- **云端**：已上传则先解云端再解本地
- **成功后清空**：单个清空两码输入；批量清空输入和列表；右侧识别/结果不清空

## 页面数据展示逻辑
- 云端检查 → 解云端（如有）→ 解本地 → **码回迁**（冷表→热表）→ 记录日志 → 显示结果

## 性能要求
- 取消关联≤3s；云端检查≤1s；云端解除≤2s；码验证≤100ms；识别≤500ms；界面≤300ms

## 兼容性要求
