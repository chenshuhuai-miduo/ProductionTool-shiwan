# P02-06 - 取消关联

## 子功能名称

取消关联

## 功能概述

取消已建立的码关联关系，使追溯数据与实物状态一致。支持单个或批量添加盒/箱/垛码到取消列表，统一识别、统一确认执行。

**适用对象**：本功能仅针对**已写入关联表**的码关联数据。生产过程中门禁不通过被剔除的品（不计数、不建立关联），其数据从未进入关联表，无需也无法通过本功能操作。

**与码替换的区别**：


| 功能       | 适用场景                                                | 关联关系         | 码去向             |
| -------- | --------------------------------------------------- | ------------ | --------------- |
| **码替换**  | 喷码损坏/无法扫读，以未关联的同类新件整体替换（新瓶换旧瓶、新盒换旧盒），新件的码继承原位置的关联链路 | 保留，新码替代旧码的位置 | 旧码退出关联链，不回收     |
| **取消关联** | 该件需从产线流程中完全移出（报废、返工、重新包装、装错位置等）；或关联建立有误需从追溯链移除      | 取消，从追溯链中移除   | 码回迁至热表，可参与后续码替换 |


> **说明**：瓶、盒、箱的码均为喷码，直接喷印在瓶盖、纸盒或纸箱上，不存在换贴的情况。码替换的本质是以一个**未关联的完整新件**替换喷码损坏的旧件，新件继承旧件在关联链中的位置；取消关联则是将该件彻底从追溯链移除。

**业务场景**：已关联但包材已报废/返工/重新包装、装错箱盒修正、数据纠错、召回退货等。

**重要说明**：

- 取消均需弹窗二次确认 + 密码 123456。
- 不能跳过上级直接取消下级：若指定码有上级关联，系统标记为不可取消并提示（有上级时须先取消上级）。
- 已上传云端：云端数据以**整垛**为单位，取消关联时须先调用云端接口取消**整垛**数据，成功后再取消本地；云端取消失败则本地不执行，结果标记失败。
- **冷热分离**：取消成功后，码是否回迁热表取决于其是否已完全脱离关联链（无父级且无子级）。"只解一层"下盒码（瓶-盒全断）回迁热表；箱码/垛码因下层仍有关联，释放码留冷表。"全部解除"下箱码涉及的箱/盒/瓶全部回迁热表；垛码受层级限制等同"只解一层"。详见 AI 文档 10.1.1。

**取消范围（两种模式，用户选择）**：


| 模式       | 含义                                                   | 适用场景                                           |
| -------- | ---------------------------------------------------- | ---------------------------------------------- |
| **只解一层** | 只取消该码对应层级的直接关联：垛码断箱-垛，箱码断盒-箱，盒码断瓶-盒；下一层关联保留          | 归错垛/箱（垛码）；换箱体保留盒内结构（箱码）；盒码与"全部"等价              |
| **全部解除** | 取消该码对应层级及以下所有关联；**垛码受层级限制，仍只断箱-垛**；箱码断盒-箱+瓶-盒；盒码断瓶-盒 | 彻底清除：换盒体+瓶全清（箱码）；返工/召回（分步：先垛码断箱-垛，再批量箱码执行全部解除） |


**各层级在两种模式下的取消范围**：

> **说明**：下表中的数字（70箱/4盒/6瓶）为石湾产线当前配置比例（1垛=70箱、1箱=4盒、1盒=6瓶），仅作示例；实际数量取决于系统参数配置，不同产线或配置下数值不同。


| 输入码     | 只解一层                         | 全部解除                         |
| ------- | ---------------------------- | ---------------------------- |
| 垛码      | 解除垛下70条箱-垛关联；盒-箱、瓶-盒**保留**   | 同左（垛码层级限制：只能解除箱-垛）           |
| 箱码（无上级） | 解除4条盒-箱关联；瓶-盒**保留**，各盒可重新上产线 | 解除4条盒-箱+24条瓶-盒，共28条；所有码可重新关联 |
| 盒码（无上级） | 解除6条瓶-盒关联；可重新关联（与"全部解除"等价）   | 同左（盒码无更下层，两模式等价）             |


> **注**：盒码两种模式效果相同；垛码两种模式效果也相同（受层级限制）。识别结果区会对这两种情况提示"模式等价"。

**瓶码**：不支持直接输入瓶码取消；扫瓶码时系统自动查找其所在盒码并替换为盒码加入列表（瓶必须以盒为单位取消）。

## 操作区设计（左侧）

### 码输入

- **标签**：码（盒/箱/垛，支持扫瓶码）；必填；18px 加粗
- **输入框**：支持手动输入/扫码；高度 40px；占位「请输入或扫描盒码、箱码、垛码（或瓶码）」
- **加入列表按钮**：蓝底，40×100px；将识别后的码加入待取消列表

**瓶码自动转换**：扫入瓶码时，系统自动查找该瓶所在的盒码，弹出提示：

```
提示：瓶码 XXX 属于盒码 YYY
     已自动转换为盒码 YYY 加入列表
     [确认]  [取消]
```

用户确认后，以盒码 YYY 加入列表；取消则清空输入框。

### 待取消列表

- **格式**：`序号. 码类型 码值 [移除]`；空时显示「暂无待取消项」
- **列表高度**：max 200px，超出可滚动
- **移除**：从列表中删除该项（不影响已执行的结果）

### 识别按钮

- 蓝底，40×80px；对列表中所有码执行识别，结果显示在右侧
- 识别结果根据当前"取消范围"选择显示对应影响说明

### 取消范围

- **位置**：识别按钮下方，确认按钮上方
- **控件**：单选按钮组（Radio Group）
- **选项**：
  - `● 只解一层` —— 默认选中
  - `○ 全部解除`
- **切换时机**：切换选项后，若已有识别结果，自动刷新识别结果区的影响说明（无需重新点击识别）

### 操作按钮

- **清空列表**：灰底，40×100px；清空输入框和待取消列表，不清空右侧结果
- **确认取消关联**：红底，40×150px；点击弹出密码确认对话框
- **警告文字**：红色「⚠ 操作不可恢复，请确认后执行」，位于按钮上方

## 操作流程

```
1. 输入/扫描码 → 点击"加入列表"
       └─ 瓶码 → 弹出提示自动转换为盒码 → 确认后以盒码加入列表
       └─ 盒/箱/垛码 → 直接加入列表

2. 可重复步骤1，添加多个码到列表

3. 点击"识别" → 系统批量查询每个码的关联状态
       └─ 右侧识别结果展示每项：✓可解除 / ✗不可解除+原因

4. 确认内容后，点击"确认取消关联" → 弹出密码确认对话框
       └─ 输入密码 → 验证通过 → 执行取消

5. 执行逻辑（系统自动处理）：
       └─ 对✓可取消项按 垛→箱→盒 顺序依次执行
       └─ 若列表中同时含垛码和该垛下的箱码：
          系统先处理垛码（断箱-垛），箱码自动变为可取消状态，再按序处理
       └─ 若模式为"全部解除"且列表中的盒码已被其上级箱码的执行连带取消，
          执行到该盒码时自动跳过（已无关联）
       └─ 每项执行结果实时追加到取消记录区
       └─ 对✗不可取消项跳过，保留在列表

6. 执行完成：
       └─ 成功项自动从列表移除
       └─ 失败项和不可取消项保留在列表，供用户处理
       └─ 输入框清空
```

## 交互功能设计

### 验证规则


| 规则     | 说明                                                              |
| ------ | --------------------------------------------------------------- |
| 码存在性   | 码值必须在关联表中存在；不在则提示「该码未在关联表中，无需取消关联」                              |
| 码类型    | 支持盒码、箱码、垛码（含自动转换后的盒码）；输入其他格式提示「码格式不支持」                          |
| 上级关联检查 | 识别时查询该码是否有上级关联；有上级则标记✗不可取消，显示上级码值；**无上级（如采集中途尚未归入上级）则直接标记✓可取消** |
| 重复添加   | 列表中已存在该码时，提示「该码已在列表中」，不重复添加                                     |


### 取消关联规则


| 规则       | 说明                                                                                                                       |
| -------- | ------------------------------------------------------------------------------------------------------------------------ |
| 二次确认与密码  | 必须弹窗 + 输入密码 123456                                                                                                       |
| 取消范围选择   | 全局生效；同一次操作中所有待取消项均使用相同的取消范围；两种模式：只解一层、全部解除                                                                               |
| 取消顺序     | 系统自动按垛→箱→盒顺序执行；若列表中的箱码其上级垛码也在列表中，系统先处理垛码（断箱-垛），之后该箱码自动变为可取消状态再处理                                                         |
| 只解一层执行逻辑 | 垛码：断全部箱-垛，盒-箱/瓶-盒保留；箱码：断全部盒-箱，瓶-盒保留；盒码：断全部瓶-盒。各码是否回迁热表取决于是否已完全脱离关联链                                                      |
| 全部解除执行逻辑 | 垛码：受层级限制，同只解一层（只断箱-垛）；箱码：递归断盒-箱+瓶-盒，所有涉及码回迁热表；盒码：同只解一层（最低层，等价）                                                           |
| 不可取消项跳过  | ✗不可取消项在执行时自动跳过，不中断整体流程；若其上级码在同一批次且先被处理，上级取消后该码重新判断为可取消并继续处理                                                              |
| 云端处理     | 云端数据以整垛为单位上传；若涉及码所在的垛已上传云端，须先调用云端接口**取消整垛数据**，成功后再取消本地；云端取消失败则本地不取消，结果标记失败。取消后如需重建关联并重新上传，须整垛重新上传                        |
| 码回迁      | 码完全脱离关联链（无父级且无子级）时回迁热表。只解一层下：盒码操作后盒/瓶回迁，垛码/箱码操作后下层仍有关联不回迁。全部解除下：箱码操作后箱/盒/瓶全部回迁。废弃实物对应的码回迁后为热表闲置记录，因物理件已废弃不会被再次扫入，不影响系统运行 |


### 帮助内容（点击 ? 显示）

- **输入码**：支持盒码、箱码、垛码；扫入瓶码时系统自动转换为其所在的盒码。不知道盒/箱/垛码时，可先前往「数据查询」查询码的归属链路。
- **取消范围**：
  - **只解一层**（默认）：只取消该码层级的直接关联，下一层保留。垛码→断箱-垛；箱码→断盒-箱（瓶-盒保留）；盒码→断瓶-盒。适用于归错垛/箱（垛码）、换箱体（箱码，保留盒内结构）等场景。操作完成后将实物重新上产线重建关联，瓶-盒层级也可前往「手工采集」页手工重建。
  - **全部解除**：取消该码层级及以下所有关联。垛码受层级限制仍只断箱-垛（与只解一层等价）；箱码断盒-箱+瓶-盒，所有码回迁热表；盒码与只解一层等价。适用于需要彻底清除箱内所有关联的场景（箱码）。召回/返工等需要从垛开始清除的场景，需先垛码断箱-垛，再批量输入箱码执行全部解除。
- **有上级时**：若码已关联至上级（如盒已在箱内），须先取消上级才能取消该码；识别结果会提示上级码值，将上级码加入列表后执行即可。
- **已上传**：云端数据以整垛为单位。若涉及码所在的垛已上传云端，系统先调用云端接口取消整垛数据，成功后再取消本地，保证两端一致。

## 功能边界

### 包含

- 盒/箱/垛码加入列表、批量识别、批量取消
- 瓶码自动转换为盒码
- 识别结果展示（可/不可取消+原因）
- 密码保护、操作日志、码回迁

### 适用场景（均针对已写入关联表的数据）


| 场景                        | 建议模式     | 操作步骤                                                                |
| ------------------------- | -------- | ------------------------------------------------------------------- |
| **采集中发现盒/瓶损坏（盒尚未归箱）**     | **只解一层** | 输入盒码（无上级，直接可取消）→ 断6条瓶-盒 → 瓶重新发回产线                                   |
| **采集中发现箱损坏（箱尚未归垛，盒完好）**   | **只解一层** | 输入箱码（无上级）→ 断4条盒-箱，瓶-盒保留 → 盒重新上产线                                    |
| **采集中发现箱损坏（箱尚未归垛，盒也有问题）** | **全部解除** | 输入箱码（无上级）→ 断4盒-箱+24瓶-盒 → 实物全部重新发回产线                                 |
| 归错了垛（箱在错误垛内）              | **只解一层** | 输入垛码 → 断70条箱-垛，各箱盒-箱保留 → 各箱重新上产线归正确垛                                |
| 垛组装错误，需重新拆分箱              | **只解一层** | 输入垛码 → 断70条箱-垛，各箱内部关联保留 → 各箱重新上产线按正确分组成垛                            |
| **换箱体（箱喷码损坏/报废，盒完好）**     | **只解一层** | ①若箱在垛内：先输入垛码断箱-垛；②输入箱码断4条盒-箱（瓶-盒保留）；③各盒转入新箱后重新上产线重建盒-箱及箱-垛关联        |
| **换盒体（盒喷码损坏/报废，瓶完好）**     | **只解一层** | ①若有上级：逐级输入垛码→箱码，用只解一层逐层释放；②输入盒码断瓶-盒；③瓶转入新盒后重新上产线或前往手工采集页重建瓶-盒关联     |
| **成垛后发现问题/返工/召回退货**       | **分步操作** | ①输入垛码（只解一层）→ 断箱-垛，各箱释放；②批量输入所有箱码（全部解除）→ 断盒-箱+瓶-盒，所有码回迁热表；不支持从垛码一步全清 |
| 数据纠错（关联建错）                | 视情况      | 只需释放至某层后重新上产线：用**只解一层**逐层处理；需清除某层及以下所有关联：用**全部解除**（箱码）              |


### 不适用

- **生产过程中剔除**：门禁不通过时，系统不计数、不建立关联，数据从未进入关联表；此类数据无需本功能操作。
- **喷码损坏/无法扫读，需整件替换**：应使用「码替换」功能，以未关联的新件替换旧件，新件的码继承原关联链路位置，不需取消关联再重建。

### 与生产闭环的关系

```
采集（建立关联）→ 成垛 → 上传
      ↑                    ↓
重新上产线             数据维护（码替换、取消关联）← 取消后码回迁至热表 ←┘
      ↑                    ↓
      └── 只解一层后，下层结构保留，实物重新上产线由产线自动重建关联
          （瓶-盒层级也可通过手工采集页面重建）
```

取消关联后：

- **只解一层**：下层关联保留，实物重新上产线由产线自动采集重建上层关联；瓶-盒层级也可前往「手工采集」页手工重建
- **全部解除（箱码）**：所有关联清除，码回迁热表，可参与后续码替换，形成「取消 → 回迁 → 再使用」闭环
- **垛码操作**：仅断箱-垛，后续需批量输入箱码继续处理；不支持从垛码一步全清
