# P01-02 - 系统设置（1号机）

## 子功能名称
系统设置 - 设备配置和参数设置

## 功能概述
为系统管理员提供1号机瓶盒关联软件的系统配置功能界面。**复用现有系统设置页面**，通过设备类别、模块上下文等方式区分1号机所需配置。1号机瓶盒数据需存入2号机所在数据库（局域网内访问），因此必须提供**数据库连接配置**（2号机IP、端口、数据库名等）。

**重要说明**：本页面需要密码保护，访问时弹窗提示输入密码，密码验证成功后允许操作。每次访问都需要重新验证（不保存登录状态）。密码保护为1号机入口的增强要求，可在打开系统设置前增加校验逻辑。

## 实现方式：复用现有系统设置

### 布局结构示意
```
┌─────────────────────────────────────────────────────────────────────────┐
│  系统设置（1号机）                                                  [×] │
├─────────────────────────────────────────────────────────────────────────┤
│  [IO设备管理] [打印机管理] [报警设置] [基础设置]                          │
├─────────────────────────────────────────────────────────────────────────┤
│  基础设置选中时，下方显示二级 Tab：                                       │
│  [数据库连接] [码位数配置]                                               │
├─────────────────────────────────────────────────────────────────────────┤
│  （1号机无虚拟垛标规则，垛标由2号机生成）                                 │
└─────────────────────────────────────────────────────────────────────────┘
```

### 一级 Tab 结构
| 一级 Tab | 说明 |
|----------|------|
| IO设备管理 | 设备列表（添加/编辑/删除）、连接测试、设备状态监控 |
| 打印机管理 | 打印机配置 |
| 报警设置 | 声音报警开关、报警延时、报警间隔、测试报警 |
| 基础设置（**新增**） | 二级 Tab：数据库连接、码位数配置 |

### 基础设置 - 二级 Tab 结构
| 二级 Tab | 说明 |
|----------|------|
| 数据库连接 | 1号机连接2号机数据库，瓶盒数据写入2号机所在库 |
| 码位数配置 | 采集时校验码位数 |

### 1号机适配要点
1. **入口与标题**：从1号机主界面进入时，窗口/标题显示为「系统设置（1号机）」。
2. **密码保护**：进入系统设置前弹出密码输入框，验证通过后再加载页面。
3. **设备配置**：通过 IO 设备管理的设备类别（如瓶码相机、盒码相机、剔除装置、报警灯等）配置1号机所需设备，无需单独表单。
4. **数据库连接**（基础设置→数据库连接）：1号机瓶盒数据需存入与2号机同一数据库，数据库部署在2号机，通过局域网访问。表单项：
   - **2号机IP地址**：数据库所在机器（2号机）的局域网IP
   - **数据库端口**：如 3306（MySQL）
   - **数据库名**：与2号机共用
   - **用户名/密码**：数据库访问凭证
   - **连接测试**：测试能否成功连接2号机数据库

5. **码位数配置**（基础设置→码位数配置）：用于采集时校验码的位数是否符合要求。输入框，支持配置：
   - **小标位数**：瓶码位数，如 14；-1 表示不校验
   - **中标位数**：盒码位数，如 14；-1 表示不校验
   - **大标位数**：箱码位数（1号机不采箱码，可配置供参考或预留）；-1 表示不校验
   - **校验逻辑**：采集时若实际采码位数与配置值不一致，则异常告警、关联失败

### Tab 顺序
**一级 Tab**：1. IO设备管理  2. 打印机管理  3. 报警设置  4. 基础设置

**基础设置下二级 Tab**：1. 数据库连接  2. 码位数配置

### 按钮说明
- 基础设置下各 Tab（数据库连接、码位数配置）内表单下方均保留**保存**按钮，跟在表单末尾，不设取消/重置按钮。

### 无需重复实现
- 设备配置表单、连接测试等交互逻辑
- 设备状态监控、底部状态栏等布局
- 打印机、报警等已有功能

## 密码保护机制（需新增）
- **访问控制**：打开系统设置页面时，首先弹出密码输入对话框
- **密码验证**：输入系统预设密码进行验证
- **验证失败**：密码错误时显示错误提示，不允许继续操作
- **验证成功**：密码验证成功后，允许加载系统设置页面
- **会话管理**：每次访问都需要重新验证，不保存登录状态

## 功能边界

### 包含的功能
1. 复用现有系统设置（IO设备管理、打印机管理、报警设置）
2. 1号机入口的密码保护
3. **基础设置**（新增一级 Tab，下含二级 Tab）：
   - **数据库连接**：2号机IP、端口、数据库名等，用于1号机将瓶盒数据写入2号机所在数据库（局域网）
   - **码位数配置**：大标、中标、小标位数输入框；-1 表示不校验；采集时位数不符则异常告警、关联失败
4. 1号机上下文下的设备配置与参数管理（按需扩展）

### 适用场景
- **系统初始化**：首次部署时的系统配置
- **设备更换**：更换硬件设备时的配置调整
- **参数调整**：根据生产需求调整业务参数
- **故障排查**：设备故障时的配置检查和测试
