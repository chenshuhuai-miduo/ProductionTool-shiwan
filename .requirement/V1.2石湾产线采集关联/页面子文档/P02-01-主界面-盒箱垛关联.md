# P02-01 - 主界面-盒箱垛关联

## 子功能名称

主界面-盒箱垛关联数据采集

## 功能概述

作为2号机盒箱垛关联软件的主界面，为生产线操作员提供盒箱垛关联数据采集的控制操作界面。界面设计简洁实用，主要功能包括：选择产品和设置生产单号、开始/停止采集、单位实时统计（含已生产垛数、总剔除数）、任务控制（按钮一行两个）、强制操作、剔除相关操作、运行状态监控。界面采用专业的工业软件设计风格、卡片式布局，针对工控机鼠标键盘操作优化。

**重要说明**：本界面为2号机系统主界面，负责盒箱垛关联数据采集，读取同步自1号机的瓶盒关联数据（同步机制见「1号机数据同步」章节），完成四级关联（瓶-盒-箱-垛）。界面设计遵循简洁原则，只保留必要的状态指示和功能按钮。

**界面结构原则**：2号机盒箱垛软件进入即呈现多Tab页面效果，默认高亮选中「数据采集」作为主界面。除系统设置复用已有弹窗交互（通过菜单/工具栏以模态对话框打开）外，其他功能入口均采用Tab交互形式。

## 界面布局设计

### 多Tab整体结构

- **进入效果**：软件启动后即呈现多Tab页面，Tab栏位于菜单栏下方
- **Tab 显示控制**：各 Tab 的显示/隐藏由**系统设置→页面配置**控制，每页面对应 Switch 开关；选上则显示、取消则隐藏。数据采集为主界面，必选不可关闭。
- **Tab形式**（默认全开）：数据采集、手工采集、数据查询、数据替换、生产统计、码包管理、取消关联、数据上传，共8个Tab
- **数据采集Tab**：默认选中并高亮，作为主界面，展示盒箱垛关联数据采集内容
- **手工采集Tab**：点击切换展示手工采集页面（P02-02）
- **数据查询Tab**：点击切换展示数据查询页面（P02-04）
- **数据替换Tab**：点击切换展示数据替换页面（P02-05）
- **生产统计Tab**：点击切换展示生产统计页面（P02-07）
- **码包管理Tab**：点击切换展示码包管理页面（P02-03）
- **取消关联Tab**：点击切换展示取消关联页面（P02-06）
- **数据上传Tab**：点击切换展示数据上传页面（P02-08）
- **弹窗类功能**：仅系统设置，通过菜单「配置」按钮以模态对话框打开，沿用已有弹窗交互
- **Tab切换**：点击不同Tab时，主内容区展示对应页面，工具栏根据当前Tab展示该Tab相关的功能入口

### 整体布局

- **布局方式**：桌面应用风格 + 菜单栏 + Tab栏 + 左中右三栏布局
- **适配要求**：工控机鼠标键盘操作优化，支持快速操作
- **主要区域**：
  - 菜单栏：文件、配置、数据、帮助
  - Tab栏：数据采集（主界面，默认选中高亮）、手工采集、数据查询、数据替换、生产统计、码包管理、取消关联、数据上传
  - **左侧栏**：上面当前生产信息、采集规格设置；下面实时上传数据区（展示垛码、箱数、上传状态）
  - **中间栏**：数据接收区、操作日志、报警信息等信息展示区域，按块分块显示
  - **右侧栏**：上面单位实时统计，下面任务控制
  - 状态栏：遵循 通用规则文档「02-用户界面设计」之状态栏设计

### 布局优先级（上优先）

- **上半屏（优先展示/操作）**：产品选择与生产单号、开始/停止采集、强制操作、关键计数
- **下半屏（次要但可追溯）**：中间栏三块（见下文「中间栏设计」），均倒序显示最新在上

## 开始采集前初始化检测与准备

点击「开始采集」时，系统按以下顺序检测；**全部通过**后方可弹出二次确认弹窗并开始采集。

### 2号机（盒箱垛关联）检测项

| 类别 | 检测项 | 说明 | 不通过时 |
|------|--------|------|----------|
| **门禁** | 码包是否已导入 | 盖外码小标、盒外码中标、箱外码大标；未导入则提示，支持在线拉取或本地导入 | 提示导入码包，阻止开始 |
| **必填** | 产品是否已选择 | 左侧「产品选择」下拉框已选中产品 | 提示「请先选择生产产品」 |
| **必填** | 生产单号是否已填写且合法 | 不能为空；仅允许字母、数字或组合；长度限制4~16位 | 提示「请先填写生产单号」或「生产单号格式不正确」 |
| **防重复** | 生产单号是否已存在生产记录 | 检测本地数据库中该单号是否有历史记录；若存在则弹窗确认（可选择取消修改或继续追加） | 弹窗确认，用户选择「取消」则阻止开始 |
| **规格** | 采集规格是否已确认 | 1垛 [70]箱、1箱 [4]盒；开始采集后不可修改 | 无（可修改） |
| **设备** | 硬件是否在线 | 盒码相机、箱码相机、剔除装置、报警灯等（按系统设置→设备配置） | 提示设备未连接，阻止开始 |
| **连接** | 数据库是否可访问 | 2号机本机库可访问；2号机需能访问1号机库（用于定时轮询拉取瓶盒关联数据，系统设置→连接→数据库连接） | 提示连接失败，阻止开始 |

### 系统级准备（一次性或按需）

- **系统设置**：设备（IO 设备管理、设备控制配置如剔除装置）、连接（数据库、接口服务）、业务（码位数、虚拟垛标、上传配置等）
- **首次部署**：配置设备接口、系统参数、产品信息并做测试验证
- **日常**：非首次启动时检查设备连接即可

### 左侧栏设计（宽度：约340px）

- **功能定位**：左侧上下两个卡片。上卡片为当前生产信息（含采集规格设置）；下卡片为实时上传数据区。

#### 模块一：当前生产信息（上卡片）

- **生产产品选择**：下拉框，点击弹出产品选择弹窗；选中后回显产品名称、产品编号（只读）；开始生产后不可更改。数据策略见下文「产品选择弹窗」。
- **生产单号**：可编辑，高度40px；默认当天日期+001（如 `20241201001`）。
  - **输入规则**：仅允许输入字母、数字或两者组合；长度限制4~16位；不能为空；不符合时提示并阻止开始采集。
  - **开始采集后不可修改**：采集运行期间输入框变为只读，停止采集后恢复可编辑。
  - **防重复机制**：点击「开始采集」时，系统检测当前生产单号是否在本地数据库中已存在生产记录；若已存在，弹窗提示「生产单号 [XXX] 已存在生产记录，是否继续使用？」，提供「取消」（返回修改）和「继续」（覆盖追加）两个选项；操作员确认继续后方可开始采集。
- **采集规格设置**（卡片内区域）：第一行「1垛 [70]箱」，第二行「1箱 [4]盒」；70、4为可编辑输入框，默认值70、4（石湾产线）；开始采集后不可修改。箱数达到预设值即满垛，自动上传。
- **输入框校验规则**：不能为空；仅允许输入整数；范围 1～999；不符合时提示并阻止开始采集。

#### 模块二：实时上传数据区（下卡片）

- **功能**：展示成垛后待上传/上传状态。
- **显示**：列表形式，每项展示垛码、箱数、上传状态。第一行垛码与上传状态两端对齐，第二行箱数左对齐；支持「待上传」「已上传」等状态标识。

### 中间栏设计（宽度：自适应，最小600px）

中间栏**自上而下固定分三块**，高度比例约为 **60% : 20% : 20%**，块间用分隔线或留白区分，便于区分扫码数据、操作记录与报警。

#### 模块一：数据接收区（上，约60%高）

- **功能定位**：展示扫码与关联数据，为主要信息区
- **区域样式**：白色背景，边框分割，可滚动
- **显示内容**：
  - 盒码/箱码/垛码等扫码数据及关联结果
  - 装箱完成、满箱/满垛提示等实时业务信息
- **显示格式**：时间戳（HH:MM:SS） + 内容，18px 字体
- **显示特性**：倒序显示，最新在上，自动滚动到顶部；文字过长时自动换行显示，不截断

#### 模块二：操作日志（中，约20%）

- **功能定位**：记录用户操作与系统关键动作
- **区域样式**：白色背景，边框分割，可滚动
- **显示内容**：设备在线状态、开始/停止采集、产品选择、强制满垛、提取工单未成垛等操作记录及关键状态
- **显示格式**：时间戳（HH:MM:SS） + 内容，16px 字体
- **显示特性**：倒序显示，最新在上，自动滚动到顶部；文字过长时自动换行显示，不截断

#### 模块三：报警信息（下，约20%）

- **功能定位**：集中展示报警与异常信息
- **区域样式**：白色背景，边框分割，可滚动
- **显示内容**：
  - 重码、未成功采集、关联失败等异常
  - 其他报错与报警信息
- **报警描述规则**：**必须描述具体原因**，便于工人快速定位；异常类型定义及完整规则详见 **AI 文档 10.1.7 码校验通用规则**，报警显示格式如下：
  - 小标不通过 → 显示「瓶码 xxx 不在小标码包范围内」
  - 中标不通过 → 显示「盒码 xxx 不在中标码包范围内」
  - 大标不通过 → 显示「箱码 xxx 不在大标码包范围内」
  - 重码 → 显示「瓶码/盒码/箱码 xxx 重复出现」
  - 错码 → 显示「瓶码/盒码/箱码 xxx 格式错误」
  - 超码/缺码 → 显示「盒码 xxx 多/少 x 个瓶码」
  - 盒箱校验不通过 → 显示「箱码 xxx 盒数不符」
- **文字颜色**：统一红色文字，不使用背景色
- **显示格式**：时间戳 + 具体异常描述（含层级、码值、原因），18px 字体
- **显示特性**：倒序显示，最新在上，初始可为空，有报警时动态追加；文字过长时自动换行显示，不截断

**交互**：上述三块均倒序显示最新在上。

### 右侧栏设计（宽度：约336px）

- **功能定位**：上面单位实时统计，下面任务控制。

#### 模块一：单位实时统计（上）

- **布局**：文字在上、数字在下；每项独立白底卡片
  - **当前箱数、每垛箱数**：文字在上、数字在下，两卡片横向并排；当前箱数红色、每垛箱数蓝色，大字号
  - **当前盒数、每箱盒数**：文字在上、数字在下，两卡片横向并排；大字号显示
  - **已生产垛数、总剔除数**：单独一行，文字在上、数字在下，样式较简洁；总剔除数40px大字号、加粗、红色
- **更新机制**：基于采集数据实时计算和更新。
- **统计范围**：
  - **已生产垛数**：本次软件启动后的累计垛数（满垛或强制满垛均计入）；软件重启后重置为0。
  - **总剔除数**：本次软件启动后的累计剔除数；软件重启后重置为0；操作员也可随时点击「剔除数清零」手动重置为0。

#### 模块二：任务控制（下）

- **功能定位**：提供生产任务执行过程中的核心控制操作
- **布局设计**：按钮集中排列，**一行两个**，按钮间距12px
- **按钮列表**：开始/停止采集、无需采集码、关闭报警、强制满垛、提取工单未成垛、收回剔除、剔除数清零
- **帮助提示**：标题旁显示圆圈问号图标 `(?)`，点击弹出该区域所有按钮的功能说明
- **视觉样式**：纯色背景、立体阴影效果、鼠标键盘操作优化

##### 主要控制功能

- **生产开始&关闭**：
  - **开始采集按钮**：
    - 功能：点击"开始采集"开始生产
    - 操作流程：
      1. 点击"开始采集"按钮
      2. 系统执行**开始采集前初始化检测**（码包门禁、产品/生产单号、设备、数据库等，详见上文「开始采集前初始化检测与准备」）
      3. 全部检测通过后弹出二次确认弹窗，展示生产信息（产品、生产单号）和包装比例（1垛X箱、1箱X盒）
      4. 用户确认后开始采集工作
      5. 按钮文字变更为"停止采集"
    - 样式：主要操作按钮，蓝色背景，白色加粗文字，高度40px，宽度120px
  - **停止采集按钮**：
    - 功能：再次点击按钮停止采集
    - 操作逻辑：
      - 当点击前有未满垛的数据，这些数据会保留，不会自动组垛（后续可以提取工单未成垛）
      - 停止采集后，按钮文字变更为"开始采集"
  - **无需采集码开关**：
    - 功能：当天不生产五码合一产品，开启后，读码器等设备不进行工作（其实软件都不用开，工厂要求，什么数据都不采集）
    - 样式：开关按钮，高度40px
    - 操作：开启后，所有读码设备不工作，不采集任何数据
  - **关闭报警**：

## 数据采集功能

### 实时数据采集展示

- **采集频率**：每2秒采集1-2个产品
- **数据格式**：
  - 盒码格式：20241201001001等
  - 箱码格式：20241201001002等
  - 垛码格式：P20241201001等
- **时间精度**：精确到毫秒（2024-12-01 09:15:23.456）
- **质量检测**：98%合格率，2%不合格率
- **实时更新**：采集数据以日志条目形式实时显示

### 四级关联采集流程

- **关联类型**：四级关联采集模式（瓶-盒-箱-垛）
- **采集规则**：
  - 第一层（瓶码）：1号机通过2台瓶码采集相机交叉去重采集，写入1号机自有数据库（旧版软件，不在本次开发范围）
  - 第二层（盒码）：1号机每计数到G个瓶码，采集关联1个盒码，写入1号机自有数据库；2号机定时同步拉取（详见「1号机数据同步」章节）
  - 第三层（箱码）：2号机基于同步到的盒码数据，每计数到N个有效盒码，采集关联1个箱码（2号机负责）；石湾产线场景下，一个箱码通常对应多个盒码
  - 第四层（垛码）：每计数到M个箱码，关联1个垛码（2号机负责）
- **盒-箱关联边界规则**（计数规则的时间边界补充）：
  - 盒码相机与箱码相机在生产线上各自独立采集，两者采集时间有先有后，任一方向均可能发生（即盒码先到或箱码先到）
  - **时间窗口**：若某个盒码的采集时间与某个箱码的采集时间相差在 **3 秒以内**（初步阈值，实际间隔可能更短），则该盒码归属于该箱，而非下一箱
  - **适用场景**：处理盒-箱关联中时间边界模糊的情况，防止在两箱交替时将盒码误归入错误的箱
  - **与计数规则关系**：本规则为补充，计数仍以 N 个有效盒码满箱为主逻辑；时间窗口用于判定边界盒码的归属
- **关联成功率**：99%成功，1%失败
- **异常处理**：关联失败时自动报警
- **数据展示**：在数据接收区以日志形式显示关联结果

### 统计计算

- **箱数统计**：基于采集数据自动累计
- **垛数统计**：按采集规格自动形成垛

## 1号机数据同步

> **本章节为唯一声明处**。文档其他位置凡涉及1号机数据来源，均引用此章节。

**背景**：1号机沿用旧版软件，不在本次开发范围内，其瓶盒关联数据写入1号机自有数据库。

**同步机制**：2号机软件**内置定时轮询**，定期从1号机数据库拉取差量数据（仅拉取上次同步后的新增数据），写入2号机本机数据库后供采集流程使用，运行期间持续执行。

- **数据内容**：瓶盒关联关系、采集时间戳
- **成功展示**：成功拉取的数据条目逐条展示在**数据接收区**
- **失败提示**：同步失败时在**报警信息区**显示具体原因，不阻断当前生产流程

## 自动上传机制

### 成垛后自动上传

- **配置依赖**：系统自动上传行为**根据系统设置→业务→上传配置**中的「自动上传开关」执行；默认开启。
- **触发时机**：当自动上传开关开启时，成垛后系统自动上传数据至米多大数据引擎；关闭时成垛后不自动上传，需手动上传。
- **上传内容**：
  - 垛码信息
  - 箱码关联关系
  - 盒码关联关系
  - 瓶码关联关系（通过2号机定时同步自1号机的瓶盒关联数据）
  - 任务信息
- **上传状态**：在数据接收区显示上传状态
- **状态反馈**：
  - 上传成功：三色声光报警灯绿灯常亮，无蜂鸣；**一分钟后自动熄灭**
  - 上传失败：三色声光报警灯红灯常亮 + 蜂鸣；报警信息区显示**具体失败原因**（具体到箱/盒/瓶）

### 三色声光报警灯控制

> **说明**：三色声光报警灯为声光一体设备，可独立控制灯光颜色和蜂鸣声音。

- **上传成功**：绿灯常亮，无蜂鸣；**一分钟后自动熄灭**
- **上传失败**：红灯常亮 + 蜂鸣；需人工点击「关闭报警」后停止
- **待上传**：不亮、不响
- **关闭报警后**：不亮、不响
- **关闭报警**：上传失败时三色声光报警灯红灯亮且蜂鸣，点击主界面「关闭报警」按钮可重置（停止灯光和蜂鸣）

## 报警系统

### 报警类型说明

- **短报警**：短暂提示音/闪烁，**自动关闭**，不需人工操作
- **长报警**：持续提示音/闪烁，需人工点击「关闭报警」按钮关闭

### 报警触发条件（2号机）

| 触发条件 | 报警类型 |
| -------- | -------- |
| 连续重码 3 次 | 长报警 |
| 上传异常（数据上传失败） | 长报警 |

> **连续重码3次定义**：在软件本次启动生命周期内，采集到的某一码（盒码/箱码）连续出现重码（即同一码在系统中已存在）且次数累计达到3次，触发长报警。退出软件后重新启动，计数重置，重新判断。


### 报警显示

- **视觉提示**：不同颜色区分报警级别
- **声音反馈**：通过三色声光报警灯发出蜂鸣提醒
- **日志记录**：所有报警自动记录到操作日志

### 报警控制

- **关闭报警**：位于采集控制块。用于关闭**长报警**（如连续重码3次、上传异常等）；点击「关闭报警」可重置三色声光报警灯（停止灯光和蜂鸣）
- **指令发送**：
  - 向三色声光报警灯发送停止指令（可分别控制灯光和蜂鸣）
  - 支持多种设备通信协议（串口RS232/RS485、网络TCP/IP）
  - 指令格式：根据设备协议定义（如Modbus、自定义协议等）
- **状态同步**：失败时显示错误提示并记录日志

## 页面数据展示逻辑

> 中间栏三个区域职责不同，各自展示内容如下。

### 数据接收区展示内容

实时采集与关联结果，所有条目格式均为 `HH:MM:SS 内容`。文字颜色：成功绿色，普通（同步数据等）黑色。

| 触发事件 | 展示内容示例 | 文字色 |
|---------|-------------|--------|
| 盒码校验通过并关联 | `10:30:10 盒码关联 20241201B0302（6瓶）` | 绿色 |
| 箱码校验通过并关联 | `10:30:15 箱码关联 20241201C0101（4盒）` | 绿色 |
| 满垛（含强制满垛） | `10:30:18 满垛 垛码 V20241201001（70箱）` | 绿色 |
| 1号机同步数据到达 | `10:30:12 接收1号机数据 - 瓶盒关联：盒码20241201B0302（6瓶）` | 黑色 |

### 操作日志展示内容

记录**用户操作**和**系统关键动作**，不包含扫码数据细节。文字颜色统一黑色。

| 触发事件 | 展示内容示例 |
|---------|-------------|
| 设备上线/离线 | `10:00:01 盒码相机 已连接` |
| 开始采集 | `10:00:05 开始采集 产品：XXX 单号：20241201001` |
| 停止采集 | `10:30:00 停止采集` |
| 产品选择 | `10:00:03 已选择产品：XXX` |
| 强制满垛（操作动作） | `10:28:00 强制满垛 当前箱数：65` |
| 提取工单未成垛 | `10:00:10 提取未成垛：箱码 20241201C0001` |
| 垛码关联失败（系统重试） | `10:30:19 垛码关联失败，重试中...` |

### 报警信息区展示内容

集中展示**异常与报警**，必须描述具体原因，便于工人快速定位。文字颜色统一红色。

| 异常类型 | 展示内容示例 |
|---------|-------------|
| 小标（瓶码）不在码包范围 | `10:30:13 瓶码 xxx 不在小标码包范围内` |
| 中标（盒码）不在码包范围 | `10:30:14 盒码 xxx 不在中标码包范围内` |
| 大标（箱码）不在码包范围 | `10:30:15 箱码 xxx 不在大标码包范围内` |
| 重码 | `10:30:14 盒码/箱码 xxx 重复出现` |
| 超码/缺码 | `10:30:13 盒码 xxx 多/少 x 个瓶码` |
| 盒箱校验不通过 | `10:30:15 箱码 xxx 盒数不符` |
| 剔除触发 | `10:30:16 剔除 箱码 20241201C0099` |
| 数据上传失败 | `10:31:00 上传失败：垛码 xxx，箱码 yyy 数量异常` |
| 1号机同步失败 | `10:30:20 1号机数据同步失败：连接1号机库超时` |

## 功能边界

### 包含的功能

1. 盒箱垛关联数据采集功能
2. 读取同步自1号机的瓶盒关联数据功能（1号机旧版软件不在本次开发范围）
3. 生产任务管理功能
4. 实时数据展示和统计功能
5. 中间栏三块（数据接收区、操作日志、报警信息）显示
6. 自动上传和报警灯控制功能
7. 码包管理、数据查询、数据维护等功能入口（通过 Tab 或菜单）

### 适用场景

- **生产线盒箱垛关联**：2号机负责的盒箱垛关联数据采集
- **实时监控**：生产过程中的实时状态监控
- **异常处理**：采集异常情况的报警和处理
- **数据上传**：成垛后自动上传数据
