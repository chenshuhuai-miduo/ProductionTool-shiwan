# P02-01 - 主界面-盒箱垛关联

## 子功能名称

主界面-盒箱垛关联数据采集

## 功能概述

作为2号机盒箱垛关联软件的主界面，为生产线操作员提供盒箱垛关联数据采集的控制操作界面。界面设计简洁实用，主要功能包括：选择产品和设置生产单号、开始/停止采集、单位实时统计（含已生产垛数、总剔除数）、任务控制（按钮一行两个）、强制操作、剔除相关操作、运行状态监控。界面采用专业的工业软件设计风格、卡片式布局，针对工控机鼠标键盘操作优化。

**重要说明**：本界面为2号机系统主界面，负责盒箱垛关联数据采集，接收1号机瓶盒关联数据，完成四级关联（瓶-盒-箱-垛）。界面设计遵循简洁原则，只保留必要的状态指示和功能按钮。

**界面结构原则**：2号机盒箱垛软件进入即呈现多Tab页面效果，默认高亮选中「数据采集」作为主界面。除系统设置复用已有弹窗交互（通过菜单/工具栏以模态对话框打开）外，其他功能入口均采用Tab交互形式。

## 界面布局设计

### 多Tab整体结构

- **进入效果**：软件启动后即呈现多Tab页面，Tab栏位于菜单栏下方
- **Tab 显示控制**：各 Tab 的显示/隐藏由**系统设置→页面配置**控制，每页面对应 Switch 开关；选上则显示、取消则隐藏。数据采集为主界面，必选不可关闭。
- **Tab形式**（默认全开）：数据采集、手工采集、数据查询、数据替换、生产统计、码包管理、取消关联、数据上传，共8个Tab
- **数据采集Tab**：默认选中并高亮，作为主界面，展示盒箱垛关联数据采集内容
- **手工采集Tab**：点击切换展示手工采集页面（P02-02）
- **数据查询Tab**：点击切换展示数据查询页面（P02-04）
- **数据替换Tab**：点击切换展示数据替换页面（P02-05）
- **生产统计Tab**：点击切换展示生产统计页面（P02-07）
- **码包管理Tab**：点击切换展示码包管理页面（P02-03）
- **取消关联Tab**：点击切换展示取消关联页面（P02-06）
- **数据上传Tab**：点击切换展示数据上传页面（P02-08）
- **弹窗类功能**：仅系统设置，通过菜单「配置」按钮以模态对话框打开，沿用已有弹窗交互
- **Tab切换**：点击不同Tab时，主内容区展示对应页面，工具栏根据当前Tab展示该Tab相关的功能入口

### 整体布局

- **布局方式**：桌面应用风格 + 菜单栏 + Tab栏 + 左中右三栏布局
- **适配要求**：工控机鼠标键盘操作优化，支持快速操作
- **主要区域**：
  - 菜单栏：文件、配置、数据、帮助
  - Tab栏：数据采集（主界面，默认选中高亮）、手工采集、数据查询、数据替换、生产统计、码包管理、取消关联、数据上传
  - **左侧栏**：上面当前生产信息、采集规格设置；下面实时上传数据区（成垛后待上传/上传状态展示）
  - **中间栏**：数据接收区、操作日志、报警信息等信息展示区域，按块分块显示
  - **右侧栏**：上面单位实时统计，下面任务控制
  - 状态栏：系统状态、时间显示等信息

### 布局优先级（上优先）

- **上半屏（优先展示/操作）**：产品选择与生产单号、开始/停止采集、强制操作、关键计数
- **下半屏（次要但可追溯）**：中间栏三块（见下文「中间栏设计」），均倒序显示最新在上

### 菜单栏设计

- **文件(F)**：
  - 退出软件：点击退出时，若存在未满垛数据，弹窗「存在未满垛(X箱)，是否暂存？」；按钮为「取消」「强制满垛」「暂存」。暂存：存库标记未成垛后允许退出；强制满垛：执行强制满垛流程后允许退出；取消：返回主界面。
- **配置**：
  - 系统设置（需要密码保护）
- **数据**：
  - 码包管理
  - 数据查询
  - 数据替换
  - 取消关联
  - 生产统计
  - 数据上传
  - 操作日志
- **帮助(H)**：
  - 操作帮助：点击弹出操作帮助对话框（P03-01），展示系统操作指南、功能说明、常见问题等内容
  - 关于系统

- **数据查询**：点击切换至数据查询页面（P02-04）

### 顶部工具栏设计

#### 弹窗类功能按钮（仅系统设置）

- **系统设置**：点击弹出系统设置对话框，支持设备配置、参数设置（需要密码保护），复用已有弹窗交互

### 左侧栏设计（宽度：约340px）

- **功能定位**：左侧上下两个卡片。上卡片为当前生产信息（含采集规格设置）；下卡片为实时上传数据区。

#### 模块一：当前生产信息（上卡片）

- **生产产品选择**：下拉框，点击弹出产品选择弹窗；选中后回显产品名称、产品编号（只读）；开始生产后不可更改。
- **生产单号**：格式 YYYYMMDD+序号，可编辑，高度40px；默认当天日期+001。
- **采集规格设置**（卡片内区域）：第一行「1垛 [70]箱」，第二行「1箱 [4]盒」；70、4为可编辑输入框，默认值70、4（石湾产线）；开始采集后不可修改。箱数达到预设值即满垛，自动上传。

#### 模块二：实时上传数据区（下卡片）

- **功能**：展示成垛后待上传/上传状态（当前待上传箱数）。
- **显示**：列表或简要信息，支持「待上传」等状态标识；具体样式按实现调整。

### 中间栏设计（宽度：自适应，最小600px）

中间栏**自上而下固定分三块**，高度比例约为 **60% : 20% : 20%**，块间用分隔线或留白区分，便于区分扫码数据、操作记录与报警。

#### 模块一：数据接收区（上，约60%高）

- **功能定位**：展示扫码与关联数据，为主要信息区
- **区域样式**：白色背景，边框分割，可滚动
- **显示内容**：
  - 盒码/箱码/垛码等扫码数据及关联结果
  - 装箱完成、满箱/满垛提示等实时业务信息
- **显示格式**：时间戳（HH:MM:SS） + 内容，18px 字体
- **显示特性**：倒序显示，最新在上，自动滚动到顶部

#### 模块二：操作日志（中，约20%）

- **功能定位**：记录用户操作与系统关键动作
- **区域样式**：白色背景，边框分割，可滚动
- **显示内容**：设备在线状态、开始/停止采集、产品选择、强制满垛、提取工单未成垛等操作记录及关键状态
- **显示格式**：时间戳（HH:MM:SS） + 内容，16px 字体
- **显示特性**：倒序显示，最新在上，自动滚动到顶部

#### 模块三：报警信息（下，约20%）

- **功能定位**：集中展示报警与异常信息
- **区域样式**：白色背景，边框分割，可滚动
- **显示内容**：
  - 重码、未成功采集、关联失败等异常
  - 其他报错与报警信息
- **报警描述规则**：**必须描述具体原因**，便于工人快速定位，示例：
  - 小标不通过 → 显示「瓶码 xxx 不在小标码包范围内」
  - 中标不通过 → 显示「盒码 xxx 不在中标码包范围内」
  - 大标不通过 → 显示「箱码 xxx 不在大标码包范围内」
  - 重码 → 显示「瓶码/盒码/箱码 xxx 重复出现」
  - 错码 → 显示「瓶码/盒码/箱码 xxx 格式错误」或「xxx 不在码包范围内」
  - 超码/缺码 → 显示「盒码 xxx 多/少 x 个瓶码」
  - 盒箱校验不通过 → 显示「箱码 xxx 盒数不符」
- **级别标识**：警告级别黄色背景、严重级别红色背景
- **显示格式**：时间戳 + 具体异常描述（含层级、码值、原因），18px 字体
- **显示特性**：倒序显示，最新在上，初始可为空，有报警时动态追加

**交互**：上述三块均倒序显示最新在上。

### 右侧栏设计（宽度：约336px）

- **功能定位**：上面单位实时统计，下面任务控制。

#### 模块一：单位实时统计（上）

- **布局**：文字在上、数字在下；每项独立白底卡片
  - **当前箱数、每垛箱数**：文字在上、数字在下，两卡片横向并排；当前箱数红色、每垛箱数蓝色，大字号
  - **当前盒数、每箱盒数**：文字在上、数字在下，两卡片横向并排；大字号显示
  - **已生产垛数、总剔除数**：单独一行，文字在上、数字在下，样式较简洁；总剔除数40px大字号、加粗、红色
- **更新机制**：基于采集数据实时计算和更新。

#### 模块二：任务控制（下）

- **功能定位**：提供生产任务执行过程中的核心控制操作
- **布局设计**：按钮集中排列，**一行两个**，按钮间距12px
- **按钮列表**：开始/停止采集、无需采集码、关闭报警、强制满垛、提取工单未成垛、收回剔除、剔除数清零
- **帮助提示**：标题旁显示圆圈问号图标 `(?)`，点击弹出该区域所有按钮的功能说明
- **视觉样式**：纯色背景、立体阴影效果、鼠标键盘操作优化

##### 主要控制功能

- **生产开始&关闭**：
  - **开始采集按钮**：
    - 功能：点击"开始采集"开始生产
    - 操作流程：
      1. 点击"开始采集"按钮
      2. 系统检测各个硬件是否在线（相机、剔除装置等）
      3. 检测通过后弹出二次确认弹窗，展示生产信息（产品、生产单号）和包装比例（1垛X箱、1箱X盒）
      4. 用户确认后开始采集工作
      5. 按钮文字变更为"停止采集"
    - 样式：主要操作按钮，蓝色背景，白色加粗文字，高度40px，宽度120px
  - **停止采集按钮**：
    - 功能：再次点击按钮停止采集
    - 操作逻辑：
      - 当点击前有未满垛的数据，这些数据会保留，不会自动组垛（后续可以提取工单未成垛）
      - 停止采集后，按钮文字变更为"开始采集"
  - **无需采集码开关**：
    - 功能：当天不生产五码合一产品，开启后，读码器等设备不进行工作（其实软件都不用开，工厂要求，什么数据都不采集）
    - 样式：开关按钮，高度40px
    - 操作：开启后，所有读码设备不工作，不采集任何数据
  - **关闭报警**：
    - 功能：报警由**自动上传异常**触发；点击「关闭报警」用于**重置**报警（停止声光报警器、蜂鸣器等）
    - 样式：标准操作按钮，高度40px

##### 强制操作功能

- **强制满垛**：
  - 功能：未达到设定箱数，也强制结束当前垛，生产垛码
  - 样式：标准操作按钮，橙色背景，白色加粗文字，高度40px，宽度120px
  - 操作流程：点击按钮 → 确认对话框 → 忽略预设要求 → 生成虚拟垛标 → 重置计数
- **提取工单未成垛**：
  - 功能：当软件关闭前的尾数板没有进行强制满垛操作时，可以通过此按钮提取"未完成垛"的数据，隔天继续生产
  - 样式：标准操作按钮，蓝色背景，白色加粗文字，高度40px，宽度150px
  - 操作流程：点击按钮 → 弹窗打开，内有输入框（支持手输或扫码）→ 输入/扫描垛的其中一箱的箱码 → 系统查出对应生产订单数据并显示 → 确认 → 信息回显到主页面，可继续生产

##### 剔除相关功能

- **收回剔除**：
  - 功能：当剔除设备进行剔除动作后没有自动收回时，点击此按钮可以让剔除设备收回当前剔除动作（PLC控制，对应系统设置设备）
  - 样式：标准操作按钮，橙色背景，白色加粗文字，高度40px，宽度120px
- **剔除数清零**：
  - 功能：将总剔除数重置为 0（总剔除数显示在单位实时统计区域）
  - 样式：标准操作按钮，灰色背景，白色加粗文字，高度40px，宽度约100px

##### 运行状态监控

- **说明**：检查瓶盒关联、禁用剔除等监控功能不在此处显示，相关信息在中间栏（数据接收区/操作日志）自动显示，无需手动操作按钮

### 帮助提示设计

#### 问号图标样式

- **图标样式**：简洁的圆圈问号 `(?)`，浅灰色
- **图标尺寸**：14-16px高度，与标题文字对齐
- **图标位置**：紧跟在「任务控制」标题后面，间距4-6px
- **鼠标交互**：悬浮时图标颜色变深，显示手型光标

#### 帮助提示触发与显示

- **触发方式**：点击问号图标 `(?)`
- **显示方式**：弹出提示对话框或浮层
- **提示框样式**：白色背景、1px浅灰边框、轻微阴影、黑色文字14-16px、内边距12-16px、最大宽度350px
- **消失时机**：点击提示框外部区域或按ESC键关闭
- **内容格式**：以列表形式展示任务控制区域所有按钮的功能说明

#### 任务控制帮助提示内容

- **开始/停止采集**：点击开始后系统检测硬件设备，检测通过后开始采集工作；再次点击停止采集，未满垛数据会保留
- **无需采集码**：当天不生产五码合一产品时开启，所有读码设备不工作，不采集任何数据
- **关闭报警**：当自动上传异常触发报警时，点击此按钮可重置报警状态（停止声光报警器、蜂鸣器等）
- **强制满垛**：未达到设定箱数也强制结束当前垛，生成虚拟垛标并重置计数
- **提取工单未成垛**：弹窗输入/扫描垛内任一箱码，查出对应生产订单并显示，确认后回显主页面继续生产
- **收回剔除**：当剔除设备进行剔除动作后没有自动收回时，点击此按钮让剔除设备收回当前剔除动作
- **剔除数清零**：将总剔除数重置为0（总剔除数显示在单位实时统计区域）

## 数据采集功能

### 实时数据采集展示

- **采集频率**：每2秒采集1-2个产品
- **数据格式**：
  - 盒码格式：20241201001001等
  - 箱码格式：20241201001002等
  - 垛码格式：P20241201001等
- **时间精度**：精确到毫秒（2024-12-01 09:15:23.456）
- **质量检测**：98%合格率，2%不合格率
- **实时更新**：采集数据以日志条目形式实时显示

### 四级关联采集流程

- **关联类型**：四级关联采集模式（瓶-盒-箱-垛）
- **采集规则**：
  - 第一层（瓶码）：通过2台瓶码采集相机交叉去重采集（1号机负责）
  - 第二层（盒码）：每计数到G个瓶码，采集关联1个盒码（1号机负责，写入共用库供2号机读取）
  - 第三层（箱码）：每计数到N个盒码，采集关联1个箱码（2号机负责）
  - 第四层（垛码）：每计数到M个箱码，关联1个垛码（2号机负责）
- **关联成功率**：99%成功，1%失败
- **异常处理**：关联失败时自动报警
- **数据展示**：在数据接收区以日志形式显示关联结果

### 统计计算

- **箱数统计**：基于采集数据自动累计
- **垛数统计**：按采集规格自动形成垛

## 1号机数据获取

**说明**：1号机与2号机共用2号机数据库，1号机瓶盒关联数据直接写入该库，2号机从同一库读取，无需额外同步机制。

- **数据来源**：1号机瓶盒关联数据写入共用数据库后，2号机实时读取
- **数据内容**：瓶盒关联关系、采集时间戳、剔除标记（若有）
- **状态显示**：在数据接收区或操作日志中显示数据接收状态

## 自动上传机制

### 成垛后自动上传

- **配置依赖**：系统自动上传行为**根据系统设置→业务→上传配置**中的「自动上传开关」执行；默认开启。
- **触发时机**：当自动上传开关开启时，成垛后系统自动上传数据至米多大数据引擎；关闭时成垛后不自动上传，需手动上传。
- **上传内容**：
  - 垛码信息
  - 箱码关联关系
  - 盒码关联关系
  - 瓶码关联关系（通过1号机数据）
  - 任务信息
- **上传状态**：在数据接收区显示上传状态
- **状态反馈**：
  - 上传成功：绿灯常亮，无蜂鸣/提示音；**一分钟后自动熄灭**
  - 上传失败：红灯常亮 + 蜂鸣；报警信息区显示**具体失败原因**（具体到箱/盒/瓶）

### 指示灯与蜂鸣控制

- **上传成功**：绿灯常亮，无蜂鸣/提示音；**一分钟后自动熄灭**
- **上传失败**：红灯常亮 + 蜂鸣；需人工点击「关闭报警」后停止
- **待上传**：不亮、不响
- **关闭报警后**：不亮、不响
- **关闭报警**：上传失败时报警器红灯亮且蜂鸣，点击主界面「关闭报警」按钮可重置（不亮不响）

## 报警系统

### 报警类型说明

- **短报警**：短暂提示音/闪烁，**自动关闭**，不需人工操作
- **长报警**：持续提示音/闪烁，需人工点击「关闭报警」按钮关闭

### 报警触发条件（2号机）

| 触发条件 | 报警类型 |
| -------- | -------- |
| 连续重码 3 次 | 长报警 |
| 上传异常（数据上传失败） | 长报警 |


### 报警显示

- **视觉提示**：不同颜色区分报警级别
- **声音反馈**：通过报警设备（声光报警器、蜂鸣器等）发出声音提醒
- **日志记录**：所有报警自动记录到操作日志

### 报警控制

- **关闭报警**：位于采集控制块。用于关闭**长报警**（如连续重码3次、上传异常等）；点击「关闭报警」可重置报警（停止声光报警器、蜂鸣器等）
- **指令发送**：
  - 向报警设备发送停止指令（如声光报警器、蜂鸣器等）
  - 支持多种设备通信协议（串口RS232/RS485、网络TCP/IP）
  - 指令格式：根据设备协议定义（如Modbus、自定义协议等）
- **状态同步**：
  - 报警设备状态与界面显示同步
  - 确认报警已成功关闭后更新界面状态
  - 失败时显示错误提示并记录日志
