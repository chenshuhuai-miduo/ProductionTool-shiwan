# P02-05 - 数据替换

## 子功能名称

数据替换 - 码替换功能

## 功能概述

为生产管理人员和维护人员提供码替换功能界面，采用桌面应用风格设计，与系统整体风格保持一致。界面提供完整的码替换操作功能，支持原码值和新码值的输入验证，适合处理码损坏、码错误等异常情况。界面包含码替换表单和确认对话框。

**重要说明**：本功能参考已有功能（功能名称为"码替换"）。用户只需输入外码（盖外码/瓶码、盒码、箱外码）进行替换，无需单独操作内码；若该码已上传至云端且对应类型有内码（如盖码、箱码），系统将**自动同步替换云端内码**。

## 交互功能设计

### 码替换流程

1. **输入原码**：
  - 手动输入或扫码输入原码（待替换的码）
  - 实时验证码值格式
  - 查询原码是否存在
2. **输入新码**：
  - 手动输入或扫码输入新码（替换后的码）
  - 实时验证码值格式
  - 验证新码的有效性和唯一性
3. **输入替换原因**（可选）：
  - 输入替换原因，用于记录操作日志
4. **确认替换**：
  - 点击"确认替换"按钮
  - 弹出确认对话框（含密码输入）
  - 输入密码
  - 密码验证通过后确认执行替换操作
5. **替换执行**：
  - 检查码关联是否已上传
  - 未上传：校验通过后直接替换本地数据
  - 已上传：先调用云端替换接口成功，再更新本地数据；若码类型有内码（盖、箱等），同步替换云端内码
  - 记录操作日志
  - 显示替换结果

### 验证规则

- **原码验证**：码层级对应关系、格式/位数（错码）定义详见 **AI 文档 10.1.7 码校验通用规则**，以下为数据替换场景的特定规则：
  - 必填验证：原码不能为空
  - **定位与可替换性**：在关联表中须能**唯一**命中瓶/盒/箱外码之一（同一字符串不得同时出现在多列）；**无任何记录**时视为未建立关联。
  - **格式/位数（错码）**（分情形）：
    - **库中已唯一命中某一层**：按**该层**配置的位数/格式校验（错码）。
    - **库中无记录**：尚不能定层，仅能用「长度是否与已启用的瓶/盒/箱位数之一相符」做筛查——与**各启用层均不符**时按**错码**提示；否则再提示**不存在关联**。
- **新码验证**：码层级对应关系、格式/位数（错码）及码包范围（小标/中标/大标不通过）定义详见 **AI 文档 10.1.7 码校验通用规则**，以下为数据替换场景的特定规则：
  - 必填验证：新码不能为空
  - 格式验证：码值格式/位数必须符合规范（即不触发错码）
  - **层级一致性验证**：新码必须与原码属于同一层级（瓶码只能替换瓶码，盒码只能替换盒码，箱码只能替换箱码）
  - **码包范围验证**：新码必须在对应层级的码包范围内（参见通用规则码层级对应表）。**冷热分离**：只查**热表**（热表中的码天然未使用）
  - **使用状态验证**：新码未被使用过，系统中不存在该码的关联关系（不能是已采集关联的码）。冷热分离时热表即未使用码，与码包范围验证合并
  - 不能与原码相同：新码不能与原码相同

### 替换规则

- **上传状态检查**：执行替换前先检查码关联关系是否已上传。
  - **未上传（仅本地）**：码关联只存在于本地数据库时，执行码有效性等规则校验后**直接替换本地数据**。
  - **已上传**：码关联已上传至云端时，须**先调用云端替换接口成功，再更新本地数据**，保证云端与本地一致。
  - **已上传且码类型有内码**（如盖码、箱码）：替换时需**同步替换云端内码**。
- **二次确认与密码**：
  - 替换操作需弹窗二次确认
  - 确认时需输入密码，验证通过后方可执行
- **用户输入只涉及外码**：
  - 盖外码（瓶码）：可以替换
  - 盒码：可以替换
  - 箱外码（箱码）：可以替换
  - 内码：用户无需单独操作，由系统自动处理（仅已上传且码类型有内码时，自动同步替换云端内码）
- **关联关系更新**：
  - 替换后自动更新关联关系
  - 保持原有的层级关联关系
- **操作日志记录**：
  - 使用项目已有的统一日志功能记录所有替换操作
  - 记录内容：操作人员、操作时间、原码值、新码值、替换原因、操作结果等
  - 日志格式与系统其他功能保持一致，便于统一查询和追溯

### 校验失败与替换失败提示

以下为用户可见**建议文案**；**格式/位数、错码**类与 **AI 文档 10.1.7 码校验通用规则**及系统统一提示保持一致时，可直接沿用通用模板，不强制与下表字面完全一致。

| 阶段 | 场景 | 建议提示文案 |
|------|------|----------------|
| 原码 | 未填写 | 请输入原码 |
| 原码 | 格式/位数不合规（错码） | （同系统错码/格式校验通用提示） |
| 原码 | 系统中不存在或未建立关联 | 原码不存在关联，无需或无法替换 |
| 新码 | 未填写 | 请输入新码 |
| 新码 | 格式/位数不合规（错码） | （同系统错码/格式校验通用提示） |
| 新码 | 与被替换原码层级不一致 | 新码必须与原码为同一层级（瓶/盒/箱） |
| 新码 | 不在当前层级码包（热表）范围内 | 新码不在码包范围内，请确认已导入对应码包 |
| 新码 | 已有关联或已被使用 | 新码已存在关联，无法作为替换目标 |
| 新码 | 与原码相同 | 新码不能与原码相同 |
| 确认 | 密码错误或未输入 | 密码错误或请输入密码（以实现为准，可合并为一句） |
| 执行 | 已上传云端，云端替换接口失败 | 云端替换失败，本地未修改（失败原因：具体原因 ← 可拼接接口返回） |
| 执行 | 网络超时、连接失败等 | 网络异常，替换失败（本地未修改）← 或与云端失败合并展示接口/异常信息 |

**说明**：

- 同一操作触发多个校验失败时，**优先展示第一条未通过项**，避免一次刷屏多条。
- **顺序建议**：
  1. **入口**：原码必填 → 新码必填 → 新码≠原码。
  2. **原码侧**：先根据关联表判断**记录条数与能否唯一定位层级**（同一原码命中瓶/盒/箱多列 → 直接提示不可替换）。**无记录**时：若已启用位数校验且码长与瓶/盒/箱**各启用层均不符** → 提示**错码**；否则 → 提示**不存在关联**。**唯一命中**后 → 再按**该层**做**错码（位数/格式）**校验。
  3. **新码侧**：层级一致 → 按该层错码（位数/格式）→ 码包范围 → 使用状态（是否已占用）。
