# P01-01 - 主界面-瓶盒关联

## 子功能名称

主界面-瓶盒关联数据采集

## 功能概述

作为1号机瓶盒关联软件的主界面，为生产线操作员提供瓶盒关联数据采集的控制操作界面。界面采用Tab结构，Tab栏有「数据采集」一个Tab，默认选中高亮。主要功能：采集规格设置（1盒6瓶）、开始/停止采集、数据接收区展示瓶码/盒码、瓶码校验（码包范围、重码、错码）、异常标记剔除。无效不计数、不建立关联、写日志、标记剔除，产线正常移动，剔除标记写入共用库供2号机读取；剔除装置在2号机盒箱垛关联之后，按箱剔除。

## 界面布局设计

### 整体布局

- **布局方式**：桌面应用风格 + 菜单栏 + Tab栏 + 左右两栏布局。
- **适配要求**：工控机鼠标键盘操作优化，支持快速操作。
- **主要区域**：
  - 菜单栏：文件、配置、数据、帮助等功能菜单（配置菜单下有系统设置子菜单）。
  - **Tab栏**：数据采集，默认选中高亮。（Tab 显示受**系统设置→页面配置**控制，1号机当前仅数据采集一页，主界面必选不可关闭）
  - **左侧栏**：采集规格设置（1盒 [6]瓶，6可修改）+ 单位实时统计 + 任务控制。
  - **右侧栏**：数据接收区 + 操作日志。
  - 状态栏：遵循 通用规则文档「02-用户界面设计」之状态栏设计

### 布局优先级（上优先）

- **上半屏（优先展示/操作）**：采集规格、开始/停止采集、单位实时统计、数据接收区
- **下半屏（次要但可追溯）**：操作日志（含设备连接状态）

**视觉说明**：Tab栏位于菜单栏下方，含「数据采集」Tab；左右两栏布局，左侧采集规格（1盒 [6]瓶，6可修改、开始采集后不可改）+ 单位实时统计（文字在上、数字在下，每项独立卡片）+ 任务控制，右侧数据接收区 + 操作日志；单位实时统计每项为白底卡片、数字大字号突出（合计瓶数红色、当前盒数蓝色）。**数据接收区**：瓶1读码与瓶2读码几乎同时；关联成功显示「盒码 xxx 关联成功」（xxx 为盒读码）。

### Tab栏设计

- **数据采集**：主界面Tab，默认选中高亮，展示瓶盒关联数据采集内容。

### 菜单栏设计

- **文件(F)**：
  - 退出软件：点击退出时，若正在采集中则提示「请先停止采集」，停止后再进行常规退出确认。
- **配置**：
  - 系统设置：点击弹出系统设置对话框，支持设备配置、参数设置（需要密码保护）
- **数据**：
  - 操作日志
- **帮助(H)**：
  - 操作帮助：点击弹出操作帮助对话框（P03-01），展示系统操作指南、功能说明、常见问题等内容
  - 关于系统

### 左侧栏设计（宽度：约500px）

自上而下分块：采集规格设置、单位实时统计、任务控制；块间用分割线区分。

#### 模块一：采集规格设置（上）

- **格式**：一行「1盒 [6]瓶」，6为可编辑输入框，默认值6（石湾产线）。
- **输入框校验规则**：不能为空；仅允许输入整数；范围 1～999；不符合时提示并阻止开始采集。
- **说明**：开始采集后不可修改。

#### 模块二：单位实时统计（中）

- **布局**：文字在上、数字在下；每项独立白底卡片，横向排列。
- **合计瓶数**：标签在上、数值在下，数值 44px 加粗、红色，实时更新。
- **当前盒数**：标签在上、数值在下，数值 44px 加粗、蓝色，实时更新。
- **说明**：不展示关联计数。

#### 模块三：任务控制（下）

- **标题行**：「任务控制」标题右侧有帮助提示按钮（?），点击弹出任务控制功能说明。
- **布局**：按钮一行两个并排占满宽度。
- **帮助提示内容**：
  - 开始采集：系统检测设备后开始采集；再次点击停止。
  - 关闭报警：长报警时点击可重置三色声光报警灯（停止灯光和蜂鸣）。
- **开始/停止采集**：主按钮，蓝色背景，高度 40px。
- **关闭报警**：红色背景按钮，高度 40px；用于关闭长报警，点击后重置三色声光报警灯（停止灯光和蜂鸣）。

### 右侧栏设计（宽度：自适应，最小600px）

自上而下分块：数据接收区、操作日志。

#### 模块一：数据接收区（上，约60%高）

- **显示内容**：盒码、瓶码及关联结果统一展示，不细分区域；含盒读码、瓶1/瓶2读码、关联成功/失败等。
- **显示格式**：时间 + 具体信息，每行一条，如 `10:30:18 盒读码：20241201000001`、`10:30:18 盒码 20241201000001 关联成功`。
- **显示特性**：倒序显示，最新在上，可滚动；关联成功绿色、关联失败红色；正常码黑字，重码/错码红色高亮。

#### 模块二：操作日志（下，约40%）

- **功能**：记录用户操作、系统关键动作、**设备连接状态**（瓶1/瓶2/盒读码器，已连接✓/未连接✗）；设备状态在此区域显示，不在任务控制区展示。
- **显示格式**：时间戳 + 内容，倒序显示。

### 数据接收区通用规则

- **显示顺序**：最新数据在顶部（索引0），旧数据在下，新数据添加后自动滚动到顶部
- **数量限制**：最多保留1000条，超过时删除列表末尾最旧的数据
- **不清空**：下一组采集时保留历史记录，不自动清空

### 数据接收区详细说明

- **显示格式**：时间 + 具体信息，每行一条，如 `10:30:18 盒读码：20241201000001`、`10:30:18 盒码 20241201000001 关联成功`、`10:30:17 瓶1读码：xxx、xxx、xxx`、`10:30:17 瓶2读码：xxx、xxx、xxx`（瓶1/瓶2读码几乎同时）；关联成功绿色、关联失败红色；正常码黑字，重码/错码红色高亮。

### 界面视觉规范

- **两栏布局**：左右分栏，卡片式分区，每项为白底卡片。
- **对齐**：单位实时统计为文字在上、数字在下；其他标签与数值同一行时左右分布，垂直居中。
- **字号**：采集规格输入框内数字 26px；计数数值 44px 加粗；卡片标题 15～16px 加粗；设备状态 14px。

## 数据采集功能

### 瓶盒关联采集流程

- **关联类型**：二级关联采集模式（盒码关联瓶码）
- **流程**：
  1. **瓶码采集**：两台相机分别读取一组6瓶，瓶1读码与瓶2读码几乎同时；可能读到的数量不同（如瓶1读4个、瓶2读6个）；数据汇合后交叉去重，得到6个码传给下一工序；输出`瓶1读码：xxx、xxx、xxx`和`瓶2读码：xxx、xxx、xxx`
  2. **瓶码校验**：判断每个瓶码是否在码包范围内；检测重码、错码等异常，标记为剔除
  3. **盒码关联**：产线移动后盒读码相机读码，系统关联6瓶码+1盒码，输出`盒码 xxx 关联成功`（xxx 为盒读码）
  4. **计数更新**：有效码计数+1；无效不计数、不建立关联、写日志、标记剔除；新数据添加到数据接收区顶部（遵循数据接收区通用规则）

### 瓶码校验与剔除

- **校验规则**：
  - 判断瓶码是否在码包范围内，不在范围内视为错码
  - 检测重码（同一码重复出现）、错码（格式错误、不在码包范围等）、超码（实际码个数比预设比例多）、缺码（实际码个数比预设比例少）
- **异常标记**：重码、错码等异常在界面用红色高亮标记，并标记为剔除
- **1号机处理**：无效不计数、不建立关联、写日志、标记剔除，产线正常移动，剔除标记写入共用库供2号机读取
- **剔除执行**：剔除装置位于2号机盒箱垛关联之后，产线到达该处时**按箱剔除**（非1号机执行，由下游剔除装置执行）

### 统计与异常

- **统计**：仅有效码计数；无效不计数、不建立关联、写日志、标记剔除；界面仅展示「合计瓶数」「当前盒数」，不展示关联计数。

## 交互功能设计

### 开始采集前初始化检测

点击「开始采集」时，系统按以下项目检测，**全部通过**后方可开始采集：

| 类别 | 检测项 | 不通过时 |
|------|--------|----------|
| **门禁** | 码包已导入（盖外码小标、盒外码中标） | 提示导入码包，阻止开始 |
| **规格** | 采集规格已确认（1盒 [6]瓶，开始后不可修改） | 无（可修改后继续） |
| **设备** | 读码器在线（瓶1读码器、瓶2读码器、盒读码器） | 操作日志显示未连接，阻止开始 |
| **连接** | 数据库可访问（1号机访问2号机库） | 提示连接失败，阻止开始 |

### 采集控制

- **开始**：设置每盒瓶数（1盒 [6]瓶，6可修改）→ 点击开始采集 → 执行初始化检测 → 全部通过后开始工作
- **停止**：点击停止采集；数据不齐时在信息区提示

### 设备状态监控

- 操作日志区域显示瓶1/瓶2/盒读码器连接状态（已连接绿色/未连接红色），不在任务控制区展示

### 状态反馈

- 操作成功绿色、失败红色

## 数据写入共用库

**说明**：1号机与2号机共用2号机数据库，1号机瓶盒关联数据直接写入该库，2号机从同一库读取，无需额外同步机制。

- **写入时机**：瓶盒关联成功后实时写入共用数据库
- **写入内容**：瓶码、盒码、瓶盒关联关系、采集时间戳、任务信息、**剔除标记**（若有）
- **说明**：剔除标记写入共用库后2号机可读取，经2号机盒箱垛关联后，由下游剔除装置按箱剔除

## 响应式设计

- **1024x768**：左侧栏约 340px，右侧栏最小 500px，字体适当缩小。
- **1366x768及以上**：左侧栏约 **500px**，右侧栏最小 **600px**；单位实时统计文字在上数字在下、任务控制分割线区分、数字大字号保持。

## 性能要求

- **操作响应时间**：≤ 500ms
- **数据更新频率**：每2秒
- **界面刷新率**：≥ 30fps
- **对话框响应**：≤ 100ms
- **对话框打开速度**：≤ 300ms
- **对话框关闭速度**：≤ 200ms
- **数据写入延迟**：瓶盒关联成功后写入共用库延迟 ≤ 100ms

## 兼容性要求

- **输入设备**：完全支持鼠标键盘操作
- **工控机适配**：针对工控机环境优化
- **分辨率支持**：1024x768 ~ 1920x1080
- **连续运行**：支持24小时连续运行

## 页面数据展示逻辑

### 码状态标识

- **合格**：绿色标识
- **重码/错码/不在码包范围**：红色标识，标记剔除
- **异常描述**：数据接收区、操作日志中的异常须**描述具体原因**，直接告知码值，如「瓶码 xxx 重复出现」「瓶码 xxx 不在码包范围内」「盒码 xxx 多/少 x 个瓶码」等

### 关联结果展示

- **成功**：数据接收区显示`盒读码：xxx`、`盒码 xxx 关联成功`（xxx 为盒读码），绿色；更新计数
- **失败**：红色错误提示

## 报警规则（1号机）

- **短报警**：短暂提示音/闪烁，**自动关闭**，不需人工操作
- **长报警**：持续提示音/闪烁，需人工点击「关闭报警」按钮关闭

| 触发条件 | 报警类型 |
| -------- | -------- |
| 重码/错码/超码/缺码等异常 | 短报警 |
| 连续重码 3 次 | 长报警 |

> **连续重码3次定义**：在软件本次启动生命周期内，采集到的某一码（瓶码/盒码）连续出现重码（即同一码在系统中已存在）且次数累计达到3次，触发长报警。退出软件后重新启动，计数重置，重新判断。

## 功能边界

### 包含的功能

1. 瓶盒关联数据采集（含瓶码校验、码包范围判断、重码/错码检测）
2. 异常标记（无效不计数、不建立关联、写日志、标记剔除，产线正常移动；剔除装置在2号机之后，按箱剔除）
3. 实时数据展示和统计
4. 数据写入共用库（含剔除标记，供2号机读取、下游按箱剔除）

### 适用场景

- **生产线瓶盒关联**：1号机负责的瓶盒关联数据采集
- **实时监控**：生产过程中的实时状态监控
- **异常处理**：采集异常情况的标记与提示
