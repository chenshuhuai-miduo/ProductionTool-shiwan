# P03-02 - 系统设置

## 子功能名称
系统设置 - 设备配置和参数设置

## 功能概述
为系统管理员提供系统配置功能界面，**1号机和2号机共用**。通过菜单「配置」→「系统设置」以模态对话框打开，需密码验证。数据库部署在2号机，1号机通过局域网访问，因此**数据库连接统一配置2号机所在IP及数据库信息**。

**重要说明**：本页面需要密码保护，访问时弹窗提示输入密码，密码验证成功后允许操作。每次访问都需要重新验证（不保存登录状态）。

## 实现方式：复用现有系统设置

### 布局结构示意
```
┌─────────────────────────────────────────────────────────────────────────┐
│  系统设置                                                           [×] │
├─────────────────────────────────────────────────────────────────────────┤
│  [IO设备管理] [打印机管理] [报警设置] [基础设置]                          │
├─────────────────────────────────────────────────────────────────────────┤
│  基础设置选中时，下方显示二级 Tab：                                       │
│  [数据库连接] [码位数配置] [虚拟垛标规则]                                │
├─────────────────────────────────────────────────────────────────────────┤
│  虚拟垛标规则 Tab 内容示例：                                             │
│  前缀    [V____]   （可修改，默认 V）                                    │
│  时间    YYYYMMDD  （只读，生成时取当前日期）                            │
│  序号    001       （只读，自动递增，最小3位）                            │
│  产线号  [A____]   （可修改）                                            │
│                                                                         │
│  [保存]                                                                  │
└─────────────────────────────────────────────────────────────────────────┘
```

### 一级 Tab 结构
| 一级 Tab | 说明 |
|----------|------|
| IO设备管理 | 设备列表（添加/编辑/删除）、连接测试、设备状态监控；**现有功能，支持按需配置** |
| 打印机管理 | 打印机配置 |
| 报警设置 | 声音报警开关、报警延时、报警间隔、测试报警 |
| 基础设置（**新增**） | 二级 Tab：数据库连接、码位数配置、虚拟垛标规则 |

### 基础设置 - 二级 Tab 结构
| 二级 Tab | 说明 |
|----------|------|
| 数据库连接 | 统一配置2号机所在IP及数据库信息（1号机、2号机均连接该库） |
| 码位数配置 | 采集时校验码位数 |
| 虚拟垛标规则 | 成垛时生成虚拟垛标的格式规则；**1号机不生成垛标，用不上可不配置留空** |

### 数据库连接（基础设置→数据库连接）
- **统一标准**：1号机和2号机均配置**2号机所在机器的IP地址**及数据库信息。
- **表单项**：
  - **2号机IP地址**：数据库所在机器（2号机）的IP，本机部署时填 localhost 或 127.0.0.1
  - **数据库端口**：如 3306（MySQL）
  - **数据库名**：与1号机共用
  - **用户名/密码**：数据库访问凭证
  - **连接测试**：测试能否成功连接数据库

### 码位数配置（基础设置→码位数配置）
- **用途**：采集时校验码的位数是否符合要求。
- **表单项**：
  - **小标位数**：瓶码（盖外码）位数，如 14；-1 表示不校验
  - **中标位数**：盒码位数，如 14；-1 表示不校验
  - **大标位数**：箱码位数，如 6；-1 表示不校验
  - **校验逻辑**：采集时若实际采码位数与配置值不一致，则异常告警、关联失败
- **说明**：1号机不采箱码，大标可配置供参考或留空；用不上不配置留空即可。

### 虚拟垛标规则（基础设置→虚拟垛标规则）
- **用途**：2号机成垛时生成虚拟垛标的格式；**1号机不生成垛标，用不上可不配置留空**。
- **格式示例**：V20260226001A、V20260226002A（前缀+年月日+序号+产线号）
- **表单项**：

| 表单项 | 可编辑 | 说明 |
|--------|--------|------|
| 前缀 | 是 | 默认 V，可修改 |
| 时间 | 否 | 年月日 YYYYMMDD，生成时取当前日期，固定不可修改 |
| 序号 | 否 | 自动递增，最小 3 位如 001、002，不可修改 |
| 产线号 | 是 | 如 A、B，用于区分不同产线 |

### Tab 顺序
**一级 Tab**：1. IO设备管理  2. 打印机管理  3. 报警设置  4. 基础设置

**基础设置下二级 Tab**：1. 数据库连接  2. 码位数配置  3. 虚拟垛标规则

### 按钮说明
- 基础设置下各 Tab 内表单下方均保留**保存**按钮，跟在表单末尾，不设取消/重置按钮。

### 无需重复实现
- 设备配置表单、连接测试等交互逻辑
- 设备状态监控、底部状态栏等布局
- 打印机、报警等已有功能

## 密码保护机制（需新增）
- **访问控制**：打开系统设置页面时，首先弹出密码输入对话框
- **密码验证**：输入系统预设密码进行验证
- **验证失败**：密码错误时显示错误提示，不允许继续操作
- **验证成功**：密码验证成功后，允许加载系统设置页面
- **会话管理**：每次访问都需要重新验证，不保存登录状态

## 功能边界

### 包含的功能
1. 复用现有系统设置（IO设备管理、打印机管理、报警设置）
2. 密码保护
3. **基础设置**（新增一级 Tab，下含二级 Tab）：
   - **数据库连接**：统一配置2号机IP、端口、数据库名、用户名密码等
   - **码位数配置**：大标、中标、小标位数输入框；-1 表示不校验；用不上不配置留空
   - **虚拟垛标规则**：前缀、时间、序号、产线号；1号机用不上可不配置留空

### 适用场景
- **系统初始化**：首次部署时的系统配置
- **设备更换**：更换硬件设备时的配置调整
- **参数调整**：根据生产需求调整业务参数
- **故障排查**：设备故障时的配置检查和测试

## 入口说明
- **1号机**：菜单「配置」→「系统设置」，打开本系统设置弹窗
- **2号机**：菜单「配置」→「系统设置」，打开本系统设置弹窗
- 两个软件共用同一套系统设置界面，窗口标题统一为「系统设置」。
