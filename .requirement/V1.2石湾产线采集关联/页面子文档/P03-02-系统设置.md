# P03-02 - 系统设置

## 子功能名称
系统设置 - 设备配置和参数设置

## 功能概述
为系统管理员提供系统配置功能界面，**仅2号机（新版软件）使用**。通过菜单「配置」→「系统设置」以模态对话框打开，需密码验证。

**重要说明**：本页面需要密码保护，访问时弹窗提示输入密码，密码验证成功后允许操作。每次访问都需要重新验证（不保存登录状态）。

## 实现方式：复用现有系统设置

### 布局结构示意

│  业务：二级 Tab [码位数配置] [虚拟垛标规则] [上传配置]                    │
│  页面配置：展示所有页面列表，每行 Switch 开关，选上则对应 Tab 显示        │
│  设备：二级 Tab [IO设备管理] [打印机管理] [报警设置]                       │

### 一级 Tab 结构（按功能域分类，常用优先）
| 一级 Tab | 说明 |
|----------|------|
| **业务** | 采集、上传等业务参数；二级 Tab：码位数配置、虚拟垛标规则、上传配置；**常用，放首位** |
| **页面配置** | 软件主界面 Tab 栏的显示控制；展示所有页面列表，每行 Switch 开关，选上则对应 Tab 显示、取消则隐藏；**常用，放第二位** |
| **设备** | 硬件、外设相关；二级 Tab：IO设备管理、打印机管理、报警设置 |
| **连接** | 网络、数据源相关；二级 Tab：数据库连接（后续可扩展 API 地址、MQ 等） |

### 页面配置（一级 Tab，无二级 Tab）
- **用途**：控制软件主界面 Tab 栏显示哪些页面。Switch 开启则对应 Tab 显示，关闭则隐藏。

**可配置页面**（展示以下 8 项）：

| 页面名称 | 功能说明 | 默认 | 可关闭 |
|----------|----------|------|--------|
| 数据采集（盒箱垛） | 主界面，盒箱垛关联数据采集 | 开 | **否**（主界面必选不可隐藏） |
| 手工采集 | 扫码枪手工采集，相机损坏时替代 | 开 | 是 |
| 数据查询 | 查询采集关联数据 | 开 | 是 |
| 数据替换 | 码替换操作 | 开 | 是 |
| 生产统计 | 生产统计报表 | 开 | 是 |
| 码包管理 | 码包导入与管理 | 开 | 是 |
| 取消关联 | 单个/批量取消关联 | 开 | 是 |
| 数据上传 | 手动上传、上传状态查询 | 开 | 是 |

- **交互**：每行右侧 Switch 开关；开启=Tab 显示，关闭=Tab 隐藏。「数据采集（瓶盒）」/「数据采集（盒箱垛）」行 Switch 置灰不可操作。
- **Tab 顺序**：列表中勾选页面的顺序即为 Tab 栏显示顺序；未勾选的不显示。
- **持久化**：配置保存至配置文件，软件重启后生效。

### 设备 - 二级 Tab 结构
| 二级 Tab | 说明 |
|----------|------|
| IO设备管理 | 设备列表（添加/编辑/删除）、连接测试、设备状态监控；**现有功能，支持按需配置** |
| 打印机管理 | 打印机配置 |
| 报警设置 | 声音报警开关、报警延时、报警间隔、测试报警 |

### 连接 - 二级 Tab 结构
| 二级 Tab | 说明 |
|----------|------|
| 数据库连接 | 本机数据库（2号机自身）+ 1号机数据库连接信息（供内置定时轮询读取同步源）；后续可扩展 API 地址、MQ 等 |

### 业务 - 二级 Tab 结构
| 二级 Tab | 说明 |
|----------|------|
| 码位数配置 | 采集时校验码位数 |
| 虚拟垛标规则 | 成垛时生成虚拟垛标的格式规则；**仅2号机使用** |
| 上传配置 | 自动上传开关等；系统自动上传行为根据此配置执行；**仅2号机使用** |

### 数据库连接（连接→数据库连接）
配置两组数据库连接：

**本机数据库（2号机自身）**：MySQL
- **IP地址**：localhost 或 127.0.0.1
- **端口**：3306
- **数据库名**：2号机业务库名
- **用户名/密码**：数据库访问凭证
- **连接测试**：测试能否成功连接

**1号机数据库（同步源）**：内置定时轮询读取1号机数据时使用；1号机旧版软件不修改，轮询只读。
- **数据库类型**：SQL Server
- **IP地址**：1号机的局域网 IP
- **端口**：如 1433（SQL Server 默认）
- **数据库名**：TkCode
- **表名**：轮询读取的数据表名，默认 `T_Code`，可修改
- **用户名/密码**：访问凭证（只读权限即可）
- **连接测试**：测试能否成功连接

### 码位数配置（业务→码位数配置）
- **用途**：采集时校验码的位数是否符合要求。
- **表单项**：
  - **小标位数**：瓶码（盖外码）位数，如 14；-1 表示不校验
  - **中标位数**：盒码位数，如 14；-1 表示不校验
  - **大标位数**：箱码位数，如 6；-1 表示不校验
  - **校验逻辑**：采集时若实际采码位数与配置值不一致，则异常告警、关联失败

### 虚拟垛标规则（业务→虚拟垛标规则）
- **用途**：成垛时生成虚拟垛标的格式。
- **格式示例**：V20260226001A12、V20260226002A35（前缀+年月日+序号+产线号+随机数）
- **表单项**：

| 表单项 | 可编辑 | 说明 |
|--------|--------|------|
| 前缀 | 是 | 默认 V，可修改 |
| 时间 | 否 | 年月日 YYYYMMDD，生成时取当前日期，固定不可修改 |
| 序号 | 否 | 自动递增，最小 3 位如 001、002，不可修改 |
| 产线号 | 是 | 如 A、B，用于区分不同产线 |
| 随机数 | 否 | 2 位，系统自动生成，不可配置 |

- **序号持久化**：序号须持久化（如从数据库查询当天同产线号最大序号+1），确保软件重启后不会产生重复垛标；不可仅用内存计数，否则关闭软件再打开会从 001 重新开始导致重码。

### 上传配置（业务→上传配置）
- **用途**：控制成垛后是否自动上传数据。
- **表单项**：

| 表单项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| 自动上传 | switch开关 | 开启 | 开启时成垛后自动上传至米多大数据引擎；关闭时成垛后不自动上传，需手动上传 |

### Tab 顺序
**一级 Tab**（常用优先）：1. 业务  2. 页面配置  3. 设备  4. 连接

**业务下二级 Tab**：1. 码位数配置  2. 虚拟垛标规则  3. 上传配置

**页面配置**：无二级 Tab，直接展示页面列表 + Switch

**设备下二级 Tab**：1. IO设备管理  2. 打印机管理  3. 报警设置

**连接下二级 Tab**：1. 数据库连接（后续可扩展）

### 按钮说明
- 各 Tab 内表单下方均保留**保存**按钮，跟在表单末尾，不设取消/重置按钮。

### 无需重复实现
- 设备配置表单、连接测试等交互逻辑
- 设备状态监控、底部状态栏等布局
- 打印机、报警等已有功能

## 密码保护机制（需新增）
- **访问控制**：打开系统设置页面时，首先弹出密码输入对话框
- **密码验证**：输入系统预设密码进行验证
- **验证失败**：密码错误时显示错误提示，不允许继续操作
- **验证成功**：密码验证成功后，允许加载系统设置页面
- **会话管理**：每次访问都需要重新验证，不保存登录状态

## 功能边界

### 包含的功能
1. 密码保护
2. **业务**（一级 Tab，下含二级 Tab）：码位数配置、虚拟垛标规则、上传配置
   - **码位数配置**：大标、中标、小标位数输入框；-1 表示不校验；用不上不配置留空
   - **虚拟垛标规则**：前缀、时间、序号、产线号、随机数
   - **上传配置**：自动上传开关，默认开启
3. **页面配置**（一级 Tab，无二级 Tab）：展示所有页面列表，每行 Switch 开关；选上则对应 Tab 显示，取消则隐藏；数据采集必选不可关闭
4. **设备**（一级 Tab，下含二级 Tab）：IO设备管理、打印机管理、报警设置
5. **连接**（一级 Tab，下含二级 Tab）：数据库连接，分组配置本机数据库（2号机自身）和1号机数据库（轮询同步源）；后续可扩展 API 地址、MQ 等

### 适用场景
- **系统初始化**：首次部署时的系统配置
- **界面精简**：通过页面配置隐藏不常用 Tab，简化操作员界面
- **设备更换**：更换硬件设备时的配置调整
- **参数调整**：根据生产需求调整业务参数
- **故障排查**：设备故障时的配置检查和测试

## 入口说明
- **2号机**：菜单「配置」→「系统设置」，以模态对话框打开，窗口标题「系统设置」。
