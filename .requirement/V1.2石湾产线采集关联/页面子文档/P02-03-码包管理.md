# P02-03 - 码包管理

## 子功能名称
码包管理

## 功能概述
码包管理为 2 号机 Tab 子界面，通过点击 Tab 栏「码包管理」进入，用于码包的导入与列表管理。**支持的码包类型**：盖外码小标、盒外码中标、箱外码大标。**拉取规则**：盖外码小标、箱外码大标支持**在线更新**（含软件启动时自动更新）；盒外码中标**仅支持本地导入**，不参与在线拉取。导入数据存入数据库；列表展示码包类型、码包名称、导入时间、导入方式、码包数量（去重）、状态、备注及操作（查看、删除）。**不提供比对相关界面**，码包比对逻辑内化在采集流程中使用。

## 界面布局设计

- **入口**：Tab 栏「码包管理」，点击后主内容区切换为本页

### 整体布局
- **布局方式**：桌面应用风格，全宽内容区
- **主要区域**：
  - 顶部操作区：在线更新、本地导入
  - 搜索/筛选区：关键词、导入时间、导入方式、码包类型
  - 码包列表区：表格 + 分页
  - 状态栏：遵循 通用规则文档「02-用户界面设计」之状态栏设计

### 顶部操作区
- **在线更新**：仅拉取**盖外码小标**、**箱外码大标**两种类型。使用统一 URL + 请求参数传 `type`；服务端按参数返回对应类型码包。盒外码中标不参与在线拉取，需通过本地导入。详见「在线导入与更新规则」。
- **本地导入**：点击后弹出「码包导入」对话框，先选择码包类型 → 再选择本地 TXT 文件 → **需输入密码 123456** → 然后确认解析入库。**盖外码小标、箱外码大标**：无网络时的备选方案；**盒外码中标**：仅能通过本地导入（不提供在线拉取）。因三种码包文件格式相同，系统无法自动识别类型，必须由用户指定。
- **帮助提示图标 (?)**：
  - 样式：圆形问号图标，紧跟在「在线更新」按钮右侧，与按钮同行对齐
  - 触发：点击图标时弹出帮助提示气泡框（Popover）
  - 气泡框位置：图标下方，自动避边
  - 提示内容如下：

    > **码包导入说明**
    > 码包文件都是指外码，请不要导入内码，防止码包泄露。
    > **在线更新**（仅盖外码小标、箱外码大标）
    > 点击「在线更新」后，系统按各类型上次成功拉取时间分别增量拉取**瓶码（盖外码小标）**、**箱码（箱外码大标）**两种码包。**盒外码中标不参与在线拉取**，需通过本地导入。首次拉取全量，后续拉取增量。部分类型拉取失败时，失败的类型不覆盖本地数据，可继续使用已有码包；失败时可本地导入或网络恢复后重试。
    >
    > **启动自动导入**
    > 软件每次启动时自动执行一次在线更新，仅拉取盖外码小标、箱外码大标，规则同上。
    >
    > **本地导入**（包材厂给的码包文件）
    > **盒外码中标仅能通过本地导入**；盖外码小标、箱外码大标在无网络时也可本地导入。
    > 1. 点击「本地导入」
    > 2. 先选择码包类型：瓶码 / 盒码 / 箱码（三种文件格式一样，需您指定）
    > 3. 再选择本地 TXT 文件
    > 4. 输入密码 123456
    > 5. 确认解析入库，导入成功后列表自动刷新
    >
    > **码包文件格式**（包材厂提供的 TXT 文件）
    > - 瓶码：每行一个瓶码
    > - 盒码：每行一个盒码
    > - 箱码：每行一个箱码
    >
    > **删除**
    > 只有该码包里的码**都还没用过**时才能删除；有已用过的码则不能删（码替换时需要校验）。删除后数据仍保留，可查记录。
    >
    > **提醒**
    > 生产前请确保码包已正确导入，否则采集时会报错。可通过「查看」检查码包内容。

### 在线导入与更新规则

**适用范围**：仅**盖外码小标**、**箱外码大标**。**盒外码中标不参与在线拉取**，仅能通过本地导入。

**触发时机**：软件启动时自动执行；或用户点击「在线更新」。均只拉取盖外码小标、箱外码大标两种类型。

**拉取策略**（避免重复拉取大量/全量码包）：

| 场景 | 拉取范围 | 说明 |
| -- | -- | -- |
| 首次拉取 | 全量 | 该类型无历史记录时，拉取全部码包 |
| 后续拉取 | 增量 | 请求参数带上「该类型上次成功拉取时间」；服务端仅返回该时间之后有变更的码包 |

**时间标记**：**按码包类型分别记录**（盖外码小标、箱外码大标各一份）。每种类型单独记录其最近一次成功拉取时间；后续拉取该类型时，以该类型自己的时间作为增量请求起始时间。某类型拉取失败时不更新该类型的时间戳，下次仍用其上次成功时间增量拉取。

**部分拉取失败处理**：

| 情况 | 处理方式 |
| ---- | -------- |
| 某类型拉取失败 | 该类型不更新、不覆盖本地已有数据；时间戳不更新；界面提示该类型拉取失败及失败原因 |
| 部分类型成功 | 成功的类型正常入库并更新时间戳；失败的类型保持原状，提示用户 |
| 全部失败 | 本地码包数据不变；提示用户可尝试本地导入或检查网络后重试 |

**兜底机制**：

| 机制 | 说明 |
| ---- | ---- |
| **本地导入** | 盖外码小标、箱外码大标：在线拉取失败或无网络时，可本地导入；盒外码中标：仅能本地导入。包材厂线下提供的 TXT 文件可随时补入 |
| **保留已有数据** | 拉取失败的码包类型不覆盖、不清空本地已有数据，采集可继续使用现有码包生产 |
| **启动时自动重试** | 软件每次启动时自动执行在线更新（仅盖外码小标、箱外码大标），若上次部分失败，本次会按各类型自己的时间戳分别请求 |
| **手动重试** | 用户可点击「在线更新」再次触发拉取；网络恢复后可立即重试 |

**重复判断**：不仅判断文件名，**还须检测码包内数据**。若文件名不同但数据内容与已导入码包重复（或高度重叠），则跳过或提示，避免重复入库。

### 搜索与筛选
- **关键词**：模糊匹配码包名称等。
- **导入时间**：日期范围（起止）。
- **导入方式**：全部/在线/本地等。
- **码包类型**：全部/盖外码小标/盒外码中标/箱外码大标。
- 支持「搜索」，表格分页展示结果。

### 码包列表表格
- **列**：码包类型、码包名称、导入时间、导入方式、码包数量（去重）、状态、备注、操作。
- **码包类型**：盖外码小标、盒外码中标、箱外码大标。
- **码包数量**：该次导入的码数量，去重后统计。
- **操作**：
  - **查看**：弹窗展示该码包下**所有码**，弹窗内支持**搜索**（按码等）；列表可分页。弹窗内表格列：**码值**、**关联状态**、**关联时间**；码未关联时，关联时间显示「-」。
  - **删除**：执行前校验，**仅当该码包内无任何已关联码时允许删除**；若有已关联码则禁止删除并提示。允许删除时，标记该条码包导入记录为已删除，并同步将该次导入的码数据标记为已删除（逻辑删除，不物理清空）。

### 数据与逻辑说明
- 导入数据**存入数据库**，每条导入生成一条码包导入记录，并写入对应码数据（**热表**，未使用状态）。
- **码包重复判断**：不仅按文件名，**还须检测码包内数据**。冷热分离时须**同时查热表+冷表**（或统一视图 all_codes），任一存在即重复；若高度重叠则跳过或提示。
- **不涉及比对的展示**：本页不做“码关系比对”的界面或结果展示；比对相关逻辑在**采集时**由系统内化使用（如采集时校验码是否在码包内等）。

### 码包删除边界
| 条件 | 是否允许删除 | 说明 |
|------|--------------|------|
| 该码包内**无任何已关联码** | 允许 | 如误导入、重复导入，可逻辑删除 |
| 该码包内**存在已关联码** | 禁止 | 码替换时新码需在码包内校验；删除会导致已关联码无法做替换 |

### 码包使用完后的处理
- **冷热分离**：T1.2.0 采用热表+冷表设计；码被采集关联成功时**落冷**（热表→冷表），取消关联时**回迁**（冷表→热表）。详见 AI 文档 10.1.1。
- **使用完的码包**：码包内码全部被采集关联后，码已落冷至冷表，码包记录保留在库中；因存在已关联码，**不可删除**。
- **数据保留**：逻辑删除的码包数据不物理清空，仅标记已删除，便于审计与追溯。

## 交互要点
- 在线更新：点击 → 按各类型上次成功拉取时间分别增量请求**盖外码小标、箱外码大标**两种类型（盒外码中标不参与）→ 成功的入库并更新时间戳，失败的提示且不覆盖 → 重复判断（文件名+数据）→ 刷新列表。
- 本地导入：点击 → 弹窗 → 选择码包类型 → 选文件 → 输入密码 123456 → 确认解析入库 → 刷新列表。
- 查看：行操作「查看」→ 弹窗展示该码包全部码，支持搜索与分页。
- 删除：行操作「删除」→ 校验该码包内是否有已关联码；无则确认后逻辑删除；有则禁止并提示。

## 功能边界
- **包含**：码包类型（盖外码小标、盒外码中标、箱外码大标）；在线更新（仅盖外码小标、箱外码大标，含启动时自动更新）；本地导入（盒外码中标仅能本地导入，盖外码小标/箱外码大标无网络时也可本地导入、密码保护）；重复判断（文件名+数据内容）；码包列表、搜索筛选、查看、删除（仅无已关联码时可删）。

## 适用场景
- **在线更新**：盖外码小标、箱外码大标，生产前或运行中从服务器拉取最新码包，有网络时首选；软件启动时自动执行。
- **本地导入**：盒外码中标仅能本地导入；盖外码小标、箱外码大标在无网络时也可本地导入（网络故障、离线环境、包材厂线下提供文件）。
- 查看与删除管理，供采集流程使用。
