# P02-02 - 手工采集

## 子功能名称
手工采集 - 相机损坏时瓶盒关联备用方案

## 功能概述
为生产线操作员提供手工采集功能界面，在相机损坏时使用，仅支持瓶盒关联。通过扫码枪手动扫码替代相机自动采集，确保生产不中断。界面采用与主界面一致的Tab切换方式展示，通过点击「手工采集」Tab进入。

**重要说明**：本功能为2号机备用方案，仅在相机损坏时使用。手工采集仅支持瓶盒关联。

## 界面布局设计

### 整体布局
- **布局方式**：左中右三栏布局，与主界面保持一致
- **适配要求**：工控机鼠标键盘操作优化
- **主要区域**：
  - **左侧栏**（约380px）：采集规格设置（1盒 [6]瓶，6可修改）+ 开始采集提示区（当前应扫 瓶码/盒码）
  - **中间栏**（自适应，最小600px）：数据接收区、操作日志
  - **右侧栏**（约380px）：单位实时统计（当前已读、每盒瓶数文字在上数字在下；生产总数单独一行）+ 任务控制（开始/停止采集、清零）
  - 状态栏：遵循 通用规则文档「02-用户界面设计」之状态栏设计

### 功能模块设计

#### 左侧栏：采集规格设置 + 开始采集提示区 + 码输入区
- **采集规格设置**：一行「1盒 [6]瓶」，6为可编辑输入框，默认值6；开始采集后不可修改
- **输入框校验规则**：不能为空；仅允许输入整数；范围 1～999；不符合时提示并阻止开始采集
- **开始采集提示区**（标题区域）：
  - **码输入区**（采集进行中才可用，位于「当前应扫」提示文字上方）：
    - 布局：输入框 + 右侧「加入」按钮，横向排列；输入框占位提示文字随当前应扫层级变化（如「扫描或输入瓶码」、「扫描或输入盒码」）
    - **扫码枪输入**：扫入后自动触发加入，无需点击按钮
    - **手动输入**：输入完成后按 Enter 键或点击「加入」按钮触发加入
    - **采集未开始时**：输入框和「加入」按钮均置灰不可用
    - **加入后**：输入框内容自动清空，焦点保持在输入框，便于连续扫码
  - **当前应扫提示**：固定文字「当前应扫：」+ 当前层级名称；层级名称（「瓶码」或「盒码」）显示为**蓝色**，其余文字为默认色；无指示灯，文字提示即可

#### 中间栏：数据接收区、操作日志
- **数据接收区**：扫码记录、关联结果（盒码关联成功、瓶码采集等）；倒序显示，最新在上；白色背景，可滚动
- **操作日志**：进入手工采集模式、设备状态等；倒序显示

#### 右侧栏：单位实时统计、任务控制
- **单位实时统计**：
  - **当前已读、每盒瓶数**：文字在上、数字在下，每项独立白底卡片，横向并排；当前已读红色、每盒瓶数蓝色，大字号
  - **生产总数**：单独一行，文字在上、数字在下，样式较简洁
- **任务控制**：开始/停止采集（主按钮，高度40px）、清零（清空生产总数，重置为0）

#### 数据接收区显示格式

所有条目格式均为 `HH:MM:SS 内容`，倒序显示，最新在上，文字过长自动换行。

**码合法性校验**：每次提交码值后，系统先校验合法性，再决定显示内容和文字颜色：

| 场景 | 显示内容示例 | 文字色 |
|------|-------------|--------|
| 瓶码合法（常规采集） | `10:30:10 瓶码采集 20241201A0001` | 黑色（默认） |
| 盒码合法（常规采集） | `10:30:18 盒码采集 20241201B0302` | 黑色（默认） |
| 瓶盒关联成功 | `10:30:18 盒码关联成功 20241201B0302（6瓶）` | 绿色 |
| 瓶码不在码包范围 | `10:30:12 瓶码 20241201A0099：码不在码包范围内` | 红色 |
| 盒码不在码包范围 | `10:30:19 盒码 20241201B0099：码不在码包范围内` | 红色 |
| 重码 | `10:30:13 瓶码 20241201A0001：重复出现` | 红色 |
| 码为空或格式错误 | `10:30:14 输入内容无效，请重新扫码或输入` | 红色 |

**校验规则**：
- 当前应扫「瓶码」时：提交的码须在**盖外码小标**码包热表中，且非重码
- 当前应扫「盒码」时：提交的码须在**盒外码中标**码包热表中，且非重码
- 校验不通过：在数据接收区红色文字展示原因，**不计入当前已读**，输入框清空等待重新输入
- 校验通过：黑色文字展示（常规），当前已读+1；达到规格数量时自动触发瓶盒关联，关联结果以绿色文字展示

## 交互功能设计

### 开始采集前初始化检测

| 类别 | 检测项 | 不通过时 |
|------|--------|----------|
| **门禁** | 码包是否已导入 | 盖外码小标、盒外码中标均须已导入；未导入则提示，阻止开始 |
| **规格** | 采集规格已确认（1盒 [6]瓶，开始后不可修改） | 无（可修改后继续） |
| **设备** | 扫码枪已就绪 | 操作日志显示设备状态，提示检查 |
| **连接** | 数据库可访问（2号机本机库） | 提示连接失败，阻止开始 |

### 手工采集流程
1. 点击「手工采集」Tab切换至手工采集页面
2. 设置采集规格（1盒 [6]瓶，6可修改）；**采集开始后不可再改**
3. 点击「开始采集」按钮，系统执行初始化检测，通过后按钮变为「停止采集」；码输入区输入框和「加入」按钮变为可用
4. 按左侧开始采集提示顺序扫码/输入码：
   - 通过扫码枪扫入或手动输入码值，按 Enter 或点击「加入」提交
   - 系统校验码合法性：
     - **合法**：数据接收区黑色文字展示（常规采集），当前已读+1，输入框清空并保持焦点
     - **不合法**：数据接收区红色文字展示具体原因，不计入当前已读，输入框清空等待重新输入
   - 左侧提示区文字随进度变化（「扫描或输入瓶码」→「扫描或输入盒码」）
5. 扫够规格数量后自动完成瓶盒关联，关联结果展示在数据接收区，生产总数累计，**当前已读归零重新计数**
6. 继续下一组扫码；需结束时点击「停止采集」
   - 若停止时存在已扫但未完成关联的瓶码（即当前已读 > 0 且尚未扫盒码），这些临时数据**直接丢弃**，当前已读重置为0，下次开始采集重新从瓶码开始

## 数据处理说明

### 数据写入数据库
- **写入时机**：每完成一组瓶盒关联（扫够瓶码数量后扫盒码，关联成功）即写入数据库
- **写入内容**：瓶码列表、盒码、瓶盒关联关系、采集时间戳，与正常相机采集关联的数据结构完全一致
- **数据一致性**：手工采集写入的数据与自动相机采集数据存入同一数据库表，后续2号机盒箱垛关联、数据查询、数据替换、取消关联等功能均可正常使用。**冷热分离**：手工采集关联成功后，同样执行**落冷**（热表→冷表），与自动采集一致。
- **不参与同步**：手工采集在2号机本地直接操作，不需要经过1号机同步流程，数据直接写入2号机（本机）数据库

## 功能边界

### 包含的功能
1. 开始采集/停止采集按钮（点击切换）
2. 手工扫码瓶盒关联
3. 开始采集提示区文字告知当前应扫层级（瓶码/盒码）；码输入区支持扫码枪/手动输入+Enter/点击「加入」按钮三种方式提交码值
4. 采集规格设置（1盒 [6]瓶，6可修改，采集中不可改）；单位实时统计（当前已读、每盒瓶数文字在上数字在下，生产总数单独一行）
5. 生产总数与清零
6. 关联数据写入数据库（与正常采集一致）

### 与主界面（P02-01）采集的关系
- **可同时运行**：手工采集（P02-02）与主界面盒箱垛关联采集（P02-01）互相独立，**不互斥**，可同时开启采集
- 二者写入同一数据库表，数据结构一致，后续盒箱垛关联流程统一消费瓶盒关联结果，无论来源为相机自动采集还是手工采集

### 适用场景
- 相机损坏时的应急生产
- 设备维护期间的临时采集
