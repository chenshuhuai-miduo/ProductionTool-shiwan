# 石湾产线硬件清单
工控机2台、采集相机5台、箱剔除装置、报警灯2盏

| 设备 | 作用 |
| --- | --- |
| 瓶码采集相机2台 | 交叉去重采集瓶数据并同步至1号机 |
| 盒码采集相机1台（瓶盒） | 采集盒数据并同步至1号机 |
| 工控机 1号机 | 接收瓶盒数据并同步至2号机 |
| 1号机报警灯 | 点击开始采集报警灯常亮，如果采集数据不齐报警灯亮红灯，正常则绿灯常亮。 |
| 工控机 2号机 | 接收一号机数据&接收盒箱数据 |
| 盒码采集相机1台（盒箱） | 采集盒数据并同步至2号机 |
| 剔除装置 | 如果2号机的瓶盒箱数据采集不齐，2号机给plc剔除信号，剔除装置接收信号做出剔除动作 |
| 2号机 报警灯 | 点击开始采集绿灯常亮，数据不齐报警亮红灯。 |
| PLC | 控制剔除设备及报警灯 |


# 盒箱垛关联软件
> 工控机 2号机

## 数据采集
<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/13007158/1770343391818-9b45e909-ad92-41e5-8653-4ba38180d94c.png)

1. **生产产品选择**
    - 生产产品选择
    - 「更新产品」：刷新产品列表（数据源从开放平台获取的数据api/sign/md.shop.products/v1/list）
    - 筛选 / 全部：用于过滤产品列表（输入后筛选对应关键词的产品；全部=重置，在可选列表展示全部）
    - 下拉选择已有产品
    - 产品编号：选择产品后自动带出（不可修改）
    - 生产单号：根据当天日期自动生产生产单号（可修改）
2. **采集控制**
    - 生产开始&关闭
        * 点击"开始采集"开始生产：点击后系统检测各个硬件是否在线（相机、剔除装置等）
        * 再次点击按钮停止采集：当点击前有未满垛的数据，这些数据会存在数据库，不会自动组垛（后续可以提取工单未成垛）
    - 无需采集码：当天不生产五码合一产品，开启后，读码器等设备不进行工作（其实软件都不用开，工厂要求，什么数据都不采集）
3. **数量实时统计**
    - 每箱盒数：红色为预设值的每箱装多少盒，蓝色为实时计数，开始采集后不可修改
    - 每垛箱数：红色为预设值的每垛装多少箱，蓝色为实时计数，开始采集后不可修改
    - 当采集达到预设值自动进行盒箱关联，自动生产垛码进行箱垛关联；
4. **强制操作**
    - 强制满垛：未达到设定箱数，也强制结束当前垛，生产垛码
    - 清空垛：清除当前垛计数
    - 提取工单未成垛：当软件关闭前的尾数板没有进行强制满垛操作时，可以通过此按钮提取"未完成垛"的数据，隔天继续生产。
5. **剔除相关**
    - 收回剔除：当剔除设备进行剔除动作后没有自动收回时，点击此按钮可以让剔除设备收回当前剔除动作（PLC控制，对应设置的信号）。
    - 总剔除数：累计剔除数量
    - 清零：剔除计数清零
6. **运行状态监控**
    - 检查瓶盒关联：检测瓶盒关联是否成功，结果输出在信息栏中
    - 禁用剔除：剔除动作是否开启（应该要强制开启，禁止关闭）
    - 读码中：生产进行中
    - 逻辑与：检测关联逻辑是否正常（显示用，无操作功能）
7. **信息栏（左下）**
    - 详细记录实时业务信息
    - 设备是否在线
    - 装箱完成
    - 满箱 / 满垛提示
8. **信息栏2（中下）**
    - 较简单的记录业务信息
9. **异常信息（右下）**
    - 异常 / 报错日志
    - 重码
    - 未成功采集
    - ···

## 数据查询
<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/13007158/1770343452055-c23e8271-541d-4673-be64-806924bebcde.png)

通过（扫码枪扫描/输入）瓶盒箱垛数据后点击对应按钮进行查询码关系

+ **按小码查询**：以扫描的小码作为最小包装单元，查询并展示其所属的上一级及更高层级包装码信息。
+ **按大码查询**：以扫描的大码作为当前包装层级，查询并展示其所包含的下一级及更低层级包装码信息。

**举例**

+ 扫盒码后点击按小码查询输出对应箱子里所有的盒
+ 扫盒码后点击按大码查询输出里面所有瓶

具体信息里面展示对应的包装比例、采集时间等信息

## 扫大码取消关联
<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/13007158/1770343490487-a9c95924-cd32-466b-91f1-f5b99153fba6.png)

通过扫描需要处理的大码，系统自动识别对应的包装单元，并将待删除的大码列表进行集中管理。

支持对选定的大码进行批量取消关联操作，一次性解除大码及其所包含下级码的关联关系，并实时统计删除数量，确保数据处理清晰可控。

1. **删大码信息栏**：作为取消关联后的操作记录区域
2. **当前扫码**：显示当前扫描输入的大码
3. **需要删除的大码列表**：展示所有已加入"待取消关联"的大码
    - 支持批量处理，避免重复扫码或遗漏
4. **删除大码数量**：实时统计当前待删除或已处理的大码数量
    - 便于操作人员确认处理范围
5. **批量取消关联**：对列表中的所有大码执行取消关联操作（需要输入密码123456）
    - 自动解除大码与其下级包装码的关联关系

## 数据上传
<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/13007158/1770343541171-15ee873f-859a-4833-b269-55001c2e1c2d.png)

当天生产结束或者生产中途点击"点击上传"按钮，系统自动将当天生产数据上传到米多大数据引擎。

1. **点击上传**：手动触发数据上传流程，将当前待上传的数据提交至服务器
2. **上传信息**：显示上传过程中的日志信息，例如：
    - 开始上传
    - 上传成功--不准确，是否上传成功要在查询垛码框进行查询
    - 上传结束提示没有可同步的数据
3. **ResultString**：显示上次触发上传后的日志，关掉软件后自动消失
4. **一号机同步信息2**：展示1号机与2号机之间的同步信息状态
5. **查询垛码**：输入需要查询的垛码点击"查询上传"展示上传是否成功
    - 用于定位某一垛数据的上传情况
6. **设为未上传**：将指定垛码状态重置为"未上传"
    - 用于：
        * 异常数据修复后重新补传
7. **设为已上传**：手动将指定垛码标记为"已上传"
    - 用于：
        * 已人工确认无误
        * 避免系统重复上传
8. **上传信息 2**：展示查询结果或指定垛码的详细上传记录
    - 包含：
        * 上传时间
        * 上传状态
        * 返回信息

## 数据采集
<!-- 这是一张图片，ocr 内容为： -->
![](https://cdn.nlark.com/yuque/0/2026/png/13007158/1770343691740-328e00b1-7dc1-4267-a928-4c7bdac08aa6.png)
