## 五、实现思路

### 5.1 产品定位

赋码采集关联系统重构T1.2（石湾版）在保持旧版核心功能基础上，采用 Java11+JavaFX17 技术栈重构升级的**系统级桌面应用程序**。软件定位为**石湾产线专用的四级关联采集关联系统**，由两个独立软件组成：**瓶盒关联软件（1号机）和盒箱垛关联软件（2号机）**，共同完成瓶-盒-箱-垛四级码关联，提供码包管理、数据采集、关联建立、数据上传等完整功能。

### 5.2 结构归母

本软件属于**赋码采集关联系统产品线**下的**石湾产线专用系统级版本**，与致美斋产线二级关联版本属同一产品线不同业务场景适配。采用统一 Java 技术架构和开发规范，针对石湾产线四级关联模式和硬件配置专门适配。


| 软件           | 部署     | 职责                  |
| ------------ | ------ | ------------------- |
| 瓶盒关联软件（1号机）  | 工控机1号机 | 瓶盒关联数据采集            |
| 盒箱垛关联软件（2号机） | 工控机2号机 | 盒箱垛关联数据采集、数据管理、数据上传 |


### 5.3 设计理念


| 理念    | 要点                           |
| ----- | ---------------------------- |
| 功能完整性 | 保持旧版所有核心功能，新增码包管理、码关系比对、自动上传 |
| 操作友好性 | 按钮帮助提示（"？"）、密码保护、清晰状态提示      |
| 技术先进性 | Java11+JavaFX17，可维护、可扩展、稳定   |
| 数据准确性 | 码包比对、重码检测、数据校验               |


### 5.4 设计思路

**双软件架构**：保持两个独立软件，分别打包出两个安装程序，共用部分代码复用。详见**八、页面清单与关系**。


| 设计要点   | 说明                                         |
| ------ | ------------------------------------------ |
| 包装比例   | 必须通过配置参数设置（G/N/M 可配置），不能写死在代码中             |
| 码位数配置  | 系统设置中配置大标、中标、小标位数；采集时校验，不符则告警关联失败；-1 表示不校验 |
| 码包在线拉取 | 统一 URL + type 参数；URL 和 type 由代码或配置文件提供     |
| 新增功能   | 码包导入与比对、自动上传与指示灯控制                         |
| 设计优化   | 操作帮助（按钮旁"？"）、重要参数密码保护（详见十一）                |

### 9.2 环节详情（条件与门禁）

**模式43**：商品包装比例 1:M:N:G（石湾产线），1垛:M箱:N盒:G瓶，G/N/M 可配置。

#### 9.2.1 环节1：生产准备


| 项目       | 内容                                            |
| -------- | --------------------------------------------- |
| **入口条件** | 用户已选择生产任务或产品                                  |
| **门禁1**  | 码包是否已导入？                                      |
| 门禁1-通过   | 进入环节2                                         |
| 门禁1-不通过  | 提示导入码包；支持在线拉取（URL+type）或本地导入（用户选择类型：小标/中标/大标） |
| **门禁2**  | 码包格式是否正确？txt，小标=每行瓶码、中标=每行盒码、大标=每行箱码          |
| 门禁2-不通过  | 显示错误提示，需重新导入                                  |
| **产出**   | 码包入库、包装比例G/N/M已配置，可开始采集                       |


**码包比对**：生产前导入；采集时（环节2~4）实时校验采集码是否在对应类型码包范围内；无包材厂回传时米多兜底校验并告知。

#### 9.2.2 环节2：瓶码采集（1号机）


| 项目        | 内容                           |
| --------- | ---------------------------- |
| **入口条件**  | 生产任务已启动，环节1已完成               |
| **门禁**    | 采集到的瓶码是否有效？                  |
| 有效判定      | 格式正确、在盖外码小标码包范围内、非重码（系统中未存在） |
| 无效判定      | 重码、错码（格式错误或不在码包范围）           |
| **通过动作**  | 瓶码有效计数+1，进入下一瓶采集             |
| **不通过动作** | 不计数，不建立关联，写日志，标记该瓶所属盒待剔除，产线正常移动 |
| **产出**    | 有效瓶码计数，用于环节3判断是否满一盒          |

#### 9.2.3 环节3：盒码关联（1号机）


| 项目         | 内容                                    |
| ---------- | ------------------------------------- |
| **入口条件**   | 瓶码有效计数达到G（如6瓶）                        |
| **门禁1**    | 实际瓶码数量是否等于G？                          |
| 门禁1-通过（=G） | 采集盒码，校验盒码有效性（格式、盒外码中标码包、非重码）          |
| 门禁1-不通过    | 超码（>G）或缺码（<G）→ 不计数，不建立关联，写日志，标记该盒待剔除，同步剔除标记至2号机 |
| **门禁2**    | 盒码是否有效？                               |
| 门禁2-通过     | 盒码有效计数+1，建立瓶-盒关联，同步2号机                |
| 门禁2-不通过    | 不计数，不建立关联，写日志，标记该盒待剔除，同步剔除标记至2号机 |
| **产出**     | 瓶-盒关联数据，同步至2号机                        |

#### 9.2.4 环节4：箱码关联（2号机）


| 项目       | 内容                      |
| -------- | ----------------------- |
| **入口条件** | 接收1号机瓶盒数据；每N盒（如4盒）组成一箱  |
| **门禁1**  | 该箱是否已被1号机标记待剔除？         |
| 门禁1-是    | 不计数，不建立关联，写日志，执行剔除，不装垛 |
| 门禁1-否    | 校验盒箱关系（N个盒码与箱码一一对应）     |
| **门禁2**  | 盒箱关系校验是否通过？             |
| 门禁2-不通过  | 不计数，不建立关联，写日志，执行剔除，不装垛 |
| **门禁3**  | 小标、中标、大标码包比对是否全部通过？     |
| 门禁3-通过   | 箱码有效计数+1，建立盒-箱关联，工人装垛   |
| 门禁3-不通过  | 不计数，不建立关联，写日志，执行剔除，不装垛 |
| **产出**   | 盒-箱关联数据，箱码计数用于环节5判断是否满垛 |

---

#### 9.2.5 环节5：垛码关联（2号机）


| 项目       | 内容                                 |
| -------- | ---------------------------------- |
| **入口条件** | 箱码有效计数达到M（如70箱）                    |
| **门禁**   | 是否触发成垛？                            |
| **通过动作** | 生成虚拟垛标（格式：前缀+年月日+序号+产线号），建立M箱与垛码关联 |
| **失败处理** | 程序内自动重试，一般不会失败；无需人工干预              |
| **产出**   | 垛-箱关联，完整四级关联（瓶-盒-箱-垛），触发环节6        |

#### 9.2.6 环节6：数据上传（2号机）


| 项目       | 内容                         |
| -------- | -------------------------- |
| **入口条件** | 成垛完成                       |
| **门禁**   | 成垛数据是否完整？                  |
| 门禁-不通过   | 记录错误，不执行上传                 |

### 10.1 业务规则

#### 10.1.1 码包管理规则

**页面职责**：码包导入（在线/本地/更新）、列表展示、查看/删除；**不展示比对结果**，比对逻辑在采集流程内执行。

**支持的码包类型**：盖外码小标、盒外码中标、箱外码大标。


| 规则类型      | 具体规则                                                     |
| ------------- | ------------------------------------------------------------ |
| **导入-来源** | 包材厂回传文件；支持在线拉取（统一 URL + type 参数，不通过系统配置）或本地导入 |
| **导入-本地** | 用户必须先选择码包类型（小标/中标/大标），再选择文件；格式相同无法自动识别 |
| **导入-累计** | 多码包可累加导入，按名称搜索；不存内码                       |
| **格式**      | txt；小标=每行一个瓶码；中标=每行一个盒码；大标=每行一个箱码 |
| **比对**      | 采集码 ∈ 对应类型码包 → 正常关联；采集码 ∉ 码包 → 不计数、不建立关联、写日志、标记剔除 |
| **删除**      | 该码包内**无任何已关联码**时允许删除；存在已关联码则禁止删除 |
| **使用完**    | 码包内码全部关联后保留在库，不自动归档                       |
| **米多兜底**  | 无包材厂回传时，软件校验并告知操作人员                       |


**成垛前码包比对**（环节4 每箱采集完成、判断成垛前执行）：


| 校验项 | 判定条件                 | 不通过动作      |
| --- | -------------------- | ---------- |
| 小标  | 箱内所有瓶码 ∈ 盖外码小标码包     | 不计数，不建立关联，写日志，标记该箱剔除 |
| 中标  | 箱内所有盒码 ∈ 盒外码中标码包     | 同上         |
| 大标  | 箱码 ∈ 箱外码大标码包 且 箱码不重复 | 同上         |
| 通过  | 小标✓ 且 中标✓ 且 大标✓      | 计入箱数、参与成垛  |


#### 10.1.2 数据采集规则

**四级关联**（G/N/M 可配置）：

```
瓶码(2相机去重) → 每G瓶→1盒码 → 每N盒→1箱码 → 每M箱→1垛码(虚拟)
```

**码有效性**：


| 类型  | 判定条件              | 处理动作                     |
| --- | ----------------- | ------------------------ |
| 有效码 | 格式正确、在对应码包范围内、非重码 | 有效计数+1                   |
| 重码  | 该码在系统中已存在         | 不计数，不建立关联，写日志，标记该盒待剔除 |
| 错码  | 格式错误或不在对应码包范围     | 不计数，不建立关联，写日志，标记该盒待剔除 |
| 超码  | 实际瓶码数 > G（如 6 瓶）  | 不计数，不建立关联，写日志，标记该盒待剔除，同步剔除标记至 2 号机 |
| 缺码  | 实际瓶码数 < G         | 不计数，不建立关联，写日志，标记该盒待剔除，同步剔除标记至 2 号机 |


**码位数校验**：系统设置可配置大标、中标、小标位数（-1 表示不校验）。采集时：实际采码位数 ≠ 配置位数 → 异常告警、关联失败。例：大标配置 6 位，实际采到 7 位 → 不符合 → 关联失败。

**剔除控制**：1 号机不建立关联、写日志、标记待剔除 → 剔除标记同步 2 号机 → 产线到达龙门架时按箱剔除；2 号机盒箱校验不通过或码包不通过 → 不计数、不建立关联、写日志 → 龙门架剔除装置剔除。

**剔除后工人处理规则**（前提：实物不丢弃，最小影响生产）：


| 剔除场景          | 工人实物处理             | 工人系统操作             |
| ------------- | ------------------ | ------------------ |
| 瓶码重码/错码/超码/缺码 | 开箱换问题瓶或整盒          | 实物修复 → 重新上线 → 系统重采 |
| 盒箱校验不通过       | 核对 N 盒与箱码，换错盒或错箱   | 实物修复 → 重新上线 → 系统重采 |
| 小标/中标/大标码包不通过 | 换瓶/盒/箱使码在对应类型码包范围内 | 实物修复 → 重新上线 → 系统重采 |
| 已标记要剔除的箱      | 按 1 号机异常类型逐项处理     | 同上                 |


**成垛数据中码错误**（实物未剔除）：


| 场景       | 工人系统操作                                    |
| -------- | ----------------------------------------- |
| 码损坏/印刷错误 | **码替换**：扫旧码 → 扫新码 → 密码确认；新码须在码包内且未用       |
| 关联关系错误   | **取消关联**：垛 → 箱 → 盒 → 瓶逐级解除；已上传的须先解云端再处理本地 |


**相机损坏**：使用 **手工采集**（P02-02）进行瓶盒关联，人工扫码补录。

#### 10.1.3 数据关联规则


| 规则          | 具体规则                                                  |
| ----------- | ----------------------------------------------------- |
| 瓶盒关联（1 号机）  | 采瓶码（多源去重）；无效不计数、不建立关联、写日志、标记剔除；每 G 有效瓶 → 采集 1 盒码并建立关联；实时同步 2 号机 |
| 盒箱垛关联（2 号机） | 接收 1 号机数据；每 N 盒 → 1 箱；成垛前执行码包比对；每 M 箱 → 生成虚拟垛标并关联     |
| 强制满垛        | 未达 M 箱时可强制结束当前垛，忽略箱数差异                                |
| 提取未成垛       | 输入/扫箱码 → 查订单 → 确认 → 回显主页面继续生产                         |


**虚拟垛标生成规则**（格式在 系统设置 → 基础设置 → 虚拟垛标规则 中配置）：


| 项目   | 规则                                                                    |
| ---- | --------------------------------------------------------------------- |
| 生成时机 | 箱码有效计数达到 M 箱（可配置）时触发                                                  |
| 生成方式 | 程序自动生成虚拟垛标，建立 M 箱与垛码的关联                                               |
| 格式   | 前缀 + 年月日 + 序号 + 产线号，如 V20260226001A；前缀默认 V、时间取当前日期、序号 3 位起自动递增、产线号可配置 |
| 配置位置 | 系统设置 → 基础设置 → 虚拟垛标规则                                                  |
| 成功   | 生成成功 → 计入垛数 → 触发自动上传                                                  |
| 失败处理 | 程序内自动重试；一般不会失败，无需人工干预                                                 |


**退出软件前判断**：

**1号机**（仅瓶盒关联，无垛概念）：

```
点击退出
  ↓
是否在采集中？ ─是→ 提示「请先停止采集」→ 停止后再判断
  ↓ 否
常规退出确认 → 退出
```

**2号机**（盒箱垛关联，有未满垛概念）：

```
点击退出
  ↓
是否在采集中？ ─是→ 提示「请先停止采集」→ 停止后再判断
  ↓ 否
是否满垛？ ─否→ 弹窗「存在未满垛(X箱)，是否暂存？」
  ↓              ├ 暂存 → 存库标记未成垛 → 允许退出
  ↓              ├ 不暂存 → 允许退出
  ↓              └ 取消 → 返回主界面
  ↓ 是/无未满垛
常规退出确认 → 退出
```

#### 10.1.4 数据上传规则
