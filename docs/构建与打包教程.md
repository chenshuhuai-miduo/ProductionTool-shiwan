## 1. 环境准备

### 1.1 必需软件

| 软件 | 版本要求 | 下载地址 | 说明 |
|------|----------|----------|------|

```powershell
[System.Environment]::SetEnvironmentVariable("PATH", "$env:PATH;%JAVA_HOME%\bin", "Machine")
```

#### 配置 MAVEN_HOME

#### 验证安装
```bash

java -version
javac -version

# 验证 Maven

```

## 2. Maven 构建打包

### 2.1 清理并编译项目

打开 PowerShell 或 CMD，进入项目根目录：

```bash
cd K:\Java\productiontools
```

#### 方式一：使用批处理脚本（推荐）

项目已提供 `build.bat` 脚本：

```bash
.\build.bat
```

该脚本会自动执行以下步骤：
1. 清理旧构建（`mvn clean`）
2. 安装依赖模块
3. 打包 bootstrap 模块

#### 方式二：手动执行 Maven 命令

```bash
# 步骤 1：清理项目
mvn clean

# 步骤 2：安装所有模块到本地仓库
mvn install -DskipTests

# 步骤 3：单独打包 bootstrap 模块（生成可执行 JAR）
cd miduo-bootstrap
mvn package -DskipTests
```

### 2.2 打包输出位置

打包成功后，JAR 文件位置：

```
miduo-bootstrap\target\miduo-ccas-launcher.jar
```

**验证 JAR 文件：**
```bash
# 查看生成的 JAR 文件
dir miduo-bootstrap\target\*.jar

# 应该看到：
# miduo-ccas-launcher.jar    # 主启动 JAR（包含所有依赖）
```

### 2.3 测试 JAR 运行

在运行 JAR 之前，确保：
1. 数据库服务已启动
2. 配置文件正确（`application.properties`）

```bash
# 运行 JAR
java -jar miduo-bootstrap\target\miduo-ccas-launcher.jar
```

**预期行为：**
- 后端服务启动（端口 8080）
- 前端 JavaFX 窗口打开
- 无报错信息

## 3. exe4j 制作 EXE 文件

### 3.1 准备工作

#### 3.1.1 准备 JRE（可选但推荐）

为了让 EXE 在没有安装 Java 的电脑上运行，需要 bundled JRE：

**方案 A：使用独立 JRE 文件夹**
```bash
# 下载 JRE 11（或复制 JDK 中的 JRE）
# 放置位置示例：K:\Java\productiontools\jre-11\
```

**方案 B：使用 jlink 定制精简 JRE**
```bash
# 在 JDK 11 环境下
cd K:\Java\productiontools

# 生成包含所需模块的 JRE
jlink --module-path "%JAVA_HOME%\jmods" ^
      --add-modules java.base,java.desktop,java.management,java.sql,java.xml,jdk.crypto.ec ^
      --output jre-custom ^
      --compress=2 ^
      --no-header-files ^
      --no-man-pages
```

#### 3.1.2 准备图标文件

准备一个 `.ico` 格式的图标文件：
```
K:\Java\productiontools\assets\app-icon.ico
```

### 3.2 exe4j 配置步骤

#### 步骤 1：启动 exe4j

1. 打开 exe4j
2. 点击 "Next"
3. 选择 "Executable infor mode" → "Next"

#### 步骤 2：设置项目名称和输出路径

```
Executable info file: K:\Java\productiontools\miduo-bootstrap\miduo.exej
Output directory: K:\Java\productiontools\miduo-bootstrap\target\exe
```

#### 步骤 3：选择 EXE 类型

```
☑ Next is a GUI application  # 选择 GUI 应用
☐ Console application        # 不选（除非需要命令行窗口）
```

#### 步骤 4：设置 EXE 文件名和图标

```
Executable name: 米多赋码采集关联系统.exe
Icon file: K:\Java\productiontools\assets\app-icon.ico
```

#### 步骤 5：配置 JRE 路径

**关键步骤！** 选择以下任一方式：

**方式 A：Use bundled JRE（推荐）**
```
☑ Use bundled JRE instance
JRE installation directory: K:\Java\productiontools\jre-11
```

**方式 B：Search for JRE**
```
☑ Search for JRE
Version range: 11 to 11
```

#### 步骤 6：添加 Classpath

点击 "Add..." 添加以下内容：

```
1. miduo-bootstrap\target\miduo-ccas-launcher.jar
2. （如果有外部依赖也添加到这里）
```

#### 步骤 7：设置 Main Class

```
Main class: com.miduo.cloud.MiduoBackendApplication
```

**注意：** 这是后端启动类，它会同时启动前端 JavaFX 应用。

#### 步骤 8：VM Options 配置（重要！）

在 "VM options" 中添加：

```
-Xms512m
-Xmx2048m
-Dfile.encoding=UTF-8
-Dsun.jnu.encoding=UTF-8
```

**说明：**
- `-Xms512m`：初始堆内存 512MB
- `-Xmx2048m`：最大堆内存 2GB
- `-Dfile.encoding=UTF-8`：防止中文乱码

#### 步骤 9：完成向导

点击 "Finish"，exe4j 会开始生成 EXE 文件。

### 3.3 高级配置（可选）

#### 3.3.1 设置工作目录

在 exe4j 配置文件中添加：

```xml
<workingDirectory>K:\Java\productiontools</workingDirectory>
```

#### 3.3.2 设置应用程序数据目录

```xml
<environmentVariables>
  <variable name="APPDATA" value="%APPDATA%\miduo"/>
</environmentVariables>
```

#### 3.3.3 错误处理配置

在 "Startup mode" → "Advanced" 中：

```
☑ Show error message on startup failure
Error message title: 米多赋码采集关联系统 - 启动失败
```

### 3.4 测试 EXE 文件

```bash
# 双击运行或在命令行运行
.\miduo-bootstrap\target\exe\米多赋码采集关联系统.exe
```

**检查项：**
- ✅ 窗口正常显示
- ✅ 图标正确
- ✅ 无 Java 错误提示
- ✅ 功能正常运行

## 5. 常见问题解决

### ❌ 问题 1：打包时提示 "找不到主类"

**原因：** Main-Class 未正确配置

**解决方案：**
检查 `miduo-bootstrap/pom.xml` 中的配置：

```xml
<plugin>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-maven-plugin</artifactId>
    <configuration>
        <mainClass>com.miduo.cloud.MiduoBackendApplication</mainClass>
    </configuration>
</plugin>
```

### ❌ 问题 2：EXE 启动后闪退

**可能原因：**
1. JRE 版本不匹配
2. 缺少配置文件
3. 端口被占用

**排查步骤：**
```bash
# 1. 以控制台模式运行查看错误
java -jar miduo-bootstrap\target\miduo-ccas-launcher.jar

# 2. 检查日志文件
type logs\application.log

# 3. 检查端口占用
netstat -ano | findstr :8080
```

### ❌ 问题 3：中文显示乱码

**解决方案：**
1. 确保 VM options 中包含 `-Dfile.encoding=UTF-8`
2. 检查配置文件编码是否为 UTF-8
3. Windows 系统区域设置改为 UTF-8

### ❌ 问题 4：exe4j 提示 "No suitable JVM found"

**解决方案：**
1. 确认已安装 JDK 11
2. 在 exe4j 中手动指定 JRE 路径
3. 或使用 bundled JRE 方式

### ❌ 问题 5：数据库连接失败

**检查项：**
```properties
# 检查 application.properties
spring.datasource.url=jdbc:jtds:sqlserver://192.168.5.22:1433/cshTest_db
spring.datasource.username=test2016
spring.datasource.password=test2016
```

**测试连接：**
```bash
# 使用 telnet 测试数据库端口
telnet 192.168.5.22 1433
```
