# 主界面数据接收区规则

> 赋码采集关联系统 - 主界面数据接收区展示与过滤规则说明

---

## 1. 概述

主界面数据接收区用于实时展示设备采集数据，采用 ListView 组件，遵循以下展示与过滤规则。

---

## 2. 数量限制

| 规则项 | 说明 |
|--------|------|
| 最大条数 | **1000 条** |
| 超出处理 | 超过 1000 条时，删除列表末尾最旧的数据 |
| 实现位置 | `MainController.addColoredTextToDataReceive()` |

```java
// 限制数据接收区最多显示1000条数据，超过则删除最旧的数据（列表末尾）
final int MAX_DATA_COUNT = 1000;
if (dataReceiveList.getItems().size() > MAX_DATA_COUNT) {
    dataReceiveList.getItems().remove(MAX_DATA_COUNT, dataReceiveList.getItems().size());
}
```

---

## 3. 设备数据过滤

### 3.1 直接显示

以下设备类别数据**直接**添加到数据接收区：

| 类别代码 | 设备名称 |
|----------|----------|
| 3 | 托盘码关联设备 |
| 4 | 箱码关联设备 |
| 5 | 报警器 |
| 6 | 剔除设备 |

### 3.2 延迟显示

以下设备数据**不在此处直接显示**，延迟到获取状态后再显示：

| 类别代码 | 设备名称 | 说明 |
|----------|----------|------|
| 1 | 码校验设备 | 通过校验接口返回后，再调用 `addColoredTextToDataReceive` 显示 |
| 2 | 箱码采集设备 | 同上，获取采集状态后显示 |

### 3.3 不显示

| 类别代码 | 设备名称 | 说明 |
|----------|----------|------|
| 7 | 扫码枪 | 完全不显示在数据接收区，仅自动填入查询码页面或码替换页面 |

---

## 4. 显示顺序与交互

| 规则项 | 说明 |
|--------|------|
| 排序 | 最新数据在顶部（索引 0），旧数据在下 |
| 滚动 | 新数据添加后自动滚动到顶部 |
| 可视区域 | ListView 高度 400px（prefHeight/maxHeight），需滚动查看历史记录 |
