# 前端启动指南

## 📋 前提条件

### 1. 环境要求
- **Java**: JDK 17 或更高版本
- **JavaFX**: 17.0.2（已包含在依赖中）
- **操作系统**: Windows / macOS / Linux

### 2. 后端服务
前端需要连接后端服务，请确保后端已启动：
```bash
# 启动后端（在项目根目录）
cd miduo-bootstrap
mvn spring-boot:run

# 或运行MiduoBackendApplication.java
```

后端默认地址：`http://localhost:8080`

## 🚀 启动方式

### 方式1: 使用IDE直接运行（推荐）

#### IDEA / Eclipse:
1. 打开项目
2. 找到以下任一类：
   - `com.miduo.cloud.frontend.FrontendMain` （推荐）
   - `com.miduo.cloud.frontend.MiduoFrontendApplication`
3. 右键 → Run 'FrontendMain.main()' 或 Run 'MiduoFrontendApplication.main()'

**注意**：如果遇到JavaFX运行时错误，需要配置VM参数：
```
--module-path "path/to/javafx-sdk/lib" 
--add-modules javafx.controls,javafx.fxml
```

### 方式2: 使用Maven运行

```bash
# 进入前端模块目录
cd miduo-frontend

# 编译
mvn clean compile

# 运行（需要配置javafx-maven-plugin）
mvn javafx:run
```

### 方式3: 打包后运行

```bash
# 打包整个项目
cd d:\javaspace\miduo-basebackend
mvn clean package -DskipTests

# 运行前端JAR（需要配置Main-Class和JavaFX依赖）
java -jar miduo-frontend/target/miduo-frontend-1.0.0-SNAPSHOT.jar
```

### 方式4: 使用命令行直接运行

```bash
# 在项目根目录
cd d:\javaspace\miduo-basebackend

# 编译所有模块
mvn clean compile

# 运行前端Main类
mvn exec:java -pl miduo-frontend -Dexec.mainClass="com.miduo.cloud.frontend.FrontendMain"
```

## ⚙️ 配置说明

### 修改后端地址

如果后端不在localhost:8080，需要修改：

**文件位置**: `miduo-frontend/src/main/java/com/miduo/cloud/frontend/util/HttpUtil.java`

```java
public class HttpUtil {
    // 修改这里的BASE_URL
    private static final String BASE_URL = "http://localhost:8080";
    
    // 或改为：
    private static final String BASE_URL = "http://192.168.1.100:8080";
}
```

### 窗口大小调整

**文件位置**: `miduo-frontend/src/main/java/com/miduo/cloud/frontend/MiduoFrontendApplication.java`

```java
@Override
public void start(Stage primaryStage) throws Exception {
    // 修改这里的窗口大小
    Scene scene = new Scene(root, 1400, 900);  // 宽度, 高度
    
    // 启动时最大化
    primaryStage.setMaximized(true);  // 改为true
}
```

## 🔧 常见问题

### 问题1: JavaFX运行时错误

**症状**: 
```
Error: JavaFX runtime components are missing
```

**解决方案**:
1. 确认JDK版本为17或更高
2. 检查Maven依赖是否正确下载
3. 在IDE中配置VM参数（见方式1）

### 问题2: FXML加载失败

**症状**:
```
javafx.fxml.LoadException: /fxml/MainWindow.fxml
```

**解决方案**:
1. 检查FXML文件是否存在于 `src/main/resources/fxml/` 目录
2. 确认Maven已正确复制资源文件
3. 运行 `mvn clean compile` 重新编译

### 问题3: 后端连接失败

**症状**: 
界面上操作无响应，控制台显示连接错误

**解决方案**:
1. 检查后端服务是否已启动：`http://localhost:8080`
2. 检查防火墙设置
3. 查看后端日志确认API是否正常

### 问题4: CSS样式未加载

**症状**: 
界面显示但样式混乱

**解决方案**:
1. 检查CSS文件是否存在于 `src/main/resources/css/` 目录
2. 检查FXML文件中的CSS引用路径
3. 运行 `mvn clean compile` 重新编译

### 问题5: 设备连接错误

**症状**: 
串口/TCP/UDP设备连接失败

**解决方案**:
1. 检查设备是否已连接
2. 检查串口号是否正确（Windows: COM1-COM9, Linux: /dev/ttyS0）
3. 确认设备驱动已安装
4. 检查端口权限（Linux需要sudo或加入dialout组）

## 🎮 功能模块

启动后可以使用以下功能：

### 1. 主界面 (MainWindow)
- 订单选择和任务管理
- 设备数据实时采集
- 码管理（采集、验证、拒绝）
- 统计数据显示
- 操作日志和报警信息

### 2. 系统配置 (SystemConfig)
- IO设备管理（串口、TCP、UDP）
- 设备参数配置
- 设备状态监控

### 3. 任务管理 (TaskManagement)
- 生产订单查询
- 订单添加、编辑、删除
- 订单状态更新

### 4. 码管理
- **码查询**: 查询箱码和垛码信息
- **码替换**: 替换错误的箱码
- **调整箱数**: 智能调整当前垛箱数
- **强制满垛**: 补充虚拟码完成当前垛
- **删除无码**: 删除当前垛无码记录

## 💡 开发提示

### 添加新界面

1. 创建FXML文件：`src/main/resources/fxml/NewView.fxml`
2. 创建Controller：`src/main/java/.../controller/NewViewController.java`
3. 在主界面或菜单中添加打开逻辑

### 调试技巧

1. **查看控制台输出**: 所有日志都会输出到控制台
2. **使用System.out.println**: 在Controller中添加调试输出
3. **检查网络请求**: 查看HttpUtil的请求和响应日志
4. **设备连接日志**: DeviceConnectionManager会输出详细的连接信息

### 性能优化

1. **异步操作**: 所有网络请求已在新线程执行
2. **UI更新**: 使用Platform.runLater()确保线程安全
3. **设备轮询**: 避免过高的轮询频率
