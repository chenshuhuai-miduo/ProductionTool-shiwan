# P02-04 / P02-05 / P02-06 / P02-07 功能实现方案

## 已确认的关键业务决策

- **产品名称来源**：JOIN `ProductInfo` 表，按 `ProductNO` 关联
- **RejectRecord**：`BottleCode` 字段存单个瓶码，一条记录=一行展示
- **取消关联/数据替换**：物理删除（`DELETE FROM`），不再用 `IsDel=1` 软删除
- **上传统计**：只展示 `IsUpload=1`（成功）和 `IsUpload=2`（异常），日期过滤用 `UploadTime`
- **剔除数统计**：`COUNT(DISTINCT CaseCode)` 统计被剔除的箱数
- **弹窗实现**：新建独立 FXML + Controller（两个弹窗）
- **弹窗继承筛选条件**：垛码列表弹窗/剔除记录弹窗继承当前生产统计查询的日期+生产单号条件

## 现状总结

- 前端 `[ShiwanM2QueryController.java](miduo-frontend/src/main/java/com/miduo/cloud/frontend/controller/ShiwanM2QueryController.java)` — 全部模拟数据
- 前端 `[ShiwanM2StatsController.java](miduo-frontend/src/main/java/com/miduo/cloud/frontend/controller/ShiwanM2StatsController.java)` — 统计写死，弹窗用 Alert 文本
- 前端 `[ShiwanM2CancelAssociationController.java](miduo-frontend/src/main/java/com/miduo/cloud/frontend/controller/ShiwanM2CancelAssociationController.java)` — 调用珠美斋 `/api/code/query` 和 `/api/code/delete-by-box-code`，在石湾M2后端不存在（404）
- 前端 `[ShiwanM2DataReplaceController.java](miduo-frontend/src/main/java/com/miduo/cloud/frontend/controller/ShiwanM2DataReplaceController.java)` — 调用珠美斋 `/api/code/replace`，在石湾M2后端不存在（404）
- 石湾M2后端不扫描 `com.miduo.cloud.variant.zhimeizhai` 包，上述珠美斋接口全部 404

## 数据库字段对应关系

**CodeRelationUpload（码关系上传表）**

- 瓶码 = `SmallSerialNumber`（L1）
- 盒码 = `MediumSerialNumber`（L2）
- 箱码 = `BigSerialNumber`（L3）
- 垛码 = `BiggerSerialNumber`（L4，成垛后写入的虚拟垛标）
- 上传状态 = `IsUpload`（0=未上传, 1=已上传, 2=上传失败, 3=上传中）
- 失败原因 = `Msg`

**RejectRecord（剔除记录表，已更新）**

- `CaseCode`：箱码（箱级剔除有值，瓶级剔除也有值）
- `BoxCode`：盒码（箱级剔除为空）
- `BottleCode`：瓶码（箱级剔除为空）
- `ProblemCode`：问题码（即导致剔除的那个码）
- `RejectReason`：剔除原因

## 一、P02-04 数据查询

### 后端接口

新增 `GET /api/shiwan-m2/code/query?code={code}` 至 `[ShiwanM2BoxCaseController](miduo-controller/src/main/java/com/miduo/cloud/controller/ShiwanM2BoxCaseController.java)`

在 `[ShiwanM2BoxCaseService](miduo-application/src/main/java/com/miduo/cloud/application/shiwan/ShiwanM2BoxCaseService.java)` 新增 `queryCodeAssociation(String code)` 方法：

**Step 1** — 定位垛码：

```sql
SELECT BiggerSerialNumber FROM CodeRelationUpload
WHERE (SmallSerialNumber=? OR MediumSerialNumber=? OR BigSerialNumber=? OR BiggerSerialNumber=?)
  AND BiggerSerialNumber IS NOT NULL AND BiggerSerialNumber != ''
LIMIT 1
```

**Step 2** — 展开全垛数据，JOIN ProductInfo 获取产品名：

```sql
SELECT c.SmallSerialNumber, c.MediumSerialNumber, c.BigSerialNumber,
       c.BiggerSerialNumber, c.AddTime, c.ProductNO, c.OrderNo,
       p.ProductName
FROM CodeRelationUpload c
LEFT JOIN ProductInfo p ON p.ProductNo = c.ProductNO
WHERE c.BiggerSerialNumber = ?
ORDER BY c.BigSerialNumber, c.MediumSerialNumber, c.SmallSerialNumber
```

响应：`{ rows: [{bottleCode,boxCode,caseCode,palletCode,addTime,productNo,productName,orderNo}], total }`

### 前端改造

`[ShiwanM2QueryController.java](miduo-frontend/src/main/java/com/miduo/cloud/frontend/controller/ShiwanM2QueryController.java)`：

- `doQuery(code)` → `HttpUtil.asyncGet("/api/shiwan-m2/code/query?code="+code, ...)`
- 行数据改为带 detail 字段的对象（或 Map），`showDetail()` 解析真实字段

## 四、P02-07 生产统计

### 后端接口（新增4个）

**① 生产统计汇总**

```
GET /api/shiwan-m2/stats/production-summary?startDate=&endDate=&orderNo=
```

```sql
-- 垛数
SELECT COUNT(DISTINCT BiggerSerialNumber) FROM CodeRelationUpload
WHERE BiggerSerialNumber != '' AND AddTime BETWEEN ? AND ? [AND OrderNo = ?]
-- 箱数
SELECT COUNT(DISTINCT BigSerialNumber) FROM CodeRelationUpload WHERE ...
-- 盒数
SELECT COUNT(DISTINCT MediumSerialNumber) FROM CodeRelationUpload WHERE ...
-- 剔除数（统计被剔除的不重复箱数）
SELECT COUNT(DISTINCT CaseCode) FROM RejectRecord WHERE RejectTime BETWEEN ? AND ? [AND OrderNo = ?]
```

**② 垛码列表（分页+垛码筛选，继承日期/生产单号条件）**

```
GET /api/shiwan-m2/stats/pallet-list?startDate=&endDate=&orderNo=&palletCode=&page=1&pageSize=20
```

```sql
SELECT BiggerSerialNumber, COUNT(DISTINCT BigSerialNumber) AS caseCount,
       MAX(OrderNo) AS orderNo, MAX(AddTime) AS associateTime
FROM CodeRelationUpload
WHERE BiggerSerialNumber IS NOT NULL AND BiggerSerialNumber != ''
  AND AddTime BETWEEN ? AND ? [AND OrderNo = ?] [AND BiggerSerialNumber LIKE ?]
GROUP BY BiggerSerialNumber ORDER BY associateTime DESC LIMIT ? OFFSET ?
```

**③ 剔除记录（分页+箱码筛选，继承日期/生产单号条件）**

```
GET /api/shiwan-m2/stats/reject-records?startDate=&endDate=&orderNo=&caseCode=&page=1&pageSize=20
```

```sql
SELECT r.Id, r.CaseCode, r.BoxCode, r.BottleCode, r.ProblemCode,
       r.RejectReason, r.RejectTime, r.OrderNo,
       p.ProductNo, p.ProductName
FROM RejectRecord r
LEFT JOIN ProductInfo p ON p.ProductNo = (
    SELECT ProductNO FROM CodeRelationUpload WHERE OrderNo = r.OrderNo LIMIT 1
)
WHERE r.RejectTime BETWEEN ? AND ? [AND r.OrderNo = ?] [AND r.CaseCode LIKE ?]
ORDER BY r.RejectTime DESC LIMIT ? OFFSET ?
```

**④ 上传记录列表（分页+状态筛选，日期过滤用 UploadTime）**

```
GET /api/shiwan-m2/stats/upload-records?startDate=&endDate=&orderNo=&status=&page=1&pageSize=20
```

```sql
SELECT BiggerSerialNumber AS palletCode,
       COUNT(DISTINCT BigSerialNumber) AS caseCount,
       MAX(OrderNo) AS orderNo,
       MAX(UploadTime) AS uploadTime,
       MAX(IsUpload) AS isUpload,
       MAX(Msg) AS errorMsg
FROM CodeRelationUpload
WHERE BiggerSerialNumber IS NOT NULL AND BiggerSerialNumber != ''
  AND UploadTime BETWEEN ? AND ?  [AND OrderNo LIKE ?]
GROUP BY BiggerSerialNumber
HAVING MAX(IsUpload) IN (1, 2) [AND MAX(IsUpload) = ?]
ORDER BY MAX(UploadTime) DESC LIMIT ? OFFSET ?
```

状态映射：成功=`IsUpload=1`，异常=`IsUpload=2`

`[ShiwanM2StatsController.java](miduo-frontend/src/main/java/com/miduo/cloud/frontend/controller/ShiwanM2StatsController.java)`：
