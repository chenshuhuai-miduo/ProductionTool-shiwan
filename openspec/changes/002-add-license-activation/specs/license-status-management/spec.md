# 规范：授权状态管理

## ADDED Requirements

### REQ-STATUS-001: 授权状态定义

#### Scenario: 5种授权状态
**Given** 系统需要确定当前授权状态  
**When** 系统检查许可证  
**Then** 系统应识别以下5种状态：
1. **未激活**：无任何许可证文件
2. **试用期内**：存在有效的试用许可证（expireDate >= 当前日期）
3. **试用已过期**：存在试用许可证但已过期（expireDate < 当前日期）
4. **已激活**：存在有效的正式许可证（expireDate >= 当前日期）
5. **已过期**：存在正式许可证但已过期（expireDate < 当前日期）

**状态优先级**：
1. 首先检查正式许可证
   - 如果存在且有效 → 已激活
   - 如果存在但过期 → 已过期
   - 如果不存在或无效 → 继续检查试用许可证
2. 然后检查试用许可证
   - 如果存在且有效 → 试用期内
   - 如果存在但过期 → 试用已过期
   - 如果不存在 → 未激活

**实现要求**：
- 状态判断逻辑清晰，优先级明确
- 状态枚举定义：LicenseStatusEnum（UNACTIVATED, TRIAL, TRIAL_EXPIRED, ACTIVATED, EXPIRED）

### REQ-STATUS-002: 状态栏显示

**Given** 主界面已加载  
**When** 系统更新状态栏  
**Then** 状态栏应显示：
- **未激活**：显示"未激活"（灰色 #757575），图标：○

  - 剩余 ≤ 3天：显示"试用剩余: XX天"（红色 #d32f2f），图标：⚠️
- **试用已过期**：显示"试用已过期"（红色 #d32f2f），图标：❌
- **已激活**：
  - 剩余 > 30天：显示"激活剩余: XX天"（绿色 #2e7d32），图标：✓
  - 剩余 30-7天：显示"激活剩余: XX天"（橙色 #f57c00），图标：⚠️
  - 剩余 7-1天：显示"激活剩余: XX天"（红色 #d32f2f），图标：⚠️
  - 剩余 < 1天：显示"激活剩余: X小时"（红色 #d32f2f），图标：⚠️
- **已过期**：显示"已过期"（红色 #d32f2f），图标：❌

### REQ-STATUS-003: 许可证信息页面状态显示

#### Scenario: 许可证信息页面根据状态显示不同内容
**Given** 用户打开许可证信息页面  
**When** 系统加载页面  
**Then** 页面应根据授权状态显示：
- **未激活状态**：
  - 状态标签：[○未激活]（灰色）
  - 提示文字："请联系米多专员获取许可证"
  - 设备信息显示
  - [激活]按钮
- **试用期内状态**：
  - 状态标签：[●试用中] 试用剩余: XX天（蓝色）
  - 许可类型显示："试用（第X次/最多2次）"
  - 设备信息显示
  - [激活正式版]按钮
- **试用已过期状态**：
  - 状态标签：[●试用已过期]（红色）
  - 如果试用次数 < 2：显示 [续试用] 和 [激活正式版] 按钮
  - 如果试用次数 = 2：仅显示 [激活正式版] 按钮，提示"试用已达上限"
- **已激活状态**：
  - 状态标签：[●已激活] 剩余XXX天（绿色/橙色/红色）
  - 授权信息显示（激活时间、到期时间、有效期）
  - 设备信息显示
  - [续期]按钮
- **已过期状态**：
  - 状态标签：[●已过期]（红色）
  - 授权信息显示
  - 设备信息显示
  - [续期]按钮

**实现要求**：

- 状态切换逻辑清晰
- 按钮行为正确
