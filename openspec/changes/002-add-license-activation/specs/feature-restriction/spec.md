# 规范：功能限制策略

## ADDED Requirements

### REQ-RESTRICTION-001: 功能限制守卫

#### Scenario: 许可证守卫类
**Given** 系统需要检查功能可用性  
**When** 调用许可证守卫类  
**Then** 系统应提供：
- `checkLicenseStatus()` - 检查当前授权状态
- `isFeatureAvailable()` - 检查功能是否可用
- `requireValidLicense()` - 要求有效许可证（如果无效则抛出异常）
- `getRemainingDays()` - 获取剩余天数

**实现要求**：
- 守卫类：`LicenseGuard.java`
- 异常类：`LicenseException.java`（授权异常）
- 异常消息清晰明确

#### Scenario: 任务启用前检查
**Given** 用户点击"启用任务"按钮  
**When** 系统执行任务启用  
**Then** 系统应：
- 调用许可证守卫检查授权状态
- 如果授权状态为"未激活"或"已过期"：
  - 阻止任务启用
  - 显示授权提示对话框
  - 引导用户激活或续期
- 如果授权状态为"试用期内"或"已激活"：
  - 允许任务启用
  - 正常执行任务启用流程

**实现要求**：
- 在`MainController.enableTask()`方法中添加检查
- 检查失败时显示友好的提示信息

#### Scenario: 码采集前检查
**Given** 设备接收到码数据  
**When** 系统处理码采集  
**Then** 系统应：
- 调用许可证守卫检查授权状态
- 如果授权状态为"未激活"或"已过期"：
  - 拒绝码采集
  - 记录拒绝原因到操作日志
  - 显示授权提示（可选）
- 如果授权状态为"试用期内"或"已激活"：
  - 允许码采集
  - 正常执行码采集流程

**实现要求**：
- 在码采集入口处添加检查
- 检查失败时不中断设备连接，仅拒绝数据采集

#### Scenario: 数据上传前检查
**Given** 系统需要上传数据到标识平台  
**When** 系统执行数据上传  
**Then** 系统应：
- 调用许可证守卫检查授权状态
- 如果授权状态为"未激活"或"已过期"：
  - 阻止数据上传
  - 记录上传失败原因
  - 显示授权提示（可选）
- 如果授权状态为"试用期内"或"已激活"：
  - 允许数据上传
  - 正常执行数据上传流程

**实现要求**：
- 在数据上传入口处添加检查
- 检查失败时不影响本地数据存储

### REQ-RESTRICTION-002: 渐进式限制策略

#### Scenario: 未激活状态限制
**Given** 授权状态为"未激活"  
**When** 用户尝试使用功能  
**Then** 系统应：
- 所有功能不可用
- 显示未激活提示对话框（首次启动时）
- 引导用户联系米多专员获取许可证
- 允许用户查看许可证信息页面

**实现要求**：
- 功能限制明确
- 提示信息友好
- 提供激活入口

#### Scenario: 已过期状态限制
**Given** 授权状态为"已过期"（正式或试用）  
**When** 用户尝试使用功能  
**Then** 系统应：
- 所有功能不可用
- 显示过期提示对话框（启动时）
- 引导用户续期或激活
- 对话框不可关闭（必须续期或退出）

**实现要求**：
- 功能完全限制
- 对话框强制显示
- 提供续期入口

#### Scenario: 试用期内/已激活状态
**Given** 授权状态为"试用期内"或"已激活"  
**When** 用户使用功能  
**Then** 系统应：
- 所有功能正常可用
- 无功能限制
- 仅在状态栏显示剩余天数

**实现要求**：
- 全功能可用
- 不限制任何功能

#### Scenario: 到期前提醒（不限制功能）
**Given** 授权状态为"试用期内"或"已激活"  
**When** 剩余天数达到提醒阈值  
**Then** 系统应：
- 功能正常可用（不限制）
- 显示提醒弹窗：
  - 试用剩余≤3天：显示试用到期提醒
  - 正式剩余30-7天：显示续期提醒（可关闭）
  - 正式剩余<7天：显示续期提醒（可关闭）
- 用户可以选择"稍后提醒"关闭弹窗

**实现要求**：
- 提醒不影响功能使用
- 提醒可关闭（试用≤3天除外）
- 提醒信息清晰

## MODIFIED Requirements

### MOD-MAIN-002: 主界面功能入口添加授权检查

#### Scenario: 关键功能入口添加授权检查
**Given** 主界面已实现  
**When** 集成功能限制策略  
**Then** 主界面应在以下位置添加授权检查：
- 任务启用按钮（`enableTask()`）
- 码采集处理（`handleDeviceData()`）
- 数据上传（`DataUploadController`）
- 其他关键功能入口

**实现要求**：
- 使用统一的`LicenseGuard`类
- 检查失败时显示友好的提示
- 不影响现有功能逻辑
