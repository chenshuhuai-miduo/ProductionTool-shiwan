# 规范：许可证验证（客户端）

## ADDED Requirements

### REQ-LICENSE-VAL-001: 许可证文件验证

#### Scenario: 应用启动时验证许可证
**Given** 应用启动  
**When** 系统读取本地许可证文件  
**Then** 系统应：
- 检查许可证文件是否存在（%APPDATA%\miduo\license\production_line.lic）
- 如果不存在：授权状态 = 未激活
- 如果存在：
  - Base64解码文件内容
  - 验证文件完整性（可选MD5校验）
  - 验证RSA签名（使用嵌入的公钥）
  - 如果签名无效：提示许可证损坏，需要重新激活
  - 如果签名有效：继续验证
  - AES256解密payload
  - 验证设备ID是否匹配（与当前设备ID比较）
  - 如果设备ID不匹配：显示设备不匹配提示，授权状态 = 设备不匹配
  - 如果设备ID匹配：继续验证
  - 验证有效期（检查expireDate与当前日期比较）
  - 计算剩余天数
  - 确定授权状态（未激活/试用期内/试用已过期/已激活/已过期）

**实现要求**：
- 验证顺序：文件存在 → 签名验证 → 设备匹配 → 有效期验证
- 任何一步验证失败，停止后续验证并返回错误
- 验证耗时 < 100ms

#### Scenario: 导入许可证时验证
**Given** 用户选择许可证文件  
**When** 系统导入许可证  
**Then** 系统应：
- 读取许可证文件
- 执行完整的验证流程（同启动时验证）
- 如果验证通过：
  - 保存许可证文件到本地（%APPDATA%\miduo\license\production_line.lic）
  - 显示激活成功界面（显示许可类型、激活时间、到期时间、有效期）
  - 更新授权状态
- 如果验证失败：
  - 显示错误提示（文件格式错误/签名无效/设备不匹配/已过期等）
  - 不保存许可证文件

**实现要求**：
- 支持文件选择对话框
- 支持拖拽文件（可选）
- 错误提示清晰明确

### REQ-LICENSE-VAL-002: 设备匹配验证

#### Scenario: 验证设备ID匹配
**Given** 许可证文件已解密  
**When** 系统验证设备匹配  
**Then** 系统应：
- 获取许可证中的deviceId
- 获取当前设备的deviceId（重新生成）
- 比较两个deviceId是否相等
- 如果相等：验证通过
- 如果不相等：验证失败，显示设备不匹配提示

**实现要求**：
- 设备ID比较使用字符串相等比较
- 设备不匹配时，显示当前设备信息，引导重新激活

### REQ-LICENSE-VAL-003: 有效期验证

#### Scenario: 验证许可证有效期
**Given** 许可证文件已解密  
**When** 系统验证有效期  
**Then** 系统应：
- 获取许可证中的expireDate
- 获取当前日期（系统时间）
- 比较expireDate与当前日期
- 如果expireDate >= 当前日期：许可证有效
- 如果expireDate < 当前日期：许可证已过期
- 计算剩余天数：expireDate - 当前日期

**实现要求**：
- 使用LocalDate进行日期比较
- 剩余天数计算精确到天
- 如果剩余天数 < 1天，显示剩余小时数

### REQ-LICENSE-VAL-004: 时间防篡改检测

#### Scenario: 检测系统时间回拨
**Given** 应用启动  
**When** 系统验证许可证  
**Then** 系统应：
- 读取上次验证时间（注册表或文件）
- 比较当前时间与上次验证时间
- 如果当前时间 < 上次验证时间：检测到时间回拨
- 处理时间回拨：
  - 正式许可：显示警告"检测到系统时间异常"，建议校准系统时间，功能正常使用
  - 试用许可：显示警告"检测到系统时间异常"，以较早的记录时间为准计算试用期，防止延长试用期
- 记录当前验证时间（用于下次验证）

**实现要求**：
- 时间记录位置：注册表（HKEY_CURRENT_USER\Software\Miduo\ProductionLine）或文件
- 时间格式：时间戳（毫秒）
- 加密存储时间记录（可选）
