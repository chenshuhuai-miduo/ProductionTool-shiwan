## 问题描述

### 业务需求

当前产线采集关联软件缺少授权管理机制，无法控制软件的使用权限和有效期。需要建立完整的激活码授权体系，包括：

1. **授权状态管理**：支持未激活、试用期内、试用已过期、已激活、已过期5种状态
2. **离线激活机制**：适用于内网隔离的工控机环境，无需联网即可完成激活
3. **试用模式**：为产线调试期提供15天试用许可，同一设备最多2次试用机会
4. **功能限制策略**：根据授权状态渐进式限制功能使用

### 用户场景

1. **首次安装用户**：软件启动后显示未激活状态，引导用户联系米多专员获取许可证
2. **试用用户**：导入试用许可证后获得15天全功能试用，到期前可续试用1次或购买正式许可
3. **正式用户**：导入正式许可证后获得长期授权（默认365天），到期前可续期

**说明**：许可证生成由独立的后台管理系统完成，管理员在后台上传设备请求文件、生成许可证等功能不在本产线工具项目中实现。

## 解决方案概述

### 架构设计

采用纯客户端架构，所有功能在客户端本地完成：

- **客户端（JavaFX）**：
  - 设备信息采集和唯一码生成
  - 设备请求文件（.devreq）导出
  - 许可证文件（.lic）导入和验证
  - 授权状态管理和UI展示
  - 功能限制策略执行

**说明**：许可证生成由独立的后台管理系统完成，不在本产线工具项目中实现。客户端仅负责导出设备请求文件、导入许可证文件并进行本地验证。

### 核心功能模块

1. **授权状态模型**：5种状态及其转换逻辑
2. **设备唯一码生成**：基于主板序列号、CPU ID、制造商生成MD5
3. **离线激活流程**：导出.devreq → 后台生成.lic → 导入.lic
4. **许可证验证**：RSA签名验证、设备匹配、有效期检查
5. **试用管理**：试用次数限制、试用期提醒
6. **功能限制**：根据授权状态渐进式限制功能

### 技术方案

- **加密算法**：AES256加密 + RSA2048签名
- **密钥管理**：RSA公钥嵌入客户端JAR资源文件，用于验证许可证签名（RSA私钥在独立的后台管理系统）
- **文件格式**：JSON格式的.devreq和.lic文件，Base64编码
- **存储位置**：`%APPDATA%\miduo\license\`
- **时间防篡改**：记录上次验证时间，检测系统时间回拨

## 风险评估

### 技术风险

- **风险**：加密算法实现复杂，可能存在安全漏洞
- **缓解措施**：使用成熟的加密库（Java内置），充分测试，代码审查

- **风险**：设备唯一码可能因硬件变更而变化
- **缓解措施**：提供设备变更处理流程，支持重新激活

### 业务风险

- **风险**：试用次数限制可能影响用户体验
- **缓解措施**：明确提示试用规则，提供正式许可购买渠道

- **风险**：离线激活流程可能增加客服工作量
- **缓解措施**：后台管理系统（独立项目）提供便捷的许可证生成界面，支持批量处理
